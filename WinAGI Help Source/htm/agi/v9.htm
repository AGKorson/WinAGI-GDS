v9: Invalid Word Number

When a player enters a command by pressing the ENTER key, AGI parses the input. If an unknown word is encountered (one that is not in the WORDS.TOK file), the number of the word that failed is stored in reserved variable v9. Game programmers can then use the word.to.string command to access the word and provide feedback to the player.

If all words entered are recognized by the parser, v9 is set to zero. It is also set to zero in every interpreter cycle when no input is provided by the player.

It is important to remember that indexing of words begins at 1. i.e. number of first word is '1', not '0'. See the word.to.string command for a detailed discussion of how AGI deals with numbering of words when being accessed by commands.

Attempting to use this variable for other purposes is not recommended, because AGI overwrites it in each interpreter cycle.
