Variables

The variable data type is an unsigned eight-bit number equivalent to a byte, or unsigned char. There are 256 variables in memory numbered from 0 to 255. To AGI, the variables are identified by their index in the variable table. In most compilers, variables are represented in source code by the letter 'v' and their index number (i.e. v0, v1, etc.)

Variables are used to store and manipulate numeric data in support of the game, and are often used as arguments in arithmetic commands such as addition and multiplication. Many AGI commands have alternate versions that use variable parameters instead of hard coded numbers (constants) to provide great flexibility to the game programmer to manage the game flow.

Because the variable data type is unsigned, it is important for programmers to be careful when using arithmetic commands. AGI does not check for overflow conditions when doing arithmetic. So if the result of arithmetic is greater than 255 or a negative value, AGI returns the modulus, i.e.

result = (a +/- b) Mod 256)

As an example, if v100 = 10, and you add 250, the new value stored in v100 will be

10 + 250 = 260; 260 - 256 = 4

Similarly, if v100 = 15, and you subtract 22, the new value stored in v100 will be

15 - 22 = -7; -7 + 256 = 249

Reserved Variables

The first 27 variables (v0 - v26) are used internally by AGI, and should not be used for storing game data. Doing so may cause unexpected behavior by AGI, including potentially fatal errors. In some cases, a game programmer may need to directly manipulate these reserved variables to achieve specific game effects, but great care should be taken to avoid creating unexpected results or errors.
