Player Interface

In the early 1980s, when AGI was created, options for player interfaces were limited. The keyboard was the primary input device, but AGI also had some limited support for joysticks, and mice.

Ego

AGI games were almost always played in a third person perspective, meaning the main character appeared on the screen, and was controlled by the game player. The main character was known as 'ego', represented by the first (index number 0) screen object. Ego can be controlled in either player control mode or program control mode.

Player Control

By default, the game operates in player control mode. In this mode, ego could be moved around the screen by use of the keyboard, a mouse, or a joystick. The player is in control of ego, deciding when and where to move it.

Player control mode is restored every time a game is restarted, a new room is entered, or when ego reaches its destination when using move.obj or move.obj.v commands. The player.control and start.motion commands also put ego in player control mode.

Program Control

AGI also allows ego's motion to be controlled programmatically, just like other screen objects. While under program control, all player controls for ego are ignored, and ego can only be moved programmatically.

To switch to program control mode, you can call the program.control command, or change ego's movement mode using the move.obj, move.obj.v or wander commands. The stop.motion command also switches ego to program control.

 

Input Queue

From the player's perspective, there are three possible input mechanisms for player input: keyboard, joystick and the mouse (only on some non-MSDOS platforms). AGI handles all input by placing key presses, mouse movements/clicks, and joystick movement/buttons into a queue as they occur; AGI then processes each input to determine what to do with it. Internally, AGI converts all player input into one of three types of input data before adding it to the input queue:

    • keyboard key presses;
    • ego movement commands;
    • and controllers
      • .

In each interpreter cycle, AGI processes the data added to the queue and acts on it accordingly.

 

Keyboard Input

Keyboard input includes parsing text entered by the player, controlling ego direction, navigating menus, and activating controllers.

AGI uses modified system timer interrupts to read the system keyboard buffer, converts designated key presses into ego movement commands, and stores them an AGI specific queue. In each interpreter cycle, AGI then pulls key information from the AGI key queue, checks if any key presses are assigned to controllers, and then processes the keys, displaying them on the input line, if input is enabled.

AGI also uses the same queue to manage input for entering save game directories and descriptions, as well as the get.string, get.num commands. Unlike normal key processing, the function that processes input for these actions also include checks for CTRL+C and CTRL+X, clearing the line when these keys are pressed. With the input line, you need to assign a controller if you want that functionality.

 

Mouse Input

Mouse input included ego control, but only in some non-MSDOS versions. Currently there is very little information available about mouse support in AGI.

 

Joystick Input

If attached, a joystick can be used to control ego direction, navigate menus and select menu items. The buttons can also be programmed as controllers.

Similar to key presses, AGI uses timer interrupts to periodically poll the joystick, and adds ego movement commands and button presses (as key presses using special codes) to the queue. See the Joystick Input topic for details.
