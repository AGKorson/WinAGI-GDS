AGI Version 2

AGI Version 2 is the one most familiar to AGI enthusiasts and was used for the majority of Sierra's games. The main distinguishing features of version 2 support for the 16 color EGA card (but still keeping the original 160 x 200 resolution), separate directory files, no compression of resources, no game ID added to VOL file and DIR file names, and VOL files limited to 5 (VOL.0 - VOL.4).

There are fourteen known public releases of AGI version 2:

    • Version 2.089

      • The first version 2 interpreter was able to detect and run on the IBM PC, IBM PCJr and the Tandy 1000. It included support for the CGA and EGA video cards. In CGA mode, it defaulted to the first color palette of 'black-cyan-magenta-white'. There was no option to force computer type and monitor type with command line arguments. This version did not support the Hercules Graphics Card (HGC) and monitor type was not tracked in variable v26.

      Picture resources did not include the plot feature. Non-MSDOS systems did not support a global volume control for sounds. Views in this version did not use the mirroring feature. The OBJECT file was not encrypted.

      Space was allocated to store 40 keyboard controller assignments but there is a bug in the code that only allocates space for 30 controllers in the evaluation table. The number of strings available was 12. Message box windows could not be displayed without pausing game play. Black was the only background color that would properly display text on the graphics screen. The status line text used the phrase "Sound is on/Sound is off".

      This version included 155 action commands. The quit command had no arguments. The status command did not support selecting inventory items (they could only be listed). The restart.game command did not support a 'no prompt' mode. The offset for evaluation commands in the debugging tracer feature was set at 160. The distance command had a bug in it that returned incorrect values when distance was greater than 255.

      The number key pad '5' button did not stop ego movement; to stop required pressing the direction key matching current ego movement. The input line width was fixed and could not be modified. This version did not include menus. Scan start information was not added to saved games.

      Reserved variables only included v0 through v22, and reserved flags only included f0 through f12. The 'o#' message format code was not available.

      The said command did not support the "rol" special word group (group 9999).

       

    • Version 2.230

      • This version is only known to be used for a rare Christmas Demo game. It added support for view loop mirroring (by using the upper bit of each loop's cel count property).

      The graphics functions that handled adding/updating/removing screen objects to the screen were completely refactored to be more compact and efficient.

      Support for the "rol" word group (group 9999) was added.

      A bug was introduced in this version that stored a value of 3 in reserved variable v22 (sound channel count) regardless of the computer type.

      The status line text was changed to show "Sound: on/Sound: off" instead of "Sound is on/Sound is off".

      The compare.strings command was modified, but it introduced a bug (incorrectly handling strings with different lengths). The check used to verify a newly placed screen object was not colliding with edges, control lines and/or other screen objects was moved from the position and position.v commands to the draw command.

      The restart and restore functions were modified to close and then reopen all VOL files.

    • Version 2.272

      • The mirroring feature for view loops was modified to use the transparency byte in each cel's header instead of the cel count for each loop.

      Use of the '5' key on the number keypad to stop ego motion was added. Beginning in this version scan start information was added to saved games.

      Version 2.272 increased the number of commands to 161, including all the menu commands, but none of them did anything; they were just placeholder commands so menus were still unavailable. The quit command was modified to use a byte value argument, allowing the game programmer to give the player an option to cancel the quit action.

      Reserved variables v23 and v24 were added.

       

    • Version 2.411

      • Support for the HGC video card was added. The overlay files for managing graphics routines was split into multiple files. Support for command line options was added.

      Version 2.411 introduced a limited implementation of the plot drawing commands for picture resources. Beginning with this version, the OBJECT file was encrypted.

      This version increased the number of commands to 170. The bug in the distance command was fixed.

      Menus were available for the first time. Multiplication and division commands were added. The offset for evaluation commands in the debugging tracer feature was increased to 240 to allow for the greater number of action commands.

      Changes in the text functions to support menus enabled both black and white as valid background colors when displaying text on the graphics screen. The bug that limited controllers to 30 was fixed, making all 50 controllers available. The number of strings increased from 12 to 24.

      Reserved variables v25 and v26, and reserved flags f13, f14 and f15 were added. The 'o#' message format code was added.

       

    • Version 2.425

      • In 2.425, the the plot drawing commands were expanded to allow for changing the size, shape and style of the plot pen, but there were several minor bugs in the implementation.

      Reserved flag f16 was added to allow automatic restarting. The bug in assigning reserved variable v22 (sound channel count) was fixed.

       

    • Version 2.426

      • The plot drawing function was refactored to fix the pen reset bug but did not fix the other two bugs.

       

    • Version 2.435

      • Another attempt to fix the bugs in the plot drawing functions, but still not completely successful.

      Minor changes were made to the PCJr sound functions to improve efficiency.

      File and disk error messages were adjusted to include extra line break between the error message and the 'Press ESC to quit' text.

       

    • Version 2.439

      • The code for the restore.game command was modified to check if a screen object has a valid view loaded before checking if the object should be animated.

      The plot command bug with missing data in the circle size 1 brush was finally fixed.

       

    • Version 2.440

      • In this version the compare.string bug that was introduced in version 2.272 was fixed.

       

    • Version 2.903

      • The next version with significant changes was 2.903, which added commands to manage scripts, set.simple and hold.key for a total of 174 commands. Note that the release.key command was not added until version 3.002.098.

      This version added support for the PS/2 computer with the VGA display. However, a minor bug was introduced in the video detection function that would incorrectly set the computer and monitor to the PS/2 with VGA for some PC models.

      The sound playback function for Tandy and PCJr sounds added a 'dissolve' feature that had each note start at a slightly higher volume and then quickly (after approximately one second) reduce to a slightly lower level for the duration of the tone.

       

    • Version 2.911

      • The code for reading in command line parameters was refactored, but this ended up introducing a bug which caused AGI to actually ignore ALL parameters.

      A disk reset command was added to the code used for opening all VOL files, although the reason why is not fully understood.

       

    • Version 2.912

      • This version removed the disk reset command added in version 2.911, presumably because they decided it wasn't really necessary after all.

       

    • Version 2.915

      • The bug in the video hardware check for PS/2 and VGA that was introduced in version 2.903, and the bug in the command line parameter function from version 2.911 were fixed.

      The message for disk errors when saving a game was modified to include write protection as a possible cause.

       

    • Version 2.917

      • This was one of the most popular versions used by Sierra. No less than six Sierra games were released with this interpreter version.

      In this version, the new room function was modified to close all VOL files. Also when calculating video offsets when drawing pixels, an additional check was added for Tandy when running on a non-EGA monitor.

       

    • Version 2.936

      • This was the last version 2 release. It added two commands, set.pri.base and discard.sound.

      The set.pri.base command included a minor bug in the code that determines the correct order to draw objects when updating the screen.

      This version also introduced a bug in the graphics update function that results in the auto loop feature changing loops for all views, regardless of the number of loops. In previous versions, views with more than four loops would not automatically update.
