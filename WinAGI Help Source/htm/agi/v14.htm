v14: Elapsed Game Time: Days

Along with v11 (seconds), v12 (minutes) and v13 (hours), AGI uses these four variables to track total runtime of a game session. Game programmers can access them to manage elements of their game that are based on time, such as having an event happen after a certain amount of time has gone by.

AGI updates its internal clock (and the timing variables) in each interpreter cycle, and will only pause the internal clock for the pause, save and restore commands.

The likelihood that anyone would play continuously until the day counter (v14) changed is ridiculously small, but if they do, AGI will track it.

Attempting to use these variables for other purposes is not recommended, because AGI overwrites them contiguously. (You could probably get away with using v14, as it would take 24 hours of continuous play before AGI would overwrite it.)
