Joystick Input

In addition to keyboard input, AGI contained support for joysticks for most platforms as a means to move the ego character around the screen. The Macintosh and Amiga versions of AGI do not appear to support the joystick (this has not been confirmed, though); this is probably because these platforms had a mouse instead.

To use the joystick in MSDOS versions, AGI first needs to initialize it; this is done with the init.joy command. This command verifies a joystick is attached, and if one is found, it prompts the player to center the joystick in order to calibrate it.

Joystick Movement

Once calibrated, AGI maintains an internal counter that is set to nine CPU clock ticks. In each interpreter cycle, if enough time has passed (at least nine ticks), AGI polls the joystick to see if it is moved far enough away from center to generate a movement command.

When the joystick is polled and change in direction is detected, AGI converts the joystick position into the corresponding ego movement direction value and adds that value to the input queue.

Joystick Buttons

Unlike joystick position, AGI polls the status of joystick buttons in every interpreter cycle. Joystick button presses get added to the input queue in the same format as a key-press. However, instead of a ASCII value or scan code, a special two-byte code is added. The first byte always has a value of 1, and the second byte is 1, 2, 3 or 4, to represent single click of A button, single click of B button, 'double-click' of A button, or 'double-click' of B button respectively.

AGI includes the ability to detect and respond to 'double-clicks' of the joystick buttons. If reserved flag f8 (double click enabled) is set (TRUE), double-click detection is enabled. If f8 is not set (FALSE), all button presses are detected as single clicks.

The delay time which determines whether a second button press is treated as a new single click or as a double-click is set by reserved variable v15 (double-click speed). The delay time is 0.05 seconds times the value of v15. Setting v15 to 0 has the same effect as setting flag f8 to FALSE (i.e. all button presses are treated as single clicks).

During the regular interpreter cycle, joystick button presses do not have any built-in functionality; they must be assigned to a controller or they are ignored.

When a dialog box or menu is displayed, the A button (single or double-click) is automatically treated the same as pressing the RETURN key, and the B button (single or double-click) is automatically treated as pressing the ESC key. Joystick movement will change the highlighted menu item, the same as the ego movement keys.
