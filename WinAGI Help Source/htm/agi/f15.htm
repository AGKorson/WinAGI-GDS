f15: Message Windows Don't Pause Game

When f15 is reset to FALSE, message windows (such as when using the print commands) will pause the game (but not the internal game clock), and the window will remain open until the player dismisses it by pressing ENTER or ESC, or until the time delay set by reserved variable v21 (window close time) passes. This is the normal, default behavior.

When f15 is to TRUE, the game will continue with regular interpreter cycles while the message window is displayed. Reserved variable v21 (window close time) is ignored. The message window will remain until another print command is entered, the graphics screen is re-displayed (by switching to text screen and then back to graphic screen), entering a new room or a close.window command is executed.

If screen objects on screen that are covered by the message window get updated in subsequent cycles, the message window will be overdrawn by the updates, so careful planning of the position of windows is needed when using this feature.

AGI resets f15 to FALSE after every message window is displayed, and after every show.pic command. This means that every time you want a message to remain visible without pausing game play, you must set f15.

This flag was not reserved by AGI and had no effect in version 2.089 and version 2.272.

Attempting to use this flag for other purposes is not recommended; it gets reset to FALSE every time a message window is shown, which would make it effectively useless unless messages are never displayed, which is not practical in AGI.
