Strings

The string data type is used to hold small amounts of text that may change throughout the course of a game. This makes them different from messages; messages cannot be altered by AGI at runtime; they are read-only.

The number of strings available in AGI is 24 (s0 - s23), except for versions 2.089, 2.272 and 3.002.149, which have a limit of 12 (s0 - s11). Currently, there is no explanation as to why these versions have a different string count.

Each string in AGI can be up to 40 character long (including a null terminator).

Strings are represented in source code by the letter 's' and their index (i.e. s0, s1, etc.) String values can be changed directly by the player through keyboard input (get.string command), or by converting messages or words typed by the player (set.string and word.to.string commands).

There are only a few commands that use strings, including the parse command and compare.strings test command in addition to those already mentioned.

Reserved String s0

The first string, s0, is special in that it is reserved for the player input prompt. See the Input Prompt and Cursor topic for more information.
