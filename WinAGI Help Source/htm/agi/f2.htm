f2: Text Input Entered by Player

When AGI parses player input (after the player presses the ENTER key), AGI sets this flag to TRUE if at least one valid word is found, or if any unknown words are found. By checking for this flag, a game programmer is able to respond to user input in code.

When the said command is called, AGI first checks to see if the player has entered anything to test. If flag f2 is FALSE, the said command returns FALSE without doing anything else.

At the beginning of each interpreter cycle, and every time a new.room command is called, AGI resets f2 to FALSE. AGI also resets f2 to FALSE when the parse command is called, before it parses the target string.

Flag f2 and flag f4 (said found match) are used together by AGI to manage evaluation of player input. When f2 (input received) is TRUE, the said command will check player input for the words passed as arguments. If a match is found, AGI then sets f4 to TRUE, so future calls to said command in the same interpreter cycle exit without doing anything.

If you are writing advanced code, you may need to manipulate the values of f2 and f4 to achieve certain effects. For example, after a call to the said command returns TRUE, you can reset f4 to FALSE so that subsequent said commands will also look for a match.

Attempting to use this flag for other purposes is not recommended, because AGI overwrites it in every interpreter cycle.
