f9: Sound Enabled

When flag f9 is set to TRUE, AGI activates a vector function that will output sound through the system's sound hardware when the sound command is called. The sound is played asynchronously, meaning it plays in the background while AGI continues to run through interpreter cycles.

When flag f9 is reset to FALSE, calls to sound still activate the function that plays a sound, but it immediately stops, and sets the 'done' flag to TRUE.

If a sound is currently playing when f9 is reset to FALSE, AGI stops playing the sound and the 'done' flag for the sound is set to TRUE, the same as if the sound played through to the end and stopped normally. See the sound resource topic and sound command topic for more information.

When a game is restarted or restored, the sound setting before the restart/restore is retained.

Each interpreter cycle, AGI checks the value of f9; if it has changed, AGI will update the sound status display in the status line, if the status line is visible. "Sound: on" is shown if f9 is TRUE, and "Sound: off" is shown if f9 is FALSE.

If you are not using any sound in your game, you can use this flag for any purpose. Just keep in mind that it may not have the correct value after restarting or restoring a game. Also keep in mind that its value will be displayed on the status bar if the status bar is visible.
