f0: Ego on Water

AGI sets this flag during every interpreter cycle when updating the position of ego. If the entire baseline of ego is on water (pixels with priority value of 3), this flag is set to TRUE. If one or more pixels have any other priority value, this flag is set false.

This can be useful to know when ego is on a water area in your game, which you can then respond to in code, for example, to switch to a 'swimming' view.

Note that if ego is not animated, it does not get updated. So flag f0 won't be modified during interpreter cycles when ego is not animated.

Attempting to use this flag for other purposes is not recommended, because AGI overwrites it in every interpreter cycle when updating ego. If you never use ego in your game, then you could use this flag without any concerns.
