Player Entered Words

This data type refers to an array of numbers and a matching array of text entries that correspond to the parsed list of words that are typed by a player. Word values are represented in source code by the letter 'w' and their index (i.e. w0, w1, w2, etc.)

This data type should not be confused with the vocabulary words list that is provided in the WORDS.TOK resource file, although it is related. The parser puts the text of each parsed word in the word text array and compares them to the words in WORDS.TOK. The word number array is filled with the numbers that correspond to entries in the WORDS.TOK list. The word number array is used internally by AGI when the said command is processed. Arguments in the said command are compared against the numbers in the word number array to determine if player input matches.

The words data arrays are filled by AGI's parser each time the player presses the ENTER key. The arrays can hold up to 10 words; if more than 10 are entered, only the first 10 are kept; the rest are discarded.

An important programming note regarding the use of player entered words in AGI commands is that the word values are referenced by their index, with the first word being index 0, the second being index 1, etc. This is somewhat confusing, since reserved variable v9 (unknown word number) and the print command format code for words expect the first word to be "w1", not "w0". For example, if the first word in the array is "LOOK", and the second word is "DOOR", passing a value of '1' to the word.to.string will assign the text value "DOOR" to a string, but using "%w1" in a message will result in the text value "LOOK" being inserted into the message.

WinAGI handles this by requiring word arguments to be 'one-based' in the logic editor, and then converts them to 'zero-based' on compilation, to maintain consistency with the print command format codes.
