AGI Version 3

The third and final version of the AGI interpreter came out for only a short while before the Sierra Creative Interpreter (SCI) hit the scene. Only a couple of games were written to use the version 3 interpreter, as Sierra began the switch to SCI around the same time version 3 was being introduced.

Beginning with version 3, the resource directory files were combined into a single file, and the directory file and VOL file names included the game ID. Up to 16 VOL files were available. Also, compression was used to reduce the size of resources (although the resource formats did not change). Version 3 also encrypted the OBJECT file before adding it to save game files.

There are five known public releases of AGI version 3:

    • Version 3.002.086

      • The first version 3 interpreter. This version increased the number of commands to 178, with addition of hide.mouse and allow.menu commands.

      The plot command bug for the size 1 circle brush used in plot commands reappeared because the data were modified again, but incorrectly. All version 3 interpreters would retain this bug.

      The sound dissolve length was slightly increased.

       

    • Version 3.002.098

      • Version 3.002.098 introduced three mouse related commands, although none of them had functionality in MSDOS. The release.key command was also added.

      It also fixed the bug in the auto loop feature, and flag f20 (force autoloop) was designated as a reserved flag.

      The bug for determining object drawing order in the set.pri.base command was fixed.

      The code that converts pixel position into the correct offset in video card memory was modified for Tandy computers with non-EGA video cards.

      There was a minor refactoring of the trace function, but it did not have any functional change.

       

    • Version 3.002.102

      • This version had no functional changes. Two data elements in the AGIDATA.OVL were moved to a different location.

       

    • Version 3.002.107

      • In this version, the only change was to the save and restore game functions. The code that reads the disk drive looking for existing saved games is run twice; once with all VOL files open, then again with all VOL files closed. The change doesn't do anything useful; it just wastes time repeating the same code. It is unclear why this change was made.

       

    • Version 3.002.149

      • In the last version of AGI made by Sierra, the number of strings allocated was reduced to 12, down from 24. Presumably this was done to increase available memory for other resources.

      The overlay files were refactored by moving code that was in IBM_OBJS.OVL and HGC_OBJS.OVL into the main executable file. HGC font data were moved from the HGC_FONT file into the HGC_GRAF.OVL overlay file.

      The close.dialogue and open.dialogue commands were disabled. The commands were directed to a null function (calls to these two commands had no effect).

      The insufficient memory error message was shortened from the descriptive "Sorry, your computer does not have enough memory to play this game. 256K of RAM is required." to just "ERROR: Insufficient memory."
