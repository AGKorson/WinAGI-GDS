v21: Message Window Delay Time

The value of reserved variable v21 determines how long message windows (such as those displayed by the print command) stay open when they are displayed. If the value is zero (default), the window will remain on screen until the player dismisses it by pressing the ENTER or ESC key, and the game is paused while the message window is displayed.

If non-zero, the window will automatically close after 0.5 * v21 seconds. The player can still press ENTER or ESC to dismiss the window earlier than that (and the game is still paused while the message window is displayed).

AGI restores v21 to a value of zero after every message window is displayed, unless flag f15 is TRUE. This means that every time you want a message to automatically disappear after a set time, you must reassign the desired delay time.

Note that if reserved flag f15 (no pause windows) is set to TRUE, then the value of v21 is ignored, and the display window will remain until it is dismissed by entering a new.room or a close.window command.

This behavior applies to message windows shown using the print and related commands, but also will affect a lot of other game related messages, such as the pause, quit, version command messages, save/restore selection messages and others.

Attempting to use this variable for other purposes is not recommended; it gets reassigned a value of zero every time a message window is shown, which would make it effectively useless unless messages are never displayed, which is not practical in AGI.
