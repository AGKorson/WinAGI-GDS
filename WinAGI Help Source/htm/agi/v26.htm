v26: Monitor Type

When AGI loads, if the MSDOS/PC platform is detected, AGI does an additional check to determine which monitor/video graphics card type is present. It then knows which specific graphics routines to call for displaying text and images correctly. AGI recognizes these display types:

Value Monitor Type
0 Color Graphics Adapter (CGA) with default Black-Cyan-Magenta-White palette (CGA)
1 CGA with alternate Blue-Green-Red-Brown palette (RGB))
2 Monochrome Hercules Graphics Card (HGC)
3 Enhanced Graphics Adapter (EGA)
4 Video Graphics Adapter (VGA; valid on PS/2 only)

See the Graphics Display topic for a detailed description of video display adapters that were supported by AGI.

The monitor type variable was often used in original Sierra games to provide customized features unique to a specific monitor. AGI maintains an internal copy of monitor type value that it uses, so as long as your code does not need to know the monitor type, using v26 for another purpose does not cause problems in AGI. Just be aware that every time the restore.game command is called, AGI resets the value of v26 to the current monitor type.

On non-MSDOS systems, it is believed that AGI does not use v26.

This variable was not reserved by AGI and had no effect in version 2.089 and version 2.272.
