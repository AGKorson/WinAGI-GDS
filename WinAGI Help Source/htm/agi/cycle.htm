The Interpreter Cycle

The heart of AGI is the interpreter cycle. Once the game starts and initializes AGI, it enters a loop that continually repeats until the player quits the game. During each cycle, AGI performs the following actions:

    • sets a delay by checking the internal clock until desired time has passed; the delay is n times 1/20 second, where n is the value of reserved variable v10 (animationInterval); for example, if v10==5, then there would be a quarter second (5/20) delay between cycles; AGI also updates the clock variables v11 (seconds), v12 (minutes), v13 (hours), and v14 (days) for each clock cycle; AGI then checks the stack (if it is blown, an error message is displayed and then AGI immediately ends);
    • clears the keyboard buffer and the controller activation table;
    • resets reserved flags f2 (input received) and f4 (said found match);
    • polls the joystick, if one is present;
    • clears reserved variables v19 (pressed key code) and v9 (unknown word number) by setting them to 0;
    • handles input on the input queue by:
      • storing key presses in reserved variable v19 (pressed key code);
      • changing ego direction if a direction control key is pressed - reserved variable v6 (ego direction) is set to the new direction, and, if under player control, movement mode is forced to normal motion;
      • setting the appropriate byte flag in the controller flag table to true if a controller input is read.
    • updates ego direction, based on current control mode:
      • player control: sets direction parameter of ego's screen object table entry to match the value of reserved variable v6 (ego direction)
      • program control: sets reserved variable v6 (ego direction) to match the value of the direction parameter of ego's screen object table entry
    • for all screen objects for which animate.obj , start.update and draw commands were issued, directions of motion are recalculated;
    • caches current score (v3) and sound state (f9);
    • Logic 0 is executed as well as any logics it calls which, in turn, can call other logics; if a new.room or new.room.v command is executed, the cycle is interrupted. All resources are removed from memory, various game parameters are reset, and the cycle continues by jumping back to execute logic 0 again. For a detailed description of the steps taken when AGI encounters a new room, see the new.room command;
    • the direction value in the screen object table entry for ego is set to the value in v6 regardless of control mode (which is why the set.dir command never works for ego);
    • if v3 (current score) or f9 (sound on) have changed (by comparing to values cached before logics were run), then status line is updated;
    • reserved variables v4 (object touching edge) and v5 (object edge code) are set to 0;
    • reserved flags f5 (new room), f6 (game restarted) and f12 (game restored) are all reset to FALSE;
    • if in graphics mode, all screen objects that are flagged for updating are updated (for each screen object, AGI sets the correct loop, moves it according to its movement mode, cycles it according to its cycle mode, and then redraw it on the graphics screen).

The cycle then repeats.

Pausing Game Cycle vs Pausing Clock

There are multiple ways to pause the game cycle, such as using menus, showing the inventory screen, restarting, saving or restoring a game, displaying messages with the print command, etc. However, in most of these cases, although the interpreter cycle is paused while the activity is resolved, the game's internal clock continues to move.

There are only three instances where AGI will pause the internal clock in addition to pausing the game cycle:

    • pause command
    • save command
    • restore command

If your game relies on timing elements specifically tied to the internal clock, you need to consider these factors to avoid unexpected results.
