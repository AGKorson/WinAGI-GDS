v20: Machine Type

When AGI loads, it does a system check to determine which computer type is running. Variable v20 is then set to match, according to the following table:

Value Computer Type
0 IBM PC compatibles
1 IBM PCJr
2 Tandy 1000
3 Apple IIe, IIc
4 Atari ST
5 Amiga
6 Macintosh
7 Cortland*
8* PS/2

Values for non-MSDOS platforms are based on original Sierra source code, with the exception of the PS/2, which was taken from an early version of fan-based AGI specifications (and has not been independently verified).

The machine type 7 was called 'Cortland' in original Sierra source code, but was marked with a question mark. Cortland was a pre-release development name of the system later named the Apple IIgs.

There are some very old references to a value of 20 as 'Amiga (OLD)', including a note in SCUMMVM calling it a rarely used old value for Amiga. This has recently been determined to be inaccurate - all known Amiga releases use a value of 5.

The computer type variable was often used in original Sierra games to provide customized features unique to a specific computer. AGI maintains an internal copy of computer type value that it uses. So as long as your code does not need to know the computer type, using v20 for another purpose does not cause problems in AGI. Just be aware that every time the restore.game command is called, AGI resets the value of v20 to the current computer type.
