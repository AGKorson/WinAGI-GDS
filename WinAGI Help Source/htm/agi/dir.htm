Directory Files

In order for AGI to know where in the VOL files individual resources are stored, AGI games include directory files that contain pointers that specify the VOL file and location of each resource.

Version 2 Directory Files

In version 2 games, each resource type has its own directory, logdir for logics, picdir for pictures, snddir for sounds, and viewdir for views.

Each directory file contains a series of three-byte entries for each resource. The size of the directory file can vary, but never be greater than 768 bytes (256 entries times three bytes per entry). Each 3-byte entry is of the following format:

 

    Byte 1           Byte 2           Byte 3
7 6 5 4 3 2 1 0  7 6 5 4 3 2 1 0  7 6 5 4 3 2 1 0
V V V V P P P P  P P P P P P P P  P P P P P P P P

 

where V = VOL number and P = position (offset into VOL file).

The index of the entries corresponds to the number of a given resource. For example, if the following three byte entry is entry number 45 in the snddir directory file,

12 3D FE

 

then the sound resource with index number 45 is located at position 0x23DFE in the VOL.1 file. In version 2, if the upper nibble of the first byte contains the value 0xF (i.e value is >= 0xF0), then AGI assumes the resource does not exist. Although only the upper nibble of the first byte is checked, all version 2 Sierra games store 0xFF in all three bytes to indicate a non-existent resource.

Although the four-bit number field used in DIR files to identify the VOL file number implied version 2 games could have up to 16 VOL files, AGI was actually limited to 5 VOL files; see the VOL file limit discussion for more information.

Version 3 Directory Files

Version 3 games used a single, combined directory file, instead of four separate files. The directory filename, like version 3 VOL files, start with the game's ID followed by 'DIR', for example, 'GRDIR'.

This single file is basically the four version 2 files joined together, with an 8 byte header giving the position of each directory within the single file. There are two bytes that contain the offset in the directory file for the beginning section of each resource type in the directory file:

Byte 0 1 2 3 4 5 6 7
     L L P P V V S S

 

where LL = offset of logdir, PP = offset of picdir, VV = offset of viewdir and SS = offset of snddir.

Each offset is two bytes in length where the first byte is the low byte and the second byte is the high byte as is the case in the whole AGI system. For example, the first two bytes will always be 0x0800 since logdir immediately follows the header, which is a fixed size of eight bytes.

The format of each of the individual directory sections then follows as above for version 2 games. In version 3, AGI checks that all three bytes are 0xFF to identify invalid entries, instead of just checking the upper nibble of the first byte.

If the combined single directory file is not found when a version 3 interpreter loads a game, the interpreter will look for version 2 style directory files. There are no known version 3 games that used the separate directories, though.
