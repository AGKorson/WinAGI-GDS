f4: Said Found Match

When the said command finds a match (i.e. the player's input matches the words passed as arguments to said), flag f4 is set to TRUE. This makes it easier for AGI to handle subsequent calls to said, because said automatically returns a value of FALSE if flag f4 is TRUE.

At the beginning of each interpreter cycle AGI resets f4 to FALSE.

Flag f2 (input received) and flag f4 are used together by AGI to manage evaluation of player input. When f2 is TRUE and f4 is FALSE, the said command will check player input for the words passed as arguments. If either f2 is FALSE (meaning there is nothing to parse), or flag f4 is TRUE (meaning said already has found a match) said will not check for input and just return false.

If you are writing advanced code, you may need to force the values of f2 and f4 to achieve certain effects. For example, after a call to the said command returns TRUE, you can reset f4 to FALSE so that future said commands in the same cycle will look for a match. See the said for more information and examples.

Attempting to use this flag for other purposes is not recommended, because AGI overwrites it in every interpreter cycle.
