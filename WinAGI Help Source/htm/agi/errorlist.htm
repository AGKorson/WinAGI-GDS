List of Trappable Errors

Error #1: view not loaded in discard.view command

v18 value: the invalid view number that was detected

This error is raised if you attempt to discard a view using the discard.view (or discard.view.v) command when the view is not loaded. This usually means the programmer entered the wrong view number by mistake. It could also mean the commands are out of order, or perhaps the discard.view command was called twice on the same view.

This error is also raised if an unloaded view is discarded during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

 

Error #2: invalid screen object number in set.view command

v18 value: the invalid screen object number that was detected

In version 2.089 only, AGI checks to see if the screen object number passed to the internal setview function (which is called by the set.view, set.view.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands) is less than or equal to the MaxScreenObj value. If it is not, i.e. the object number is invalid, AGI raises this error.

This error is also raised if an invalid screen object number is encountered while setting a view during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

NOTE: This error is only trapped in AGI version 2.089.

 

Error #3: view not loaded in set.view command

v18 value: the invalid view number that was detected

When assigning a view to a screen object in the internal setview function (which is called by the set.view, set.view.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands), AGI raises this error if the view is not loaded. You must load the view before using it.

This error is also raised if an unloaded view is encountered while setting a view during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

 

Error #4: invalid screen object number in set.loop command

v18 value: the invalid screen object number that was detected

In version 2.089 only, AGI checks to see if the screen object number passed to the internal setloop function (which is called by the set.loop, set.loop.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands) is less than or equal to the MaxScreenObj value. If it is not, i.e. the object number is invalid, AGI raises this error.

This error is also raised if an invalid screen object number is encountered while setting a loop during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

NOTE: This error is only trapped in AGI version 2.089.

 

Error #5: invalid loop number in set.loop command

v18 value: the screen object that the loop was being assigned to

When assigning a loop to a screen object in the internal setloop function (which is called by the set.loop, set.loop.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands), AGI raises this error if the loop is not valid (i.e. the assigned loop number exceeds the assigned view's maximum number of loops).

NOTE: There is a bug in AGI in the code that checks for this error. Loop numbers start at zero, and valid value run from 0 to the loop count-1. So loop values equal to the loop count should return this error. But instead, AGI only raises this error if the loop number is GREATER than the loop count. This means that if you pass a loop value that is exactly equal to the loop count, AGI thinks it is a valid loop number, and will attempt to load invalid loop data, which will almost always crash AGI.

This error is also raised if an invalid loop number is encountered while assigning a loop during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

 

Error #6: view not assigned in set.loop command

v18 value: the screen object that the view was being assigned to

When assigning a loop to a screen object in the internal setloop function (which is called by the set.loop, set.loop.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands), AGI raises this error if there is not a valid view already assigned to the screen object. You must first assign a view before attempting to set a loop.

This error is also raised if an unassigned view is encountered while assigning a loop during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

 

Error #7: invalid screen object number in set.cel command

v18 value: the invalid screen object number that was detected

In version 2.089 only, AGI checks to see if the screen object number passed to the internal setcel function (which is called by the set.cel, set.cel.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands) is less than or equal to the MaxScreenObj value. If it is not, i.e. the object number is invalid, AGI raises this error.

This error is also raised if an invalid screen object number is encountered while setting a cel during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

NOTE: This error is only trapped in AGI version 2.089.

 

Error #8: invalid cel number in set.cel command

v18 value: the screen object that the cel was being assigned to

When assigning a cel to a screen object in the internal setcel function (which is called by the set.cel, set.cel.v, set.loop, set.loop.v, set.view, set.view.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands), AGI raises this error if the cel is not valid (i.e. the assigned cel number exceeds the assigned view's maximum number of cels for the assigned loop).

This error is also raised if an invalid cel number is encountered while assigning a cel during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

 

Error #9: sound not loaded in sound command

v18 value: the invalid sound number that was detected

If the sound referenced in a sound command has not been loaded, AGI raises this error. You must first load a sound before you can play it.

 

Error #10: view not assigned in set.cel command

v18 value: the screen object that the view was being assigned to

When assigning a cel to a screen object in the internal setcel function (which is called by the set.cel, set.cel.v, set.loop, set.loop.v, set.view, set.view.v, show.obj, show.obj.v, add.to.pic and add.to.pic.v commands), AGI raises this error if there is not a valid view already assigned to the screen object. You must first assign a view before attempting to set a loop.

This error is also raised if an unassigned view is encountered while assigning a loop during restoration of a saved game. This is unlikely though, unless the save game file is corrupted.

 

Error #11: script buffer overflow

v18 value: the current script buffer size

In order to properly restore a game from a save game file, AGI tracks the loading and unloading of resources in a 'script' buffer. If the number of script entries exceeds the size of the script buffer, this error is raised. See the AGI Scripts topic for additional information on managing scripts.

 

Error #12: invalid screen object number in erase command

v18 value: the invalid screen object number that was detected

AGI checks to see if the screen object number passed to the erase command is less than or equal to the MaxScreenObj value. If it is not, i.e. the object number is invalid, AGI raises this error.

 

Error #13: invalid screen object number in animate.obj command

v18 value: the invalid screen object number that was detected

AGI checks to see if the screen object number passed to the animate.obj command is less than or equal to the MaxScreenObj value. If it is not, i.e. the object number is invalid, AGI raises this error.

 

Error #14: invalid message number

v18 value: the invalid message number that was detected

If AGI tries to access an undeclared message (such as when printing, setting menus and menu items, or other commands or functions that need to access messages) that is less than the logic message count, AGI raises this error. This error can easily be avoided by declaring 'null' messages for any message numbers that aren't going to be used. WinAGI automatically declares null messages for any messages that are not explicitly declared in a logic.

Note that if the undeclared message number is greater than the logic message count, no error is raised, and AGI will attempt to access an invalid memory location, resulting in unexpected data being displayed.

 

Error #15: invalid test command

v18 value: not used; set to zero

This error is raised if AGI encounters a test command value greater than 38 while processing a logic. The reason this value is 38, even though there are only 19 valid test commands (including the undocumented return.false command), is because of a bug; in the source code for AGI, Sierra calcuated the maximum number of valid test commands by dividing the size of the command jump table by 2, but there are actually four bytes per entry in that table. So using a test command byte value from 19 to 37 won't cause a trappable error, but instead will result in AGI using invalid memory to try and process the test, which will almost always crash AGI.

 

Error #16: invalid action command

v18 value: not used; set to zero

If AGI encounters an action command value that is greater than the number of valid AGI commands for that version, AGI raises this error. The number of action commands grew steadily with subsequent versions; version 2.089 had 155 valid AGI action commands, while the last version, 3.002.0149, had 181 action commands. Backwards compatibility occasionally becomes an issue if a game designed for a later version is run on an earlier version which doesn't have all the commands that later version had. This error check is likely to help programmers detect if a player is using the wrong interpreter version.

 

Error #17: not used

This error code was not used in AGI. Some early versions of AGI Specifications list this error code as occurring when the start.update command was called with an invalid screen object number, but no AGI versions (2.089 and above) use this error code.

 

Error #18: picture not loaded in draw.pic command

v18 value: the invalid picture number that was detected

If the value passed to the draw.pic command is for a picture that hasn't been previously loaded, AGI raises this error. Make sure a picture is loaded before trying to draw it to the screen.

This error is also raised if the overlay.pic command is called on a picture that is not loaded.

 

Error #19: invalid screen object number in draw command

v18 value: the invalid screen object number that was detected

AGI checks to see if the screen object number passed to the draw command is less than or equal to the MaxScreenObj value. If it is not, i.e. the object number is invalid, AGI raises this error.

 

Error #20: invalid cel data in draw command

v18 value: the screen object number that was passed to the draw command

AGI checks that a screen object has a valid pointer to cel data before drawing it to the screen with the draw command. If the cel data are invalid (usually due to not assigning a view) this error is raised.

 

Error #21: invalid picture number in discard.pic command

v18 value: the invalid picture number that was detected

If the value passed to the discard.pic command is for a picture that hasn't been previously loaded, AGI raises this error. Don't try to discard a picture that hasn't been loaded.

 

Error #22: not used

This error code was not used in AGI. Some early versions of AGI Specifications list this error code as occurring during certain situations involving the set.scan.start command, but no AGI versions (2.089 and above) use this error code.

 

Error #23 (0x17): invalid inventory item number

v18 value: the invalid inventory item number that was detected

When using commands that access inventory items such as get, put and drop, AGI checks the item number, and if it exceeds the number of items in the OBJECT file, this error is raised.
