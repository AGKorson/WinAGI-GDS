AGI Text Display Mode

The text display mode is used to display text on the screen when graphical data are not required. A help screen is a common use for the text mode. The inventory item status screen also uses the text mode. To switch to the text mode, use the text.screen command. Every time the text screen is displayed, it is cleared, using the current background color value. The default background color when AGI starts is black. Regardless of the graphics card, the resolution in text mode is 40 columns by 25 rows.

Hercules Graphics Card (HGC)

On the HGC display, each character is 16 pixels wide by 14 pixels high. If the foreground color is less than or equal to the background color, the text will display as black text on white foreground. Otherwise the text will display as white text on a black foreground.

Color Graphics Adapter (CGA)

On the CGA display, the characters are the same size as in the graphics mode (8 pixels wide by 8 pixels high). Unlike the graphics mode, all sixteen colors are available for foreground and background color, just like on the EGA display.

Enhanced Graphics Adapter (EGA)

On the EGA display, each character is 8 pixels wide by 16 pixels high in text mode. Although the characters have more available pixels than on the graphics screen, they are still rather blocky text characters. Both the foreground and background can be set to any of the sixteen colors available in AGI using the set.text.attribute command.

Video Graphics Adapter (VGA)

When used on the IBM PC, the VGA display used the EGA compatibility mode. Its graphics and text characteristics were identical to the EGA card.

Text Mode Properties

The text display mode uses separate memory space from the graphics display. This means that when in text mode, you can still run commands that affect the graphics display, such as adding or moving objects, changing views, etc. even though the graphics display is not visible. Once you are finished with the text mode, the graphics command will switch back to the graphics display, and resume the game, continuing with the the graphic display's current state. While the text mode is active, the normal interpreter cycle continues, but no screen objects are updated.

Only commands that are specifically designed for use with the text screen should be used while displaying the text screen. While it is possible, the text mode should not be used to show menus, print messages and receive commands from the player. Because the text mode does not automatically clear the screen after text is displayed, these features are difficult to manage on the text screen.

The text mode is also used when the status command is used to display inventory items. After dismissing the status screen AGI always goes back to the graphics screen, regardless of which mode was active when the command was called.

Fonts and Characters

AGI used the default font included in MSDOS to display text characters on screen. See the Font and Character Codes in AGI topic for details on how AGI managed drawing characters on screen.
