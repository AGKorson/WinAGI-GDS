v6: Ego Direction

AGI uses v6 as a shortcut to access ego's current direction of motion. Although using the get.dir command will return ego's direction, a programmer can instead simply check the value of v6 to get ego's direction.

Also, unlike other objects, ego's direction can't be set by using the set.dir command, not even when in program control mode. When in program control mode, ego's direction can only be changed by changing the value of v6.

Because of the way the interpreter cycle handles the ego object direction status, using set.dir to change ego's direction does not work (regardless of control mode). The set.dir command modifies ego's object status in the screen object table. Then after all logics are processed, the value in v6 is copied back to the screen object table, overwriting the value set by the set.dir command.

This also means that using v6 to change ego's direction will always override player's keyboard input (in player control mode) and any move or wander command (in program control mode).

Do not assign a value greater than 8 to v6. The algorithm that AGI uses to convert the direction value into x and y offsets will behave unpredictably, and the auto-loop feature will try to set a non-existent loop; either of these conditions will likely crash AGI. See the set.dir command for more details.

Do not attempt to use this variable for any other purpose.
