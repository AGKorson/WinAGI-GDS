v16: Ego View Number

every time a new.room or new.room.v command is executed, AGI stores the view number for the view currently assigned to ego in variable v16. Presumably, this is to make it easy for the game programmer to reload and set the same view to ego using the load.view.v and set.view.v commands. This is necessary because changing rooms unloads all game resources that are not in the room.0 portion of the memory stack, and clears all screen objects including ego.

AGI does not use this variable in any other way; it is just an aid to assist game programmers. It can be used by a game programmer as another variable for any game purposes as long as the programmer understands that it gets reset to ego's view number with each new.room command.

Attempting to use this variable for other purposes is not recommended, because AGI overwrites it in every new room.
