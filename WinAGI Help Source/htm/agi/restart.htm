Restarting

The restart.game command is used to restart a game from the beginning. Restarting a game is similar to restoring a game, except all memory is reset to its default state, all objects are unloaded, and all resources are removed from the memory heap.

If a sound is playing when restart.game command is called, AGI stops it before doing anything else.

In versions 2.425 and above, if reserved flag f16 (auto restart) is TRUE, the game will automatically restart. If f16 is FALSE, a message box is displayed asking the player to confirm the restart. In version 2.089, 2.272 and 2.411 flag f16 had no effect and the player was always asked to confirm the restart.

Unlike the save and restore functions, AGI does not pause the internal clock while the restart dialog is shown.

The restart process performs the following steps:

    • clears the input line
    • caches the value of reserved flag f9 (sound on)
    • clears the heap (which unloads all resources)
    • reloads the OBJECT file
    • resets the screen object table
    • resets all variables to 0 and all flags to FALSE
    • clears the controller evaluation table
    • clears all resource linked lists
    • reserved variables v20 (computer type), v26 (monitor type), v22 (sound channel count) and reserved flag f11 (noise channel available) are restored to match current hardware capabilities (in versions that use those reserved variables)
    • reserved variable v24 (max input length) is reset to 41 (in all versions except 2.089)
    • reserved flags f5 (new room) and f6 (game restarted) are set to TRUE
    • reserved flag f9 (sound on) is restored from the value cached earlier
    • if a trace logic was loaded before the restore, it gets reloaded
    • re-enables all menu items
    • redraws the cursor
    • begins a new interpreter cycle

There are a number of things that the restart command does NOT do that are important to be aware of:

    • restart does not clear the graphics display. What ever is displayed when the command is called will remain there until a new picture is loaded and displayed. This is rarely a concern, as a new room and picture is usually loaded immediately at the start of the first cycle after the restart occurs. Also, the state of the status line (on or off) is not affected by a restart.
    • the string table is not cleared; all strings retain their values from before the restart
    • trace window configuration is not affected
    • the screen configuration is not affected
