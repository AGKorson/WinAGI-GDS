v24: Input Line Maximum Length

AGI uses this this variable to set the maximum length of input line. every time a key-press is read from the input queue, AGI checks the value of this variable to determine if there is room for the character on the input line. The input line is limited to the lower value of v24 or 40 minus the length of the input prompt. (This is to make sure the input never goes past the end of the row.) All key-presses that would exceed this limit are discarded by AGI.

When AGI starts, v24 is set to 41 as a default.

There is a minor bug in AGI that will cause the cursor on the input line to jump up one row in certain situations if v24 is set to a value long enough for the cursor to reach the right edge (column 39). To prevent this, you should set v24 so that the cursor character never reaches the last column of your input line. See the Input Line discussion in the Keyboard Input topic for additional details.

This variable was not reserved by AGI and had no effect in version 2.089.

Attempting to use this variable for other purposes is not recommended, unless your game does not use text input.
