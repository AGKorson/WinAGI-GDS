AGI Debugging

When testing their games, Sierra included debug code that took advantage of AGI's debugging capabilities. Most AGI games were shipped with the debug code intact, but with the activation code removed. Some game versions were released which allowed the player to enter the debug mode.

Most of the games activate their debug modes by pushing the ALT-D key combination. Some games used a secret phrase to enter the debug mode.

    • Gold Rush: type 'bird man'
    • King's Quest 3: type 'rat shit' (in some versions ALT-D works instead)
    • Police Quest: type 'stink bug'
    • Space Quest 2: type 'backstage' or 'dbg' (type 'tester' to bring up Roger's screen coordinates)

 

Debug features of the Interpreter

The AGI interpreter contains several features and debugging commands that allow a game developers to monitor and troubleshoot their game during development. These include:

    • logging feature with the log command, which allows the programmer to capture information on the game that can be analyzed later to make corrections and improvements to the game;
    • ability to display game information such as current interpreter version and memory usage, useful for monitoring and making adjustments to avoid memory problems;
    • ability to display screen object information, which can be useful in troubleshooting interaction of objects on screen;
    • an option to display the priority screen in order to monitor and improve the '3-D' appearance of the game; and
    • a command tracer that allows the programmer to step through each line of code in a logic which can be helpful in tracking down bugs.

Standard Debug Mode Words

Most of Sierra's AGI games contained a debug logic that contained special code used for debugging. This special debug logic was fairly standard among the games, and contained the following words that could be typed by the player while debugging:

    • TP: Teleport to another room (by using get.number, then calling new.room.v)
    • SHOW PRIORITY: Show the current room's priority bands and control lines (by calling show.pri.screen command)
    • GET OBJECT: Get any inventory item (by using get.number, and then calling get.v)
    • GIMME GIMME: Get all inventory items (by running a short loop that calls get.v for all inventory items in the game)
    • SHOW FLAG: Display the value of one of the 256 flags (by using get.number, and then using simple if statement to print the flag's value)
    • SET FLAG: Set one of the 256 flags to TRUE (by using get.number, and then using set.v)
    • RESET FLAG: Reset one of the 256 flags (by using get.number, and then using reset.v)
    • SHOW VAR: Display the value of one of the 256 variables (by using get.number, and then printing the variable's value using the %v<number> message format code)
    • SET VAR: Change the value of one of the 256 variables (by using get.number, and then lindirect.v)
    • POS: Change ego's position (by using get.number, and then reposition.to)
    • COORDINATES: Display ego's current screen coordinates (by using get.posn)
    • OBJECT: Display screen object information (by using get.number, and then object.status.v)
    • OBJECT ROOM: Display which room an object is found in (by using get.number, and then get.room.v)
    • LOG: Write text to the log file (using the log command)
    • SET PRIORITY: Set the priority of ego by using get.number, and then set.priority.v)
    • RELEASE PRIORITY: Restore ego priority to automatic, based on position

WinAGI includes a debug logic in its template game that includes these basic debug functions.
