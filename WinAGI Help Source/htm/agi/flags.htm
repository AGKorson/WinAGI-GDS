Flags

The flag data type is a single bit boolean number; flag values can be either 1 (TRUE) or 0 (FALSE). There are 256 flags in memory numbered from 0 to 255. As with variables, flags are identified by their index in the flag table, and are represented in source code by the letter 'f' and their index number (i.e. f0, f1, etc.)

Flags are primarily used to indicate the status of events. For example, a flag can be used to indicate if a door in the game is open or closed, or if sound is enabled or disabled.

Flags are also used in several commands to indicate completion of an action, or attainment of a goal, such as a screen object reaching a desired position.

Reserved Flags

The first 17 flags (f0 - f16) and f20 (in version 3.002.102 and higher) are used internally by AGI, and should not be used for storing game data. Doing so may cause unexpected behavior, including potentially fatal errors. In some cases, a game programmer may need to directly manipulate these reserved flags to achieve specific game effects, but great care should be taken to avoid creating unexpected results or errors.

f1: Ego is Obscured

f2: Input Received

f3: Ego on Signal Line

f4: Said Found Match

f5: New Room

f6: Game Restarted

f8: Joystick Double-Click Enabled

f9: Sound On

f10: Command Tracer Enabled

f11: Noise Channel Available

f12: Game Restored

f13: Inventory Selection Enabled

f14: Menu Enabled

f15: Message Windows Don't Pause Game

f16: Auto-restart

It is not known why flags f17-f19 were skipped when the 'forced auto-loop' reserved flag was added. Flags f17-f19 do not have any special meaning in any interpreter version.
