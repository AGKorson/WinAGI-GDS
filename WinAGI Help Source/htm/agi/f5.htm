f5: New Room First Cycle

When the new.room command is called, AGI sets flag f5 to TRUE. This allows programmers to handle all the initialization steps that need to be done for the new room. This is why the code for most new rooms begins by checking if flag f5 is TRUE; if it is, then all the room initialization code is run; if not, then the 'regular' room code that gets run in each cycle is run. At the end of each interpreter cycle AGI resets f5 to FALSE.

This flag is also set to TRUE when AGI first loads, and when the restart.game command is called.

Attempting to use this flag for other purposes is not recommended, because AGI overwrites it in every interpreter cycle.
