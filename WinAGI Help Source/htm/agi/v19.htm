v19: Pressed Key Code

When polling the keyboard at the beginning of each interpreter cycle, AGI first resets v19 to zero. Then, if an ASCII value key-press is read from the input queue, the ASCII value is placed in variable v19. v19 is also set when the have.key command checks the keyboard buffer.

The have.key command is often used to create loops that wait for a keypress. Just testing the value of v19 is not sufficient, because it only gets updated at the beginning of each cycle, or when have.key is called:

[ this loop won't work
wait:
if(v19 != 0) [ will never be true, even if a key is pressed
  {
  goto(wait);
  }
...
[ this is correct way to check for a keypress
v19 = 0; [ clear the key value first
wait2:
if(!have.key())
  {
  goto(wait2);
  }
...

Only ASCII values are stored in v19. Keypresses that don't generate an ASCII code (function keys, shift, alt or control, etc.) are not captured in v19. Any ASCII keys that are assigned to controllers will also not be captured in v19.

Attempting to use this variable for other purposes is not recommended; it gets updated with each key-press; it gets overwritten every time the player pressed a key on the keyboard.
