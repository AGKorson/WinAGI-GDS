v2: Ego Border Edge Code

In every interpreter cycle, AGI sets the value of v2 based on the current location of ego. As long as ego is not touching an edge, v2 is set to a value of 0. When ego touches (hits) one of the edges, v2 is set to indicate which edge was touched:

    • 0 = Touched nothing
    • 1 = Horizon (or top edge of screen if ego is ignoring horizon)
    • 2 = Right edge of screen
    • 3 = Bottom edge of screen
    • 4 = Left edge of screen

This primary use for this variable is to determine when to change rooms as ego moves about in the game. If the value of v2 is non-zero, logic code can use the value of v2 to change to the appropriate room using the new.room command.

Attempting to use this variable for other purposes is not recommended, because AGI overwrites it in each interpreter cycle.
