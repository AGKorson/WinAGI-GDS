AGI Game Resources

Resources are the game-specific information that makes up an AGI game. AGI loads and unloads resources as necessary during game play to make the game work. The resources that are used by AGI include:

Logics

Logics contain the commands that control the overall game play.

Pictures

Scenery and background images are known as picture resources in AGI.

Sounds

Sound resources are used for background music and sound effects.

Views

Animated graphics (sprites) that represent moving people, animals, and other objects are called views in AGI.

WORDS.TOK File

This resource contains a list of words that a player can use to interact with the game via the keyboard.

OBJECT File

The OBJECT file contains a list of text strings that identify all the things that a player can pick up (add to inventory) in a game.

Volume and Directory Files

With the exception of the words list (WORDS.TOK) and the inventory items list (OBJECT), all AGI resources are kept in storage files known as 'VOL' files. Separate directory files provide the location information so AGI can extract resources as needed.

Resource Headers

To facilitate loading, each resource in the VOL file has a header which includes a signature to verify a valid resource and size information.

In version 2 games, the resource header was five bytes:

Byte Meaning
0 - 1 File signature (0x12, 0x34)
2 VOL file number that contains this resource
3 - 4 Length of resource (not including the 5 byte header) (LO-HI)

The signature and VOL number were used to validate the resource; if neither matched, AGI assumed the wrong disk was loaded, and would prompt the user to insert the correct disk.

Version 3 games introduced compression, so the header for resources was expanded to seven bytes:

Byte Meaning
0 - 1 File signature (0x12, 0x34)
2 VOL file that contains this resource (bit 7 set if resource is a compressed picture)
3 - 4 Uncompressed size of resource (LO-HI)
5 - 6 Compressed size of resource (not including the 7 byte header) (LO-HI)

The header contains the size of the resource in the VOL file (compressed size) as well as the size of the resource once uncompressed. Most of the resources in version 3 games were compressed, but some of them were not. AGI determines whether the resource is compressed by comparing the values of the two sizes given in the header information. If they are equal, then it knows that the resource is stored uncompressed. If the sizes do not match (or if bit 7 of the volume byte is set, indicating a picture is compressed), then AGI knows the resource is compressed, and uncompresses it.

As described in the Compression topic, AGI uses a form of LZW compression for logics, sounds, and views, and a form of run length encoding for pictures. Bit 7 of header byte 2 is used by AGI to determine which form of compression is used for a given resource. If the bit is set, then the compressed resource is a picture resource. If it is not, either the resource is not compressed, or it is a compressed logic, sound or view.

Although there are no known examples from Sierra games, there is nothing preventing a picture resource from being compressed using the same LZW compression as the other resources; if bit 7 of header byte two is not set and the file sizes are different, AGI would decompress it using the LZW process.
