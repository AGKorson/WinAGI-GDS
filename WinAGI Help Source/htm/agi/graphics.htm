Graphics and Text Displays in AGI

AGI used two display modes; a graphical mode that displays pictures and screen object animations, and a text only mode.

Graphics Adapters Supported by AGI

In the 1980s, when AGI was created, displays were very limited in their capabilities and resolution compared with modern displays. When AGI was first released, there were three common display adapters for the IBM PC: the Hercules Graphics Card (HGC), the Color Graphics Adapter (CGA) and the Enhanced Graphics Adapter (EGA). The Video Graphics Adapter (VGA) became available several years later, near the end of AGI's run.

Display details included below focus on the IBM PC and its allowable graphics cards. Information for other platforms is not covered in detail.

The overlay files that came with AGI handled the task of converting AGI graphical output data into display specific format. AGI would load the appropriate overlay files based on the detected (or specified) graphics controller.

Hercules Graphics Card (HGC)

The HGC was a monochrome graphics card (supporting green, amber, or, less often, black and white monitors). It included both a text mode (80 x 25) and a graphics mode (720 x 348). In order to display a text screen of 40 x 25, AGI ran HGC in graphics mode and used custom code to display text, requiring a separate file that contained the font glyph data. Also,there was a significant loss of detail when playing games on an HGC display since AGI used 16 different colors, which HGC had to simulate by dithering pixels.

Color Graphics Adapter (CGA)

The CGA graphics controller was IBM's own graphics controller that competed with the HGC. While it offered up to sixteen colors (in most modes, it could only display four of the sixteen colors at a time), it actually had less resolution (640 x 200 graphics) and thus its text modes were more blocky.

AGI used the 320 x 200 graphics mode which could only display four colors. The graphics mode had two built in palettes: black-cyan-magenta-white (CGA palette) and blue-green-red-brown (RGB palette). AGI used the 40 x 25 text mode when displaying the text screen.

Enhanced Graphics Adapter (EGA)

The most common display adapter (and the one that took advantage of all features in AGI) was the EGA. The EGA graphics card was capable of displaying 16 colors simultaneously (out of a palette of 64), and had resolutions up to 640 x 350. AGI used the 320 x 200 display resolution with 16 colors for graphics and the 40 x 25 text mode on EGA displays.

Video Graphics Adapter (VGA)

AGI could also run on the later VGA displays. Although VGA offered better resolution and color palette choices, when run on a PC, AGI used the VGA card's EGA compatible mode and ran with the same resolution and colors as EGA when displayed on a VGA system.

When run on a PS/2 with a VGA card, AGI used the VGA card's higher resolution for text characters, with each character drawn to an 8 x 16 pixel grid, instead of 8 x 8 as with EGA. Text was still limited to 25 rows of 40 columns.

AGI Color Palette

In all display modes, AGI used sixteen colors internally. Resources reference the colors by index. AGI would determine what the actual color to be displayed would be depending on whatever display mode was active. For EGA displays, sixteen distinct colors were displayed. On CGA displays, AGI used dithering of the four available colors in the active palette to create sixteen shades. The HGC display, being monochrome, used texturing to simulate sixteen colors.
