Inventory Items

The inventory item data type is an unsigned eight bit number stored in the OBJECT file. Inventory items usually represent items that a player may pick up and carry during the game. Most inventory items have a corresponding view that is used to draw the item on the game screen. A close up view of the item is usually included as well, so that when a player 'looks' at the item, they can see a detailed image of it, including a short description. The show.obj command is used to display a close up view of an inventory item.

Each inventory item is identified by a number, and includes a text field with the inventory item's name/description, and a one-byte numerical value used to denote the 'location' or 'room' of the item.

Room Number

The location byte associated with an item can be used to track the room in which an item is located. You can use the inventory item commands, such as get, drop, put to manipulate the room value during game play. AGI does not automatically do anything with the assigned room number; i.e., inventory items are not displayed when the current room changes to match the room assigned to an inventory item.

The value 255 has special meaning for the room number. Inventory items with a room number of 255 are considered to be in the player's 'inventory', i.e. ego is 'carrying' them. The has test command returns TRUE if an item's room number equals 255.

Inventory Status

AGI includes a built in feature that will list all inventory items currently in ego's possession (i.e. have a room number of 255). The status command will switch AGI to text mode and display "You are carrying" and then list all items being carried.

If reserved flag f13 (inventory selection enabled) is set to TRUE, the inventory screen enables the player to select an item. The player uses the cursor keys to highlight an item. Pressing ENTER chooses the item, and closes the screen. Pressing ESC cancels the selection, and closes the screen.

If the player selects an item, the item number will be stored in reserved variable v25 (selected inventory item). If the player cancels by pressing ESC, AGI stores a value of 255 in v25.

When displaying the item text, AGI does not format it, so you can't use format codes. However, if you use the '%o#' format code in a print or display command, format codes embedded in the item text will work. For example, if item1 is '%v100 gold pieces', v100 = 50, and item1 is in player inventory, the status command will show:

%v100 gold pieces

but using v99 = 1; print("%o99"); will show:

50 gold pieces

There is a practical limit to 44 items that can be in ego's inventory. If more there are more than 44, the status command will try to display items to invalid cursor positions on the inventory screen, which will often result in crashing AGI.

Item 0

The first item (index 0) in most Sierra games is a null character (a single question mark'?'), and is treated as a null item. Other item indexes that aren't used by the game are also set to this null character. Assigning the first item to null character is not required, but most fan-based games continue the tradition of having item 0 be a null item.

One reason to avoid using zero as a valid item number is that AGI uses a value of zero for reserved variable v25 (selected inventory item) if there are no items in the player's inventory and the player presses ENTER on the status screen.

Similarly, you should avoid using item number 255. Pressing ESC on the status screen (canceling the selection) puts the value 255 in v25 (selected inventory item). It is not a big concern, though, because it is extremely unlikely a game would ever have that many items.
