set.cel

The set.cel command assigns a cel to a screen object.

 

Syntax

set.cel(obj oA, byt CEL);

Remarks

The current cel of screen object oA is set to CEL.

If a view has not been assigned to the object, AGI raises trappable error #10. If the value of CEL is not valid (exceeds last cel number available in the loop), AGI raises trappable error #8.

The number of the last cel in a loop can be determined using the last.cel command.

After the cel is assigned to the object, AGI validates that the cel will still fit on screen. See the Cycling Objects topic for details on the cel position check. If the position check results in the object being moved to avoid being off-edge, it will not update its position on the next interpreter cycle. This usually has no noticeable effect, but in some situations where precise positioning and motion are needed, it could cause unexpected behavior.

The set.cel command clears the internal 'no advance' bit field flag that is used by end.of.loop and reverse.loop. If set.cel is executed before end.of.loop or reverse.loop, the object skip its next cycle update, but if set.cel is executed AFTER end.of.loop or reverse.loop, the object will not skip a cycle, and will cycle immediately in the next update. This normally isn't noticeable in the game, but it could cause unexpected behavior is some situations. You should always call set.cel before end.of.loop or reverse.loop.

In version 2.089 only, AGI raises trappable error #7 if object number oA is invalid. In all later versions, AGI does not check that object number oA is a valid screen object. If it is not, this command will overwrite other data on the heap, which may cause unexpected results, including possibly crashing AGI.

 

Example

[ typical code to add a static view
animate.obj(o1);
load.view(8);
set.view(o1, 8);
set.loop(o1, 1);
set.cel(o1, 2);
position(o1, 20, 132);
stop.cycling(o1);
draw(o1);

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

47 (0x2F)
