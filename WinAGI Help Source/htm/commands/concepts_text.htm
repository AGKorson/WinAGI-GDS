Working with Text

AGI provides the capability to work with textual data on the graphics screen (where the background picture and objects are drawn), and also on a separate text only screen.

To switch to the text screen, use the text.screen command. The text screen uses a separate area of memory, so the graphics screen is not lost while in text mode. Use the graphics command to switch back to the graphics mode.

Text on the Text Screen

You shouldn't use the print command on the text screen; it only prints black text on a white background, with no border, and will not automatically clear the screen

The display and display.v commands will display text with the foreground and background colors as set by the set.text.attribute command. Both the foreground and background can be set to any of the sixteen colors available in AGI.

The clear.lines and clear.text.rect commands will always set the background to black; the color argument is ignored for the text screen. A sample workaround is provided in the clear.text.rect topic to clear the text screen so it matches the background color selected with set.text.attribute.

While it is possible, the text mode should not be used to show menus. The menu items are only visible when selected, no borders are drawn, and the menu does not clear itself from the screen when it is dismissed.

If enabled, the input line will work on the text screen. However, a minor glitch in AGI causes the input line to move to the top of the screen without the prompt or cursor when the text.screen command is called (instead of the position defined by the configure.screen command). A workaround is to use the prevent.input command just before switching to text mode, then calling the accept.input command immediately afterward. This forces the input line to its correct position.

Text on the Graphics Screen

Text can also be displayed on the graphics screen. The primary command for displaying text is the print command and its variants.

Text displayed via the print command includes a thin red line border. The foreground color is always black, and the background color is always white. The window will clear itself from the screen when it is dismissed.

There are three modes available for dismissing the print window:

  • If reserved flag f15 (no pause windows) is reset (FALSE) and reserved variable v21 (window close time) = 0, game play is halted and the window remains until the player presses the ENTER or ESC key. This the default behavior.
  • If reserved flag f15 (no pause windows) is reset (FALSE) and reserved variable v21 (window close time) > 0, game play is halted and the window remains open until v21 * 0.5 seconds elapse, or until the player presses the ENTER or ESC key, whichever comes first.
  • If reserved flag f15 (no pause windows) is set to TRUE game play continues while the message is displayed. The value of v21 (window close time) is ignored. Use the close.window command to dismiss the window.

Flag f15 is reset to FALSE and v21 is set to 0 after each print command executes. This means that every time you want a message to automatically disappear after a set time, you must set f15 and reassign the desired delay time.

The display and display.v commands work the same as on the text screen. However, the color choices are limited. The background color can only be black or white. If the background color is white, the foreground color is black, regardless of what was set by the set.text.attribute command. If the background color is black, then the text is displayed with the foreground color as set by the set.text.attribute command.

There is a simple hack you can use if you want to display black text on colored background on the graphics screen. Use the set.text.attribute command with a value of 0 for background color, and a foreground color >= to 128. Due to a bug in AGI, this will result in black text on a colored background, with the color being determined by the lower nibble (lower 4 bits) of the foreground value. Note that extended characters won't draw correctly in this scenario, and there is no way to get any other colored foreground to show; it can only be black.
