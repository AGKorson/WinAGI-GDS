unanimate.all

The unanimate.all command deactivates all screen objects.

 

Syntax

unanimate.all();

Remarks

The unanimate.all command marks all screen objects as 'uninitialized', and 'not drawn' by clearing the ANIMATED and DRAWN screen object bit field flags. All objects will no longer be updated, moved or cycled on screen. This means objects must be re-initialized before they can be used again.

All other screen object bit field flags and screen object data are unchanged. If you later re-animate an object, you must be sure to explicitly set all other object properties, otherwise AGI will use the previous values, which may result in unexpected results.

Although it would seem that unanimating screen objects would erase them from the screen, unanimate.all doesn't actually erase any screen objects. This appears to be a bug in AGI, because after AGI removes the background bitmaps for the screen objects from memory, it does not redraw the screen like it does in all other instances where the background bitmaps are deleted. A workaround is to use the show.pic command to force AGI to redraw the screen with the screen objects removed.

Example

#define disintegrate_all f72
if(isset(disintegrate_all))
  {
  unanimate.all();
  show.pic(); [ force a redraw to make them actually disappear
  }

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

34 (0x22)
