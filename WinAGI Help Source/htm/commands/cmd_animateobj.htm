animate.obj

The animate.obj command initializes a screen object so it can be used on screen.

 

Syntax

animate.obj(obj oA);

Remarks

Screen objects must be initialized with the animate.obj command before they can be used in any other commands. Commands that are run on an uninitialized screen objects will interact with invalid data. If you attempt to read data from an uninitialized screen object (by using commands such as get.posn), AGI will not raise an error; but it will return invalid data, which should not be relied on in your logic script. Using commands that set screen object parameters (such as set.dir) will also not raise any errors, but will essentially have no effect, as AGI ignores uninitialized screen objects as it goes through the interpreter cycle.

If animate.obj is executed on an object that has already been animated, the command does nothing.

When a screen object is initialized, all screen object bit field flags are cleared, except for UPDATE, CYCLE, and ANIMATED flags. This enables the screen object for updating and cycling. The screen object's movement mode and cycle mode are both set to 'normal', and its direction is set to 'STOPPED'.

All other screen object data are left unchanged. If you are re-using a screen object in the same room, this means you must be sure to set all other object properties manually after re-animating, otherwise AGI will use the previous values, which may result in unexpected results.

If the screen object number oA exceeds the max screen object count, AGI will raise trappable error #13.

If you call the animate.obj command on a screen object that is already initialized, the command is ignored.

 

Example

[ basic steps needed to create
[ and display a screen object
animate.obj(o2);
load.view(4);
set.view(o2,4);
position(o2,80,120);
draw(o2);

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

33 (0x21)
