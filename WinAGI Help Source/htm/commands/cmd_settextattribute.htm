set.text.attribute

The set.text.attribute command sets the foreground and background color for the display commands.

 

Syntax

set.text.attribute(byt FG, byt BG);

Remarks

This command sets the foreground and background colors used by AGI to display text with the display or display.v commands to FG and BG respectively. The input line also uses the foreground and background colors set by this command. The default foreground color is white, and the default background color is black for both the graphics screen and the text screen.

The set.text.attribute command combines the FG and BG into a single value, with the FG value in the lower nibble (lower four bits) and the BG in the upper nibble (higher four bits). When text is drawn on the screen, this single byte value is passed to the DOS function that draws the text on the screen.

When the text screen is active, the DOS function that draws the character on the screen uses the attribute values as expected; setting the text foreground to FG and the text background to BG. This is why all combinations of colors are possible on the text screen.

On the graphics screen, because of the way that DOS interprets the combined attribute value, it is impossible to have a text background color other than black or white. AGI deals with this by adjusting the colors passed to set.text.attribute as follows: when BG is black, the FG can be any other color and is not modified; but if BG is any value other than black, AGI changes the background color to white and sets the foreground color to black, ignoring the passed FG value completely. See the Text Color Options discussion in the Font and Character Codes in AGI topic for a detailed explanation.

In general, color settings are retained when switching between the graphics screen and the text screen, but there are a few nuances. First of all, when using the graphics screen, the background color that is kept by the system is either black or white as described above; the actual value passed is not kept by the system. If you switch to the text screen, the background value will also be black or white. You will need to rerun the set.text.attribute command to get any other background color. The foreground color is retained when switching to the text screen (even if it shows as black due to a white background on the graphics screen).

When switching from the text screen to the graphics screen, AGI will keep the previously set foreground color, but it always resets background to black.

On the Hercules Graphics Card (HGC) display, there are only two colors available (black and white). When displaying text (on either screen) if the set foreground value is less than or equal to the set background value, AGI draws black text on a white background, otherwise it draws white text on black background.

The text screen on a Color Graphics Adapter (CGA) display supports sixteen colors, so it will show all combinations of background and foreground color just like Enhanced Graphics Adapter (EGA) displays. On the graphics screen, CGA only has four colors in two available palettes (default CGA palette and alternate RGB palette). AGI uses a table to choose foreground text colors when the background is black. If background is not black, AGI uses black foreground on white background regardless of current background/foreground color settings, same as EGA displays):

Foreground Color CGA-0 (CGA) CGA-1 (RGB)
0 (black) black black
1 (blue) green cyan
2 (green) green cyan
3 (cyan) green cyan
4 (red) red magenta
5 (magenta) red magenta
6 (brown) red magenta
7 (light gray) brown white
8 (dark gray) brown white
9 (light blue) green cyan
10 (light green) green cyan
11 (light cyan) red magenta
12 (light red) red magenta
13 (light magenta) red magenta
14 (yellow) red magenta
15 (white) brown white

Using values for FG and BG greater than 15 will not cause AGI to crash, but may result in unexpected results. AGI uses only the lower nibble (four least significant bits) of each number to determine the actual colors to display. Internally, AGI combines the FG and BG values into a single 8 bit number that is used by the text drawing functions. Because of the way the numbers are combined, it is actually possible to draw black text on non-black backgrounds on the graphics screen. If FG is greater than 128 and BG is zero, AGI will use the higher nibble (upper four bits) of FG as the background color, and calls to display and display.v will draw black text on colored background. Using the text.screen command when this is the case will also clear the text screen to the 'hacked' foreground color.

 

Example

text.screen();
set.text.attribute(1, 14);
display(5, 5, "message text"); [ this will display blue text on a yellow background
pause();
graphics();
set.text.atrribute(1, 14);
display(5, 5, "message text"); [ this will display black text on white background
pause();
set.text.attribute(1, 0);
display(5, 5, "message text"); [ this will display blue text on black background

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

109 (0x6D)
