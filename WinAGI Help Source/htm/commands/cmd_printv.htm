print.v

The print.v command indirectly displays a message in a window.

 

Syntax

print.v(var vMSG);

Remarks

The text of the message referred to by variable vMSG is displayed in a bordered window.

Reserved flag f15 (no pause windows) and reserved variable v21 (window close time) determine how the print.v command affects game play:

    • f15 reset to FALSE and v21 == 0 (the default condition): displayed windows (including those shown by the print.v command) pause the game cycle, and remain until the player presses ENTER or ESC key. Although the game cycle is paused, the internal clock does not stop, and clock variables v11, v12, v13, v14 will continue to update.
    • f15 reset to FALSE and v21 > 0: the window will also pause the game cycle, and will be dismissed when the player presses ENTER or ESC key, or after v21 * 0.5 seconds, whichever comes first.
    • f15 is set to TRUE (v21 is ignored): the game cycle is NOT paused, and the window remains open indefinitely. The message window stays open and game play continues until a close.window command is called to dismiss the message, or another message is displayed.

After every time a window is displayed (including by using the print.v command), if f15 is TRUE, it is reset to FALSE and variable v21 is not modified. If f15 is FALSE, variable v21 is reset to 0.

(In version 2.089 only, reserved flag f15 is not used; message windows will never print without pausing.)

Messages can contain special format codes to enhance the output. See the Message Format Codes topic for details on available message codes.

 

Example

v50 = 5;
#message 5 "You fall into the water and drown!"
print(v50);  [ prints message 5

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

102 (0x66)
