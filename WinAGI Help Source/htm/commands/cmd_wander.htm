wander

The wander command sets a screen object's movement mode to 'wander' meaning it will randomly move around the screen.

 

Syntax

wander(obj oA);

Remarks

The movement mode of screen object oA is set to 'wander'. While wandering, AGI will randomly change the direction and distance the object travels. The initial distance is set to a random number >=6 and <=50. If the object encounters any obstacle (such as a horizon, screen edge, control line, block, etc.), it chooses a new random direction and checks the current distance counter value. If the counter is less than six, a new random distance counter (6 to 50) is chosen.

However, in what appears to bug, if the counter reaches zero without hitting an obstacle, the counter gets set to 255, and not a random value. The function then chooses a new random direction. The effect of this is that objects are more likely to go long distances until they hit an edge, and shorter distances are much rarer. This is usually not noticeable because most games tend to have relatively small areas for objects to move in and several obstacles that the object can interact with.

There is another bug in the wander function when the object is moving diagonally and hits the right or left edge. In this case, instead of considering it an 'obstacle' and changing a new random direction, the object will continue moving up or down along the edge; since the loop for diagonal movement is the left/right facing loop, the object appears to 'moonwalk'.

Setting the movement mode to 'wander' will automatically enable the object for updating if not currently enabled. The cycling property is not modified by the wander command.

If oA is ego, it automatically switches to program control mode.

If the set.dir command is executed while an object is wandering, it will move in the assigned direction until the random function chooses a new direction.

Do not use movement modes (move.obj, follow.ego or wander) and one-time cycling modes (end.of.loop and reverse.loop) on an object at the same time. The byte in the screen object data table that AGI uses to store the fDONE flag is also used by the cycling modes. If you use a movement mode command while a one-time cycle command is active, the flag number gets overwritten, and when the object finishes cycling, the wrong flag will be set.

AGI does not check that object number oA is a valid screen object. If it is not, this command will overwrite other data on the heap, which may cause unexpected results, including possibly crashing AGI.

Example

#define f68 drunk
if(isset(drunk))
  {
  wander(ego);  [ ego will wander aimlessly
  }

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

84 (0x54)
