Programming Menus and Keyboard Shortcuts

Menus and keyboard shortcut keys supplement the text input line for player interaction with the game. Menus and keyboard shortcuts are assigned to controllers that generate trappable events when the menu items are selected or the shortcut keys are pressed.

There are 50 controllers available (numbered from 0-49) to which menu items and shortcut keys can be assigned. To test whether the menu item or shortcut was selected, use the controller test command.

Setting Up Menus

Detailed information on how menus work in AGI can be found in the Menus topic.

When creating menus, keep the following points in mind:

    • If you create a menu in a logic other than logic 0, that logic (and any other resources loaded at the time) get stuck on the memory heap, wasting memory.
    • Only one menu can be created per game. Once you execute the submit.menu command, no changes, additions or deletions can be made. Subsequent calls to submit.menu will have no effect other than to reset the room.0 pointer, resulting in more wasted space on the memory heap.
    • The first menu item added to each menu determines the width of the menu borders. Make sure you pad the first item if necessary so the menu borders are wide enough for all items to display properly on the menu.
    • Multiple menu items and keyboard shortcuts can be assigned to the same controller. For example, a menu item called "Quit" and a keyboard shortcut CTRL+Q can both be assigned to controller c1. Testing for controller c1 will return TRUE if the menu item is selected, or if the shortcut is pressed.

Setting Up Keyboard Shortcuts

Keyboard shortcuts are assigned to controllers using the set.key command. When assigning keyboard combinations to controllers, keep the following in mind:

    • The ego control movement keys (arrow keys, home, end, page up, page down) have built-in functionality in AGI and cannot be assigned to controllers. If you use set.key on these keys, they will take up space in the controller assignment table but will never activate.
    • Assigning keys that have special functions in AGI, such as RETURN, SPACE, ESC, is possible, but also may result in unpredictable behavior.
    • Some keys on modern keyboards were not available on keyboards when AGI was created (e.g. F11, F12). Attempts to assign these keys to controllers usually will fail. Limit key assignments to keys available on the original equipment.
    • When assigned to a controller, the interpreter will not process the key for text input. The key press is replaced by the controller in the input queue. Also thehave.key command will not trap keys that are assigned to controllers.
    • Although there is room for 50 keyboard controller assignments, only 39 can actually be assigned (beginning with version 3.002.149 this was changed to 49). If you use the set.key command after 39 controllers have already been assigned, it will be ignored.

 

Key presses are captured by AGI as either a two-byte keyboard scan code or a two-byte ASCII code. If the key-press represents one of the original 7-bit ASCII characters, MSDOS passes the ASCII value; if it is not an ASCII character, the scan code is passed. AGI compares key-presses to assignments in the controller assignment table to determine if a controller has been activated. This means that key information passed by the set.key command must be in the same format as key-press information captured by MSDOS. If the key-press is an ASCII code, the first byte contains the ASCII code, and the second byte is zero. If the key-press is a keyboard scan code, the first byte is zero, and the second byte is the scan code.

Joystick Buttons

Joystick button presses can also be assigned using the set.key command. In this case, the first byte passed must always be a one (1), and the second byte identifies which button and whether it is a single click or double click of the button:

First Byte Second Byte  Button Press
1 1  Left button single click
1 2  Right button single click
1 3  Left button double click
1 4  Right button double click
