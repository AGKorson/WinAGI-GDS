set.pri.base

The set.pri.base command adjusts the size of the priority bands on the priority screen.

 

Syntax

set.pri.base(byt A);

Remarks

This command adjusts the size of the priority bands. The height of the base band (priority = 4) is set to A. All other bands from 5 to 14 are then spaced evenly over the remaining area of the screen.

The default height of the base band when AGI starts is 48 (pixel rows 0 - 47). This arrangement works for most rooms to simulate a 3-D screen, but if you have a room with a more complex perspectives, the set.pri.base command can be helpful.

Do not use a value greater than 158, otherwise some priority bands will be missing from the screen due to the way that AGI calculates the bands. If you use a value greater than 166, then the entire screen will be priority 4, regardless of objects' position.

This command initially included a bug in the code that determines the virtual Y value for fixed priority objects so the object would always draw on top of other objects with the same priority. The error only shows up when using a custom priority base with an object that has a fixed priority and another object without fixed priority is in the same priority band. The bug was fixed in version 3.002.098.

 

Example

set.pri.base(60); [ this widens the base band by 12 pixels

 

Required Interpreter Version

Available in version 2.936 and above.

 

Byte-Code Value

174 (0xAE)
