trace.info

The trace.info command sets up the trace mode.

 

Syntax

trace.info(byt LOGNUM, byt TOP, byt HEIGHT);

Remarks

The trace.info command is usually run before activating trace mode, and sets the position of the trace window using the TOP value, with a window height set by the HEIGHT value. TOP and HEIGHT are in units of text rows (0 - 24). It also identifies logic LOGNUM as the source for trace commands as messages.

If trace.info is not called before using the trace mode, the default location of the trace window is text row 1, and the default height is 15 rows. If the trace mode is enabled before a new.room command is executed, the trace window will display commands as numbers as if the trace.info command was never called.

Keep in mind that due to a bug in AGI, using any value for TOP except 1 will result in the trace window being drawn incorrectly; it does not affect the performance of the tracer function, it just looks bad.

To have the tracer display commands with text (instead of the command number), tracer logic LOGNUM should contain all AGI commands as messages. The number of the message corresponds to the action command with the same byte-code (i.e. message 1 should be "increment", message 2 should be "decrement", etc.) Test commands get an offset of either 220 (for versions 2.089 and 2.272) or 240 (for all other versions), i.e. message 241 should be "equaln", message 242 should be "equalv", etc.

The trace.info command does not load the trace logic LOGNUM. Before using the trace.on command, the trace logic must be loaded, or the trace window will display incorrect logic number information due to a bug in the trace function. While the trace mode is active and a valid logic number was passed to trace.info, AGI will automatically load the tracer logic after any new.room command is executed.

For a detailed discussion of how the trace mode works, see the AGI Command Tracer topic.

If logic LOGNUM does not exist, AGI will raise a runtime error and crash when it tries to load the logic.

Using values of TOP and HEIGHT that are greater than 24 will result in a garbled display.

 

Example

set(enableTrace); [ enable trace mode
if(said("trace"))
  {
  trace.info(TracerLogic, 1, 10);  [ assume TracerLogic is a valid logic ID
  trace.on();
  }

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

150 (0x96)
