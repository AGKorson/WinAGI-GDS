configure.screen

The configure.screen command sets the location of the play area (visual graphic screen), input line and status line.

 

Syntax

configure.screen(byt PLAYTOP, byt INPUTLINE, byt STATUSLINE);

Remarks

The units for the three arguments are in screen lines (0 - 24 range).

When AGI starts, the default values for PLAYTOP, INPUTLINE and STATUSLINE are 0, 21, 23 respectively. Most Sierra AGI game did not use these values.

PLAYTOP is the top line of the playing area (the actual background picture). The default value is 0, but it is usually set to 1. If it is left at 0, the menu bar will cover up the first eight rows of pixels when it is displayed.

PLAYTOP should always be limited to a value between 0 and 4, since the playing area is 168 pixels high (21 lines). Setting it to 5 or higher will result in portions of the playing area being off screen, and unpredictable graphical output.

The input line location is set by INPUTLINE. The default position is row 23, and most Sierra games put the input line just below the bottom of the screen, usually on row 22.

The default position of the status line is row 21, just below the background picture. Most Sierra games used a value of 0, so the status line would show on the same row that the menu bar is on. Note that the menu bar cannot be moved; it will always display on line 0.

Normally, the configure.screen command is called once, when a game first starts. Once called, the configuration will stay set, even if a game restart or game restore occurs. But configure.screen can be called any time in a game if a change is desired.

If the playing area is relocated while a picture is currently being displayed, AGI does not erase the existing picture; it will draw the picture at the new location, and there will be portions of the previous picture visible above or below the current picture. Use the clear.lines or clear.text.rect to clear those unwanted areas.

When using configure.screen while a game is in progress, be aware that the input line does not immediately change location when configure.screen is run. This is because configure.screen does not change the current cursor location; it only changes the stored value that AGI uses to reset the input line. The cursor location is adjusted to match the value of the input line when the line is reset after a player presses ENTER or when a new.room or graphics command is encountered.

The input line location is also accessed by several other commands, including accept.input, prevent.input, echo.line, and get.num. If you change the location of the input line using the configure.screen command, these commands will not work properly until the input line is reset, because the cursor and input line location will not match.

Avoid using values for these arguments that result in either the input line or status line being displayed inside the play area. The results will be unpredictable. Also, avoid using values greater than 24 for input line or status line; if you do, the screen will not display correctly due to buffer overflows in the functions where AGI draws the input line and status line.

 

Example

configure.screen(1, 22, 0); [ typical configuration with status line 
                            [ on top and input line on bottom, just below the play area

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

111 (0x6F)
