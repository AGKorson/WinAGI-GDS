print.at.v

The print.at.v command indirectly displays a message in a window at a specified location.

 

Syntax

print.at.v(var vMSG, byt ROW, byt COL, byt MAXWIDTH);

Remarks

IMPORTANT NOTE: This '.v' command is different from others in that only the first argument is a variable. The other arguments are byte values. All the other '.v' commands require ALL arguments to be variables.

The text of the message referred to by variable vMSG is displayed in a bordered window. The background color is always white, and the foreground color is always black.

The window is positioned so the upper left corner of the text is placed at row ROW, column COL. The ROW value is relative to the top row of the background picture image, so for example, if you configured the screen with a PLAYTOP value of 1, then a ROW value of 5 will print on screen line 6.

The width and height of the window is determined by the length of the formatted message and the MAXWIDTH argument. If zero is passed for MAXWIDTH, it defaults to 30. The message is formatted to add line breaks such that no text extends further than MAXWIDTH. If the length of the text is <= MAXWIDTH - 1, the text will fit on one line and the actual width will be the total text length plus one. If the text is too long to fit on one line, AGI will automatically add line breaks, and the total width will be one more than the length of the longest line of text, up to MAXWIDTH. If there are no spaces and the text is longer than MAXWIDTH AGI will force a break and set the width to MAXWIDTH.

When determining values for ROW, COL and MAXWIDTH, the programmer should ensure the window will fit on the screen. If the combination of ROW, COL and MAXWIDTH results in any part of the window falling outside the picture boundaries, the window borders and text will not display properly. Specifically:

- You should not use a MAXWIDTH value of 1 as this may crash AGI.

- If COL <= 1 the window will not draw correctly because there is not enough room to draw the left window border correctly;

- If COL plus the actual width is >= 39 the window will not draw correctly because there is not enough room to draw the right window border;

- If ROW plus the number of displayed lines > 23, the window will not draw correctly due to overlapping off the bottom edge of the screen;

- If any part of the window extends beyond the background picture image, it may not clear correctly. For example, using a value for ROW of 0 will erase part of the status line if the status line is also set to display on row 0.

 

Reserved flag f15 (no pause windows) and reserved variable v21 (window close time) determine how the print.at.v command affects game play:

    • f15 reset to FALSE and v21 == 0 (the default condition): displayed windows (including those shown by the print.at.v command) pause the game cycle, and remain until the player presses ENTER or ESC key. Although the game cycle is paused, the internal clock does not stop, and clock variables v11, v12, v13, v14 will continue to update.
    • f15 reset to FALSE and v21 > 0: the window will also pause the game cycle, and will be dismissed when the player presses ENTER or ESC key, or after v21 * 0.5 seconds, whichever comes first.
    • f15 is set to TRUE (v21 is ignored): the game cycle is NOT paused, and the window remains open indefinitely. The message window stays open and game play continues until a close.window command is called to dismiss the message, or another message is displayed.

After every time a window is displayed (including by using the print.at.v command), if f15 is TRUE, it is reset to FALSE and variable v21 is not modified. If f15 is FALSE, variable v21 is reset to 0.

(In version 2.089 only, reserved flag f15 is not used; message windows will never print without pausing.)

Messages can contain special format codes to enhance the output. See the Message Format Codes topic for details on available message codes.

In versions 2.089 and 2.272 the number of arguments AGI uses when displaying trace information for print.at is mistakenly set to 3, so when tracing commands in those versions, the trace window will only show the first three arguments. The bug only affects what gets displayed during tracing; the commands still execute correctly. The bug was fixed in version 2.411.

 

Example

#message 5 "The snake eyes you warily."
v50 = 5;
print.at.v(v50, 8, 18, 15);  [ prints message 5 at desired location

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

152 (0x98)
