adj.ego.move.to.x.y

The adj.ego.move.to.x.y command appears to affect 'click to move' functionality in the later versions of AGI.

 

Syntax

adj.ego.move.to.x.y();

Remarks

The details of this command are still not fully known. The information provided here may not be accurate.

It has been reported this command sets a reserved flag, although the exact number is not known. This flag would be used in conjunction with the mouse to support 'click to move' functionality. ('Click to move' means the user clicks a point on the AGI screen, and ego will automatically move to that point, stopping once it arrives.)

This command is not known to exist in any MSDOS version of AGI.

 

Example

adj.ego.move.to.x.y();

 

Required Interpreter Version

According to several AGI websites, this command is supposed to be available in "the last versions of AGI version 3", however, there are no known MSDOS based versions that have this command. Which platforms and which version numbers support this command is not yet known.

WinAGI does not support this command.

 

Byte-Code Value

182 (0xB6)
