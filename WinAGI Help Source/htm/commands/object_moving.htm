Moving Objects

The direction that each screen object is moving is represented by a numerical value. This value ranges from 0 to 8:

When an object is moving normally (and has a direction greater than 0), it will continue moving until it is told to stop, move in another direction, or runs into an obstacle (such as a control line, a block, or another object).

Step Size

Each incremental change in the object's position as a result of moving is called a step. When moving, both the horizontal and vertical values are adjusted by the step size.

Every time a new.room or new.room.v command is called, step size for all objects is set to default value of one. The step size can be changed with the step.size command. Do not set step size to 0; the object will never move regardless of the movement mode assigned.

(Note that when AGI begins, all screen objects have a default step size of zero; the default does not change to one until the first new.room command.)

Step Time

The step time property determines how often an object moves. In each interpreter cycle, AGI counts down on a step counter; when the counter reaches zero, the object is moved, and the counter is reset to the step time.

Every time a new.room or new.room.v command is called, step time for all objects is set to default value of one. Step time can be changed with the step.time command. Step time can be set to zero, in which case the object will move once in every interpreter cycle. When step time is set to zero, objects will never observe blocks, and the auto loop feature is disabled.

Don't confuse step time with cycle time. Step time determines how often an object moves. Cycle time determines how often the object's cel changes.

Movement Effect on Cycling

Movement and cycling happen independently of each other. This means that when an object stops moving, it does not automatically stop cycling. You need to use the stop.cycling command to force an object to stop cycling.

Moving Non-ego Objects

For screen objects other than ego, the object's direction can be changed using the set.dir command or by assigning a movement mode. The get.dir command will allow store an object's direction in a variable. As discussed in the Cycling Objects section, changing direction may result in an automatic change in the current loop as well.

There are four movement modes available to control the motion of screen objects:

normal

The object responds to the set.dir command; it will move in a straight line until told to stop, or it hits an obstacle. This is the default mode for the object. Use the normal.motion command to return to normal movement after using one of the other modes.

move object

The object will move toward the target point specified in the move.obj or move.obj.v command. The object will move toward the target point until the distance is zero (or less than the step size specified in the movement command, if > 0). The object's direction is set to zero, the flag specified in the command ('done' flag) is then set, and the movement mode is set back to normal.

If an obstacle is encountered, the object will just stop in place; it will continue to attempt to move each cycle. If the obstacle is another screen object or a block, once the obstacle is gone the object will continue moving. You need to test carefully when using move commands, to ensure the object can actually get where it wants to go. You can use multiple move commands to have an object navigate around obstacles; see the move.obj command for an example of how to do this.

Note that the set.dir command has no effect if used on an object in this movement mode.

follow ego

Use the follow.ego to set this movement mode. The object will move toward the current ego position until the distance is zero (or less than the value specified in the follow.ego command, if > 0). The flag specified in the command ('done' flag) is then set, and the movement mode is set back to normal.

In each cycle, as ego moves, the object will adjust its direction accordingly. If an obstacle is encountered, the object will choose a random direction to try to find a way around the obstacle; once past the obstacle, the object will then continue moving toward ego.

Keep in mind that if specified step size is 1 and the object is NOT ignoring other objects, it will never reach ego, because it can't move onto the same space as ego due to object collision. It will instead wander around the near vicinity of ego.

The set.dir command has no effect if used on an object that is following ego.

wander

Use the wander command to set this movement mode. The object will wander aimlessly about the screen. The interpreter randomly chooses a direction, and an initial distance from 6 to 50 pixels. When the object moves that number of pixels in the chosen direction, AGI chooses another random direction and distance. If an obstacle is encountered, the object will immediately choose another random direction (but only chooses a new distance if the current distance counter is less than six) and continue wandering.

The distance value counts down each cycle, and when it reaches zero, it should choose a new random direction and distance. However, there is a bug in the code that instead sets the distance to 255, so the object tends mostly change direction by bumping into an obstacle or other object. In most situations, the effect is not very noticeable, because rooms typically don't have large open spaces where this bug can be easily noticed.

Another bug occurs if the object hits the edge of the screen (left/right or top/bottom) while moving diagonally. In this case, the object will continue moving along the edge (the non-blocked direction will continue to change) without being marked as 'stopped'. The object has to hit a corner before AGI will force it to choose a new random direction. When moving along the right or left edges in this manner, the object will do a 'moon walk', because the loop will still show the left/right view even though the object is moving vertically.

The set.dir command will temporarily set the direction of the object, until another random direction and distance is chosen.

In addition to setting the movement mode, movement must be enabled before the object will actually move. Objects are initialized with movement enabled by default. Use the stop.motion command to stop movement. Use the start.motion command to enable movement. Keep in mind that the stop.motion command will also cancel the current movement mode. If you re-enable motion later, you will need to reassign a movement mode to move the object.

 

Moving the ego Object

The ego object (object 0) is different from other screen objects. The ego object can operate in two control modes, player control and program control.

Use the program.control command to switch ego to program control. While in program control, ego essentially behaves as any other object. The cycle modes, movement modes, and positioning commands work as they do for other objects.

The exception to this is the set.dir command; it has no effect on the ego object. To programmatically set ego's direction, change the value of reserved variable v6. The get.dir command does work with the ego object, but it is just as easy to check the value of v6 instead.

Use the player.control command to switch back to player control. (When AGI starts, player control is the default.) Switching to player control also switches the ego object back to normal movement mode.

If the move.obj or wander commands are issued while in player control mode, AGI first switches to program control, then changes the movement mode.

The follow.ego command has no effect when used on the ego object (for obvious reasons). On the very next interpreter cycle it will immediately reach its destination and switch back to normal motion mode. The 'done' flag will be set.
