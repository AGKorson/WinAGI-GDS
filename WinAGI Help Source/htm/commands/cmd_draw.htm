draw

The draw command draws an object on the visual screen so it will be visible in subsequent interpreter cycles.

 

Syntax

draw(obj oA);

Remarks

A screen object will not appear on the graphics screen until the object is drawn using the draw command. Prior to drawing, the screen object should be initialized, assigned a view, and positioned.

If screen object oA is not properly initialized when the draw command is executed, AGI will try to access invalid memory while trying to position and draw the object, with unpredictable results, including possibly crashing AGI.

If a view has not been assigned to screen object oA, AGI will raise trappable error #20.

When the draw command is executed, AGI first checks that the object's number is less than or equal to the max screen object number; if not, it raises trappable error #19.

Once the object number is validated, AGI enables it for updating (if not already enabled). This means that if you want stop updating the object, you must call stop.update AFTER the draw command, not before.

After enabling updates, the draw command validates the object's position using the shuffling method. The object is then added to the screen, where it will be visible and updated in subsequent interpreter cycles.

If the draw command is used on an object that is already visible AGI ignores the command.

Use the erase command to erase a screen object from the screen.

 

Example

animate.obj(o5); [ initialize first
load.view(8);
set.view(o5, 8);
position(o5, 72, 144);
draw(o2);  [ draw it AFTER initializing

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

35 (0x23)
