Cycling Objects

The animation of a screen object is called "cycling". This is the successive display of several cels from the same loop in order. When you initialize an object, it cycles the cels in the first loop (loop 0) by default. To change the loop number, use the set.loop command. To change the speed of cycling, use the cycle.time command.

Objects must be enabled for updating in order to cycle. In addition, each object maintains a separate cycling property that must also be enabled. Both updating and cycling properties are enabled by default when a screen object is initialized.

Cycle Time

The cycle time property determines how often an object's cel is changed. In each interpreter cycle, AGI counts down on a cycle counter; when the counter reaches zero, the object's cel is cycled according to the current cycle mode and the counter is reset to the cycle time.

Every time a new.room or new.room.v command is called, cycle time for all objects is set to default value of one. Cycle time can be changed with the cycle.time command. Cycle time can be set to zero, in which case the object will not cycle its cel, regardless of the assigned cycle mode.

(Note that when AGI begins, all screen objects have a default cycle time of zero; the default does not change to one until the first new.room command.)

Don't confuse cycle time with step time. Cycle time determines how often the object's cel changes. Step time determines how often an object moves.

It is also important to understand that cycling and motion occur independently. When an object stops moving (either by setting its direction to zero, or by encountering an obstacle) it will continue to cycle. This means the programmer has to manually stop cycling when the object stops moving, otherwise the object will continue to cycle in place. It is surprising that Sierra did not include built-in functionality to automatically start/stop cycling for moving objects (similar to how AGI automatically assigns loops to moving objects). For sample code that shows how to manage synchronizing cycling and movement for the ego object, see the stop.cycling command topic.

Cel Position Validation

When the new cel is assigned to the object, AGI validates that the cel will still fit on screen. If the new width or height would result in the object being over the right or top edge respectively, AGI automatically repositions it on the edge. AGI then sets the internal 'repositioned' flag which means the object will not move in the next interpreter cycle.

After checking the edges, if the object got repositioned for overlapping the top edge, AGI then checks that the object fits below the horizon (if the object is not ignoring the horizon). This check is actually unnecessary, since the object's position is always checked during the graphics update portion of the interpreter cycle and will be moved below the horizon if needed at that time.

This position check is not the same as the shuffle function that AGI uses in other situations to validate screen position of screen objects, such as during collision checks.

Auto Loop Feature

To facilitate changing the profile of the object as it moves about the screen, AGI has a built-in feature that automatically assigns a loop based on the object's direction of motion. For example, when walking to the right, show a right-facing profile, when walking to the rear, show a rear view, etc. See the Autoloop topic for additional information.

Cycle Modes

There are four modes of cycling available to an object:

Forward Cycle

In the forward cycle mode, the cels are displayed in increasing numerical order, i.e., cel 0, then cel 1, etc. After the last cel is reached, AGI goes back to cel 0 and starts over. The view will continue to cycle indefinitely. This is the default mode. Use the normal.cycle command to switch back to this mode.

Reverse Cycle

In the reverse cycle mode, the cels are displayed in decreasing numerical order, i.e., cel X, cel X-1, etc. (where X is last cel). After the first cel is reached, AGI goes back to cel X and starts over. The view will continue to cycle indefinitely. Use the reverse.cycle command to switch to this mode.

End of Loop

In this mode, the cels cycle in the forward direction, but once the last cel is shown, cycling stops. Use the end.of.loop command to switch to this mode.

Reverse Loop

In this mode, the cels cycle in the reverse direction, but once the first cel is shown, cycling stops. Use the reverse.loop command to switch to this mode.

 

When changing modes, the current cel for the object is not reset. The cycling will begin at the current cel, and continue in the direction determined by the new cycle mode. You may need to manually set the cel so the animation works correctly. This is most important for the end.of.loop and reverse.loop commands. For example, if the current cel is the last cel, and you switch to end.of.loop mode, nothing will happen because the object is already showing the last cel.

Cycling can be disabled by using the stop.cycling command and re-enabled with the start.cycling command.

If you want to display a single cel without animation, set the loop number and cel number you want to display using the set.loop and set.cel commands and then use the stop.cycling command.
