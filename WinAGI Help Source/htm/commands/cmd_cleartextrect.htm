clear.text.rect

The clear.text.rect command clears a rectangular text area on the screen.

 

Syntax

clear.text.rect(byt R1, byt C1, byt R2, byt C2, byt COLOR);

Remarks

This command can be used on both the text screen and the graphics screen.

The command clears a rectangular area from row R1, column C1 to row R2, column C2. AGI does not validate the argument values, so if either row argument is >24, or either column argument is >39, or R2 < R1 or C2 < C1, the results will be unpredictable.

If the area cleared will encompass the entire row (C1 = 0 and C2 = 39), it is more efficient to use the clear.lines command.

On the graphics screen, the color used to clear the area is specified by COLOR. If COLOR is 0, the lines are cleared to black. if COLOR in nonzero, white is used to clear the area. Keep in mind that if any screen objects move over the cleared area, the screen will revert back to the original pixels once the object has moved past; the cleared area is not stored in memory.

On the text screen, the color argument COLOR is ignored. The area is always cleared to black.

If you want the cleared area to have a color other than black or white, a workaround, using a single space message and the display.v command, is demonstrated in the example below.

 

Example

clear.text.rect(2, 2, 10, 20, 0);

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

154 (0x9A)
