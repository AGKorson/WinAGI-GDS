new.room

The new.room command changes the current room number and forces an update.

 

Syntax

new.room(byt ROOMNUM);

Remarks

The new.room command causes AGI to halt execution of the current script. The current room number is changed to ROOMNUM. The resources on the memory heap are dumped (with the exception of logic 0). The following actions are then taken to switch to the new room:

    • if a sound is playing, it is stopped and the 'done' flag is set to TRUE;
    • the memory heap is cleared by resetting the memory pointer to room.0;
    • the keyboard buffer and joystick status are cleared;
    • the script table is cleared;
    • the screen objects table is cleared - for each screen object, the DRAWN and ANIMATED bit field flags are cleared and the UPDATE bit field flag is set; the step size, step time, step counter, cycle time and cycle counter properties are set to 1; pointers for cel data, view data and background data are all set to zero; all other bit field flags and object properties are unaffected;
    • all resource linked lists are cleared;
    • ego is set to player control;
    • if a block is set, it is cleared;
    • horizon is set to default value of 36;
    • reserved variable v0 (current room) is copied to reserved variable v1 (previous room);
      • , and v0 is set to ROOMNUM;
    • reserved variables v4 (object touching edge) and v5 (object edge code) are set to zero;
    • reserved variable v16 (ego view number) is set to current ego view number;
    • all open VOL files are closed (they will automatically be reopened when the logic for this room is loaded);
    • the logic for current room (v0) is loaded (note that the logic is loaded, but is not called automatically; you should use the call.v command in logic 0 to call the room's logic in each cycle);
    • if a trace logic is specified, it gets loaded;
    • if ego was touching an edge on the previous screen, it is positioned to opposite side of the new room ( i.e. if ego touched right edge, it is moved to left edge);
    • reserved variable v2 (ego edge code) is set to zero;
    • reserved flag f5 (new room) is set to TRUE;
    • the controller flag table is cleared;
    • the status line, if enabled, is redrawn;
    • if the input line is enabled, it is redrawn, any text already on the input line before the new.room call is maintained;
    • reserved variable v9 (unknown word number) is set to zero;
    • reserved flag f2 (input received) is reset to FALSE (but reserved flag f4 (said found match) is not modified; this appears to be a minor bug, but it usually not an issue);
    • the current score (v3) is cached;
    • the interpreter cycle is then continued by jumping to the point where logic 0 is executed.

Keep in mind that the new.room command interrupts the interpreter cycle, so any commands after a new.room command will never get executed.

Also note that the new.room command does not affect the current graphics display mode; it will remain in the same mode.

 

Example

[ typical code that changes rooms when ego reaches an edge
if (edgeEgoHit == LEFT) 
  {
  new.room(75);
  }

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

18 (0x12)
