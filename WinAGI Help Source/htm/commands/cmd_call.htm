call

The call command executes a logic script.

 

Syntax

call(byt LOGICNUM);

Remarks

The logic with resource number LOGICNUM is executed one time. If it is not currently loaded when called, it is loaded, then executed, and then unloaded after returning. If the logic does not exist, AGI will display a 'resource not found' message, and immediately quit.

A script entry is not made for logics that are loaded/unloaded in support of the call command.

If a logic is likely to be called more than once or twice, it is better to load it using the load.logics command. This avoids having AGI continually load and unload the same logic over and over.

If other resources are loaded in a logic that was not loaded before being called, those resources will also be unloaded after the called logic returns. If you intend to use resources loaded in a called logic after the call returns, make sure you load the logic before calling to avoid unexpected memory corruption.

There is also a bug in the code for the call command that will improperly reset AGI's heap memory pointer if a new.room or new.room.v command is called from within a logic that was not explicitly loaded. Internally, AGI tracks the memory location where the temporary logic gets loaded. After the new.room command resets the memory pointer to room.0, control of the game loop goes back through the called logic, where AGI resets the memory pointer to where it was prior to the called logic being loaded, which will include all resources in use at that time. Memory will be correctly restored if another new.room command is encountered, but the temporary overflow condition can certainly cause unexpected memory issues if not recognized.

 

Example

[ typical error handler
[ assume logic ID for error handling logic is 'ErrHandler'
if(errorNumber > 0)
  {
  call(ErrHandler);
  }

 

Required Interpreter Version

Available in all AGI versions.

 

Byte-Code Value

22 (0x16)
