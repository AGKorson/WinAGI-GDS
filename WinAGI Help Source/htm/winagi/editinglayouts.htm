Editing Layouts

Add rooms and arrange them based on your game design. Use the Show or Hide Room commands if you need to change a logic's IsRoom property.

Use the layout editor's grid property to help align rooms.

Then add exits to build the paths that connect the rooms. Make adjustments by editing exits.

Use transfer points to reduce clutter on the layout.

Add comments to insert notes to aid in game development.

The status bar displays useful information while using the layout editor.

Use the Layout tab of the Settings window to adjust various settings specific to the Layout Editor.

Scale Control

Use the scale controls to control the scale of the layout editor. The mouse wheel can also be used to adjust scale.

Adding Rooms

Use the Add Room tool to add new room to the layout (which adds a new logic to your game). Click on the drawing surface where you want the new room. The Add New Resource dialog is shown, allowing you to assign a number and Resource ID, and optionally include a matching picture resource.

Use the Edit Logic command (or double-click on the room) to open the logic for editing.

Showing and Hiding Rooms

To toggle the IsRoom property of logics, use the Show and Hide Room commands (or toolbar buttons).

To hide a room, select it , then use the Hide room command or button. (Note that you can't hide multiple selections at one time; you must select and hide them individually.)

To show a room that currently has its IsRoom property set to false, make sure there is nothing currently selected, then use the Show Room command or button. The room will be placed in center of drawing window. You can then move it to a new position.

Moving Objects

Using the select tool, objects (rooms, transfer points, error points and comments) can be moved by dragging them with the mouse.

You can select multiple objects by dragging the mouse over the objects, or by holding down the CTRL key and clicking on multiple objects.

Selection handles (small black squares) are shown around objects as they are selected. If multiple objects are selected, a selection rectangle (dotted line) is also shown. Dragging any object is a selected group will move all of them by the same relative amount.

You can also move selected objects by using the cursor keys. When pressed by themselves, each key press moves the objects one minor grid unit. Use the Shift key to move objects in larger increments (one major grid unit) or the Control key to move a single pixel at a time.

Deleting Objects

Use the Delete command (or toolbar button) to delete selected objects.

Deleting a room in the layout editor will actually remove the logic from your game. Make sure you really want to remove it from your game before using the Delete command. If there is a picture resource with the same number associated with the logic being deleted, you are given the choice to delete it as well.

Use the Hide Room command if you want to keep the logic in your game, but want to change its IsRoom property to false.

Object Order

Objects are drawn in the order they are added to the layout. Normally, this isn't of concern, but if you decide to arrange objects so they overlap, the most recently added object will be on top. Use the Move Up, Move Down, Bring to Front and Send to Back edit commands to adjust the drawing order of objects.

Adding Exits

To add an exit between two logics, select one of the exit tools:

    • One way edge exit. Use this tool to add a single edge exit to a logic.
    • Two way edge exit. Use this tool to add reciprocal edge exits between two logics.
    • Other exit. Use this tool to add an exit that tests for a condition other than an edge code.

Place the cursor over the desired 'from' room. For edge exits, make sure the cursor is near the edge you wish to use. Click and drag the exit to the 'to' room. Release the mouse to add the exit.

Once added, you can use the Edit Logic command (or double-click on an exit) to open the logic to edit the source code for the exit.

Editing, Moving and Deleting Exits

To move an exit, first select it. Selection handles (small black squares) are placed at each end.

Drag a selection handle to change the 'from' or 'to' room of an exit. WinAGI will automatically comment out source code or add new source code to reflect the change.

You can move two way exits as a pair or individually. To move them as a pair, click on the exit line, then drag either selection handle.

To move them individually, click on the one of the exit arrowheads, not the line. A red selection handle is shown to indicate you have selected only one direction of the two way exit. Drag either black handle to move the exit (not the red handle).

Inserting Transfer Points

To insert a transfer point, select an exit, and use the Insert Transfer Point command (or use the toolbar button).

The default location for the transfer point pair is in the middle of the selected exit. You can move them to a different location once they are added.

Double-clicking on any transfer point will automatically select the other transfer point in the pair, scrolling the drawing surface, if necessary.

Adding Comments

You can add comment blocks to include production notes, to-do items, or explanatory remarks about your game and layout.

To add a comment, select the Add Comment tool. Click on the drawing surface to set the upper left corner, then drag the comment box to the desired size.

A text cursor is placed in the newly drawn comment box. Press the Enter key after entering comments to finish.

To edit an existing comment, double-click it to enable the text cursor.

To resize a comment, select it, then drag one of the selection handles.

Status Bar

The status bar displays the following information:

    • Scale: The current image scale. 1 is minimum scale; 9 is maximum scale.
    • Tool: The currently selected tool. If a tool is in hold mode, the tool is prefaced by the word "HOLD".
    • Selection Information: Display depends on what is selected.

      • If a room is selected, its Resource ID is displayed.

      If an exit is selected, the exit comment tags, the exit type, and the from and to logics are displayed.

      If a transfer point is selected, the two logics associated with the transfer are displayed.

    • Cursor Position: the X and Y coordinates of the cursor are displayed in the rightmost status bar panels.
