Saving Resources

When changes are made to a resource, the Save command is enabled.

 

If a game is NOT open:

The resource is saved to file, using its current file name. If the resource is new, and doesn't have a filename yet, the Save As dialog is opened so you can select a filename.

 

If a game IS open:

If the resource you are editing is NOT part of your game, it gets saved to file as described in the previous paragraph.

If the resource IS part of your game, your game files are updated. The resource is automatically stored in the first VOL file that has enough room for it. The DIR file is automatically updated for the resource's new location.

Save As...

The Save As command is available for resources that are NOT in a game. This command allows you to save a resource to a different filename.

For resources that are in a game, the Export Resource command can be used to save a resource to a file, or to save it in a format other than native AGI resource format.

 

Save File Format

The default file format for saving picture, sound, view, OBJECT and WORDS.TOK resources as files is the native AGI resource format. See the Exporting Resources topic for other formats that are available for these resource types.

Saving Logic Source Code Files and Logic Resource Files

When it comes to saving, logics are slightly different from other resources.

When saved, the source code file (*.lgc or *.txt) is the actual file saved. The resource in a game's VOL file is not affected by the Save command.

Use the Compile command to convert the source code into a compiled resource. Once compiled, the resource is updated in the games VOL files. Only logics in a game can be compiled.

To save the actual compiled logic resource to a file (*.agl), use the Export Resource command.
