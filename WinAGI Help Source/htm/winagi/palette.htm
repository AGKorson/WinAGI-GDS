NOTE: Changing colors in WinAGI while editing your game has no effect on the colors used by AGI when the game is running unless using advanced features as described below.

Color Limitations in AGI

AGI's original color palette is limited to sixteen colors (not including the rarely used limited palettes for HGC and CGA in RGB mode). See the AGI Color Palette topic for additional information.

There is a fan made 'hack' to AGI, called AGIPAL, that allows you to change palette colors using the shake.screen command. Nick Sonneveld's AGI emulator NAGI, in version 2.09 and later, also supports use of modified color palettes.

The AGI Power Pack also provides the ability to change color palettes.

Custom Color Palette

If you want to design a game that uses its own color palette for special effects, WinAGI provides you the ability to change the colors in the palette while you are designing your game.

To access the Modify Color Palette dialog, use the Color Palette option on the Tools menu.

When a game is loaded, the custom colors chosen with the Custom Color Palette tool will apply only to the loaded game. To change the default colors to use when a new game is created, or when no game is loaded, open the Custom Color Palette tool when no game is loaded.

 

 

 

Palette Display Grid

All sixteen colors in the current palette are displayed in a grid. Select a color index by clicking it. The default color for this index is shown below along side the current color. Click the change color button to display the Select Color dialog, which allows you to select a new color.

Default Color

The default color for each color index in the palette is defined by the original Sierra AGI, and shown in the left side of the color selection box.

Current Color

The current color for each color index is shown in the right side of the color selection box. To change the current color, Click the Change Color button (the small button with the ellipsis [...]). Alternatively, you can double-click a color in the display grid to edit a color.

Load Palette

NAGI version 2.09 and higher provide the capability to run games with different color palettes. The colors are defined in the nagi.ini file that should exist in your game's directory. WinAGI makes it easy to coordinate the color palette used by allowing you to import and export the color palette settings directly from/to any nagi.ini file. Use the Load Palette button to select a nagi.ini file from which to extract a color palette.

Save Palette

Once you have a palette that you are satisfied with, you can easily add them to a nagi.ini file by using the Save Palette command.

(The file used to load a palette doesn't have to be a full nagi.ini file; it can be any text file as long as long as it has the color data in the correct format. Similarly, you can save the palette to any text file; the save function will add the data in the correct nagi.ini format. Also, nagi uses the British spelling 'colour' instead of 'color', in case you are examining the files in a text editor, looking for the palette data.)

Restore Default Palette

If you make changes to your color palette, then later decide you want to go back to the original AGI palette, use this option to reset all sixteen colors back to their default values.
