Logic Compiler Error 6016

'<commandtype>' command not defined

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, when using alternate syntax for action and test commands, the commands must be declared with the exact spelling (and all lower case) as indicated below:

Action Commands:

    • increment
    • decrement
    • assignn
    • assignv
    • addn
    • addv
    • subn
    • subv
    • lindirectn
    • lindirectv
    • rindirect

Test Commands:

    • equaln
    • equalv
    • lessn
    • lessv
    • greatern
    • greaterv
    • isset

To correct this error, you will need to add a define for the specified command that has the correct spelling.

%action Addn(VAR, NUM) 1   [ uppercase 'A' instead of lowercase 'a'
v99 += 1;   [ NOT OK, addn command not defined correctly
...
%test equal.n(VAR, NUM) 1 [ not spelled as 'equaln'
if (v99 == 1) {    [ NOT OK, equaln command not defined correctly
...
