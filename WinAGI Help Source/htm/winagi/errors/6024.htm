Logic Compiler Error 6024

'<includefile>' already included

Only applicable when using Sierra syntax.

This error indicates the file you are attempting to include has already been included and processed. Sierra's logic compiler CG.EXE does not check if a file has already been included, so attempting to include it again would raise duplicate definition errors for all of the defines and declarations in the file. WinAGI uses this error instead to minimize error entries and provide more clarity.

This error often comes up when using nested includes and forgetting which includes are in which file. For example,if all your logics include 'gamedefs.h', and 'gamedefs.h' includes 'sysdefs.h', and you then add a separate entry in your logic code to also include 'sysdefs.h', this error will be raised.

To clear the error, remove the duplicate include entry.
