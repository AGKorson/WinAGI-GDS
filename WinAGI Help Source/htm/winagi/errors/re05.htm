Resource Load Error RE05

<restype> <number> has an invalid resource header at location <location> in volume file <number>

When WinAGI attempts to load a resource, if the first two bytes are not 0x12 and 0x34, it is not a valid AGI resource. This is most likely due to invalid pointer in the DIR file, usually indicating an abandoned resource where the DIR file was never updated. There are several instances of this error in Sierra games. Removing the resource from your game is the only way to fix this anomaly.
