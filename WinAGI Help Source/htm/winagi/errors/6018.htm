Logic Compiler Error 6018

Fan syntax assignment statement not supported in Sierra syntax

Only applicable when using Sierra syntax.

Sierra syntax does not support all forms of alternate syntax that Fan syntax supports. Sierra's alternate syntax for increment and decrement requires the '++' or '--' symbol to appear before the variable; it cannot come after the variable. Also, Sierra syntax does not support any alternate syntax for the mul.n, mul.v, div.n, and div.v commands. You must use the full command syntax for those commands.

v99++;   [ NOT OK in Sierra syntax; OK in Fan syntax
v99 *= 2;   [ NOT OK in Sierra syntax; OK in Fan syntax
...
++v99;   [ OK in Sierra syntax and Fan syntax
muln(v99, 2);   [ OK in Sierra syntax and Fan syntax
...
