Logic Compiler Error 6027

Too many goto statements

Only applicable when using Sierra syntax.

This error is raised when there are too many goto statements associated with a label that has not yet been defined.

When the compiler processes a goto statement, if the label comes AFTER the goto statement, the compiler cannot calculate the jump value so it uses a placeholder and saves the location where the jump value will go. If multiple goto statements use the same label, each location is saved as described. Once the label is found, the compiler fills in the jump values for the goto statements using the stored locations.

To maintain compatibility with Sierra's logic compiler CG.EXE, the number of jump locations that can be saved for an undefined label location is 19.

Once the label is defined, jump values do not need to be saved, so the number of gotos is no longer limited.

To clear this error, you will need to refactor your code to use fewer goto statements.
