Logic Compiler Error 6031

<labelname> is already in use as a define name and cannot be used as a label

Only applicable when using Sierra syntax.

Sierra's logic compiler CG.EXE has a bug that actually will allow the use of a define as a label, and a goto statement can use it, but it always inserts an invalid jump value. WinAGI instead throws this error.

%define Num1 1
...
goto Num1   [ NOT OK; can't go to a define value
...
