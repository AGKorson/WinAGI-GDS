Logic Compiler Error 6014

Invalid define value '<token>'

Only applicable when using Sierra syntax.

When using the #define preprocessor token, the value assigned can be a number, a literal string, a symbol, or a previously defined token name. If the value is an undefined identifier, this error will be raised.

Using the symbol ':' as a define value will also raise this error. While Sierra's logic compiler CG.EXE will allow you do do this, if you try to use ":" as a define name in your source code, CG.EXE will crash with a buffer overflow error.

[ order of defines does matter:
%define avar bvar   [ NOT OK; bvar not defined yet
%define bvar 1
...
%define bvar 1
%define avar bvar  [ OK; avar and bvar will both have a value of '1'
...
[ literal strings are allowed
%define msgtext "a message"  [ OK
%message 1 msgtext           [ this will compile as '%message 1 "a message"'
...
[ you can create defines for most symbols
%define equals ==    [ OK, but not very useful
if (v1 equals 1) {   [ this will compile as 'v1 == 1'
}
...
[ but : is not allowed
%define colon :   [ NOT OK
