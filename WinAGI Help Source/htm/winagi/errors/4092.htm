Logic Compiler Error 4092

Undefined token '<tokentext>' in 'if' statement

This compiler error is raised when the compiler expects a test command, or a variable or flag as part of an alternate syntax, but an undefined token is encountered. This is most likely caused by typographical errors, but can also occur if the Use Reserved Define Names property is disabled for logics that previously used reserved names. Check the spelling of test commands and variable names, re-enable the Use Reserved Define Names option, or add a global or local define name to correct this.

#define avar   v99
if (avra == 0)  [ NOT OK; misspelled 'avar'
