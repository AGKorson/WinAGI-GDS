Logic Compiler Error 6006

Invalid argument type or syntax error in <commandtype> preprocessor statement

Only applicable when using Sierra syntax.

When defining action and test commands, the argument list for the command must be a comma separated list of valid argument tokens (NUM, FLAG, VAR, OBJECT, VIEW, MSGNUM, or WORDLIST).

Using any other argument value, or failing to properly separate the arguments with commas will raise this error.

%action assignn(VAR NUM) 1  [ NOT OK; missing ','
%action assignn(variable, number) 1  [ NOT OK; invalid argument types
%action assign(VAR, NUM) 1  [ OK
...
