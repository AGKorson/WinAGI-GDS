Logic Compiler Error 6015

Using message number of zero will corrupt the compiled logic

Only applicable when using Sierra syntax.

Due to the way AGI manages encryption of message data in logic resources, there is no message with index of zero. Although Sierra's logic compiler CG.EXE will compile it without displaying an error, the logic will actually have a corrupted message data section.

You will need to renumber the message to a value between 1 and 255 to clear this error.

%message "BAD MESSAGE" 0  [ NOT OK; will corrupt the output data
