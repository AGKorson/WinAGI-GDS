Logic Compiler Error 6028

Using '&' or '|' in tokens other than " &&" or "||" will crash Sierra's compiler

Only applicable when using Sierra syntax.

Due to a bug in Sierra's logic compiler CG.EXE, the characters '&' and '|' should never be appended to another token. They must always be separated by a space, tab character or new line character.

If a token tries to add one of these characters, CG.EXE will enter an infinite loop and freeze up. To To maintain compatibility with CG.EXE, WinAGI will raise this error when one of these characters are appended to another token.

You will need to remove the '&' or '|' character from your source code in order to clear this error.

#define badtoken abc&   [ NOT OK, will crash CG.EXE
addn(v99, 1&);   [ NOT OK, will crash CG.EXE
v99 =& 1;    [ NOT OK, will crash CG.EXE
if (v99 == 1 &| v99 == 2)   [ NOT OK, will crash CG.EXE
