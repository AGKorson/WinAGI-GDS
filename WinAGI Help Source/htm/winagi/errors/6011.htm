Logic Compiler Error 6011

Argument list mismatch in <commandtype> command preprocessor statement

Only applicable when using Sierra syntax.

This error is raised when the size of the argument list for an action or test command does not match the expected size. Although Sierra's logic compiler CG.EXE will allow the command to be defined this way, attempting to use it in an AGI game will cause it to crash because the incorrect argument count will cause a misalignment in the logic data as it is processed.

You will need to correct the command definition to include the correct number of commands.

%action increment(FLAG, NUM) 1   [ NOT OK; increment only uses one argument
%action increment(FLAG) 1   [ OK
...
