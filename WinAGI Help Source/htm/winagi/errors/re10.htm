Resource Load Error RE10

Sound <number> has one or more invalid track pointers

This error is raised if WinAGI detects a track offset pointer that is beyond the end of the sound resource. If you attempt to play this sound in AGI, the interpreter will read data from the erroneous location in memory, with unpredictable results.

WinAGI will ignore the invalid pointer and clear the affected track. Any tracks with valid pointers will still be loaded. There are several known instances of Sierra games where sounds have invalid track pointers.

Opening and saving the sound in WinAGI will correct this anomaly.
