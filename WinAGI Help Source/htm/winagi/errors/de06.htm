Logic Source Decompiler Error DE06

Expected block end does not align with calculated block end at resource index <pos>

This warning is raised when the calculated end of a block is in the middle of another AGI command. For example, if the calculated end of a block is at position 100, but there is an AGI command at position 99 that has three arguments, the next valid command is at position 103; jumping to position 100 will most likely end up crashing AGI.

This is likely due to a corrupt logic resource, or an error that existed in the original Sierra release disks. If it is an original error, it probably is in abandoned code that never gets run, otherwise the error would probably been identified long ago.

WinAGI ignores the misalignment and will continue to decompile the logic. You should confirm that the VOL file containing this logic is not corrupt. Regardless, you will need to carefully examine the decompiled source code, and adjust the code as necessary before recompiling.
