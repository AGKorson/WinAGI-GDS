Logic Compiler Error 4070

Variable or number value expected; '<token>' is not a variable or number

When using alternate syntax in an if block, the compiler expects a variable or number value after a comparison operator such as '=='. This compiler error is raised if the compiler encounters a symbol or token that is neither a variable or number. This is often due to a typographical error, a misspelled argument, or turning off reserved defines without explicitly defining names for reserved variables and flags. Some examples:

if(v99 === v100)        [ NOT OK, extra '=' is unexpected symbol
...
if(currantRoom == 0)   [ NOT OK, spelling error
...
if(currentRoom == 0)   [ will throw an error if IncludeReserved property of your game 
                       [ is false and currentRoom is not otherwise defined
...
