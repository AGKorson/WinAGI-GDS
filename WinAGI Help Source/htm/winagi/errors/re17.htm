Resource Load Error RE17

Invalid data in OBJECT file, unable to decrypt

If WinAGI determines that an OBJECT file is not cleartext, it assumes it is encrypted and will attempt to decrypt it. This error indicates that the file is not cleartext, but after decryption, the file is still not a valid OBJECT file.

The OBJECT file is corrupt and most likely unrecoverable. WinAGI treats the file as having a single null item. You will need to re-enter items manually to recover.
