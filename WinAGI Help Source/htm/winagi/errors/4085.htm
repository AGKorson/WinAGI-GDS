Logic Compiler Error 4085

Invalid variable or number value '<define_name>' in arithmetic statement

This compiler error is raised if the compiler encounters an invalid variable value or invalid numerical value when attempting to compile an arithmetic command using alternate syntax, for example:

v99 = v99 + v300;  [ NOT OK, invalid argument value
v99 = v99 + 400;   [ NOT OK, number is too large
v99 = v99 + baddefine; [ NOT OK, if baddefine is not defined locally or as a global define
