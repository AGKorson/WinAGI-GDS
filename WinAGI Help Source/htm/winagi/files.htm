Command Line Options

WinAGI can include a file as a command line argument. WinAGI will open the file automatically after startup. Valid files that can included as a command line argument include OBJECT, WORDS.TOK, *.WAG, *.LGC, *.AGL, *.AGP, *.AGS, and *.AGV.

You can also double-click on a file with one of these extensions to start WinAGI and open/begin editing that file.

File Explorer Drag/Drop Support

You can drag a game's WAG file from Windows File Explorer to the main WinAGI window to open it. You can also drag and drop resource files to import them. WinAGI will automatically enable/disable drag-drop based on the file's extension.

Drag/drop is disabled by Windows if WinAGI is run as Administrator; this is a security feature of Windows. There is no need to run WinAGI as Administrator, so drag/drop should always be available.

 

Console Mode

WinAGI also includes a windowless console mode that provides limited support for importing an existing AGI game, compiling games, and exporting and importing resources.

Usage: winagi.exe --console <dir|gamefile> actionswitches [/q]

valid action switches:

    • /i: imports an existing game in <dir> and creates new wag file with default GameID. Optional import parameters can be added as additional arguments:
        • /C:# sets the code page (default is 437)
        • /E:<ext> sets the file extension to use for source code files (default is 'lgc')
        • /R:<resdir> sets the resource directory (default is 'src')
        • /S use Sierra syntax (default is Fan syntax)
        • /nG don't include global variables in logic source files
        • /nR don't include reserved words in logic source files
        • /nI don't include resource IDs in logic source files
    • /c does a full game compile of <gamefile> in its directory, replacing existing VOL and DIR files
    • /r rebuilds VOL files for <gamefile> without recompiling logics
    • /d compiles all changed logics in <gamefile>, but doesn't rebuild VOL files"; rebuilt logics are put in the first VOL file with available space
    • /e exports all resources in <gamefile> as individual files to the game's source directory
    • /e:L# | /e:P# | /e:V# | /e:S# exports a single resource of the specified type and number in <gamefile> to the game's resource directory
    • /a:L# | /a:C# | /a:P# | /a:V# | /a:S#> adds the resource specified by <resfile> to <gamefile> as the specified type and number. Use 'L' to import a compiled logic resource and 'C' to import a text source file.
    • /help or /? displays help text explaining how to use the console mode. Must be the first parameter on the command line
    • /quiet or /q: optional; suppresses all non-essential status messages. Must be last parameter on the command line.

 

File Extensions

WinAGI recognizes the following file extensions:

WinAGI Game File (*.wag)

The information that WinAGI uses to enhance the development of a game (such as resource IDs, logic compilation status, etc.) are stored in a WinAGI Game File. This file usually has the same name as your game ID, with an extension of wag, but you can change it if you wish. The WinAGI game file is not needed to run an AGI game, it is only used by WinAGI during game development.

 

WinAGI Layout File (*.wal)

The layout file stores the layout information for an AGI game. The layout file is not needed to run an AGI game; it is only used by WinAGI during game development. This file is created and updated automatically by WinAGI. See the Layout Editor topic for more information about game layouts.

 

Logic Resource File (*.agl)

A compiled logic resource, extracted from an AGI game. Equivalent to LOGIC.001, LOGIC.002, etc. files in AGI Studio. Compiled logics contain the actual binary code used by AGI, not the text based source code.

 

Logic Source Code File (*.lgc)

The source code (text file) that is used by the WinAGI compiler to generate a logic resource. Equivalent to LOGIC0.TXT, LOGIC1.TXT, etc. files in AGI Studio.

 

Picture Resource File (*.agp)

A compiled picture resource, extracted from an AGI game. Equivalent to PICTURE.001, PICTURE.002, etc. files in AGI Studio.

 

Sound Resource File (*.ags)

A compiled sound resource, extracted from an AGI game. Equivalent to SOUND.001, SOUND.002, etc. files in AGI Studio.

 

Sound Script File (*.ass)

A text based script file that conforms to the sound script specifications established by Nick Sonneveld. The scripts are compatible with Nick's script-to-sound and sound-to-script tools.

 

View Resource File (*.agv)

A compiled view resource, extracted from an AGI game. Equivalent to VIEW.001, VIEW.002, etc. files in AGI Studio.
