Picture Editor View Test Mode

  The view test mode is an extremely useful feature that allows you to test how views will interact with a picture. This makes it easy to verify picture priorities are drawn correctly to eliminate priority related bugs without having to compile and run a game.

 

To switch the picture editor to test mode, select the View Test Mode command on the Resource menu or click the Test Mode toolbar button.

When the editor enables view test mode, all editing is disabled, regardless of display mode. You can switch to test mode while the display mode is in Step Draw. Just be aware that if you use test mode while the display mode is in Step Draw, you will be seeing and testing your picture only up to the currently selected command. Normally, you will want to test views in Full Draw.

 

 

In the test mode, the main status bar shows the current image scale and mode. When the test view stops, the reason is displayed until the view is set in motion again.

The position and priority panels will display either the current cursor position, or the test view position. Press the space bar to toggle between the two. When displaying test view position information, the panels include the prefix 'v'.

The Test View

When entering the test mode for the first time with a picture, you must choose the view you will test with. For pictures that are part of a game, you will be asked to choose a view from within the current game. For pictures that are NOT part of a game, you must select a view resource file.

During testing, you can change the test view by selecting the Test View command on the Edit menu.

 

Controlling the Test View

Place the test view on the picture by left clicking on the screen. If the test options are set to use the horizon, the cursor changes to the no-drop icon.

To reposition the test view to a new location, just left click again, and the view will move to the new location.

You can also move the test view on the screen by using the cursor keys, just as you would in an AGI game.

If the test view encounters a boundary (edge, barrier, etc) it will stop automatically, and display the reason for stopping in the status bar.

 

Test Options

During testing, you have full control over the behavior of the test view. Use the Test Options command on the Edit menu to display the Picture Test Options window:

The default settings to be used when testing pictures can be changed on the Pictures tab of the Settings window.

 

Horizon

Sets the value for the picture's horizon. When the test view moves up on the screen, it will stop when its vertical position reaches this value (unless set to ignore the horizon).

 

Ignore Horizon

When checked, the test view will ignore the horizon value.

 

Ignore Blocks

When checked, the test view will ignore conditional barrier lines.

 

Object Restrictions

You can restrict the test view to water (priority value of 3), non-water (any priority value EXCEPT 3) or no restrictions.

 

Speed

You can adjust the speed at which the test view moves and cycles. The speeds correspond roughly to the same speeds available in most AGI games.

 

Object Priority

Normally, you'll want to leave the priority of the test object on automatic; as the view moves up and down the screen, its priority will automatically change. This allows you to test how well your priority screen creates the 3D illusion. If you want to force a priority on a test view, regardless of its vertical position, select the priority value you want here.

 

Test View Loop

In auto mode, the loop is chosen based on object direction. To force the view to always use a certain loop, select it here.

 

Test View Cel

If you select a specific loop, the cel dropdown box is enabled. When Auto is selected, the test view will cycle through all cels in the selected loop as it moves about the screen. If you want the test view to show a single cel regardless of movement, select it here.

 

Cycle at Rest

When this option is selected, the test object will stop cycling when it is not moving. Uncheck this option to have the object cycle even when not moving.
