Menu Editor

The Menu Editor is used to create and edit AGI menus.

To open the Menu Editor, use the Menu Editor command on the Tools menu, or use the main toolbar button.

If a game is loaded, WinAGI will look for an existing menu in your game logics. If found, you will have the option to edit the game's menu, or open the default menu. If no game is loaded, the default menu will open automatically.

When looking for an existing menu structure to edit, WinAGI expects to find a series of set.menu commands and set.menu.item commands, ending with a submit.menu command. Typically, the menu structure is located in logic 0, and there is normally only one menu structure set up in a game. If you game has multiple menu structures, only the first one found will be editable in the Menu Editor.

Menu Editor Components

For a description of the various components of the Menu Editor window, click on the area of interest:

Toolbar

Several common actions for creating/editing menus are included as toolbar buttons for easy access. See the topics below for how to use these actions.

Menu Tree

The menu tree displays the current menu system in a hierarchical structure.

Click on a menu or menu item to select it. Or use the arrow keys. The up and down keys will select menu items in the current menu. The right and left arrows move from menu to menu.

Menu Preview

The menu preview area shows the menu as it will appear in your game.

The background will be the first picture in your game, if a game is loaded. If a game is not currently loaded, the background will be a black screen with a sample text display.

You can change the displayed background picture by double-clicking on the background or using the Change Background command on the Resource menu. Then choose the picture you would like to use as a background.

Property Window

The caption (and assigned controller, if a menu item is selected) are displayed in the property window. Double-click a property to edit it.

If the Controller property of a menu item is blank (such as with a newly added item), the menu item will display in red text in the treeview to remind you it needs to have a controller assigned.

Conditions

In most cases, menus and menu items are added without conditions. But sometimes you may want to only add certain menu items or menu items if certain conditions are met. Common examples are Debug menus, which are often only added if a debug flag is set, or a menu item to toggle the monitor, which is only added if the game is running on a PC with a CGA monitor.

You can use the Condition property for this purpose. For menus, if the Condition property is not blank, then the menu and all its items will be placed in an if block, using the Condition text as the argument of the if statement. All subsequent menus will also be in the if block until a menu with a blank or different Condition is found. For example, if the Condition for the Debug menu is "isset(f30)", when WinAGI saves the menu, it will look like this:

if (isset(f30))
    {
    set.menu("Debug");
    set.menu.item("Debug Help        ", cDebugHelp);
    set.menu.item("Ego Info   <Alt-E>", cShowEgo);
    set.menu.item("Pri Screen <Alt-P>", cShowPriority);
    set.menu.item("Memory     <Alt-M>", cShowMem);
    }

You can also set a Condition for individual menu items. All items with the same Condition will be placed in the same if block. For example, use a Condition of "machineType == PC && monitorType < 2" for the toggle monitor menu item to get this result:

...
set.menu.item("Sound On/Off <F2>", cToggleSound);
if (machineType == PC && monitorType < 2)
    {
    set.menu.item("Color/BW <Ctrl R>", cToggleMonitor);
    }
set.menu.item("Clock On/Off <F6>", cClock);
...

Inserting Menus and Menu Items

To insert a menu, select an existing menu, then use the Insert Menu command on the Edit menu (or use the toolbar button).

WinAGI will prevent you from adding too many menus in order to ensure the menus will fit on the AGI screen, which is limited to 40 columns of text.

To insert a menu item, select an existing menu item on the menu where you want another menu item, then use the Insert Menu Item command on the Edit menu (or use the toolbar button).

WinAGI limits the number of menu items in a menu to 22 so they will fit on the screen.

Deleting Menus and Menu Items

Use the Delete command (or toolbar button) to delete menu items and menus. If you delete a menu, all the items in that menu will also be deleted.

Editing Menus and Menu Items

Double-click on the menu or menu item you wish to edit, either on the menu tree, preview area, or on the property window. Make changes (in the property window), then press Enter. To cancel an edit, press Escape.

WinAGI will automatically add spaces (padding) to menu items so they are all the same length.

To create a menu separator, enter at least one minus sign (-) or leave the caption blank. WinAGI will automatically add enough minus signs to create the menu separator. When other items on this menu are updated, the separator will automatically shorten or lengthen.

Menu/Menu Item Order

To change the order of menu items use the Move Up and Move Down commands. You can also move menus left or right by using these commands.

Auto-align Shortcut Keys

WinAGI can automatically right adjust any shortcut key assignments by inserting extra spaces between the menu caption and the shortcut key. Shortcut keys must be included inside '>' and '<' brackets for AGI to automatically align them. You can toggle this feature by using the Right Align Hot Keys command on the Resource menu.

NOTE: Adding a shortcut key to a menu item DOES NOT create the actual shortcut in your game. You must assign shortcut keys in your logic code using the set.key command.

Saving Menus

If editing an existing game's menu, use the Update Source Logic command to insert the menu back into the logic it was extracted from. It will replace the existing menu (including all lines between the first set.menu command and the submit.menu command.

Use the Save As Default command to replace the current 'default_menu.txt' file with the current menu structure.

You can also use the Copy to Clipboard command on the Edit menu (or use the toolbar button) to place the current menu system on the clipboard. You can then paste it into a logic or text file and then add additional customizations.

Default Menu

A default menu is stored in the WinAGI program directory as a text file with the name 'default_menu.txt'. If a game is not loaded, or if you choose not to edit your game's menu structure, the default menu is loaded when the Menu Editor is opened.

The Restore to Default command will discard the current menu system in the editor, and load the default menu system.
