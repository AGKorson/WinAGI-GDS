Load Game Errors

If WinAGI is unable to open or import a game, it will display an error message indicating the reason the game failed to open. Additional information on each specific error is provided below.

 

 

507: UNHANDLED ERROR <number >: <errormsg >\n\n" +<stacktrace >

This error indicates an error occurred that WinAGI did not anticipate. It may indicate a possible bug in WinAGI. The error number and message, along with the stack information should provide information on how to address the problem, or you can leave a comment on the WinAGI github page.

 

 

527: directory '<directoryname >' has no wag files

This error indicates that WinAGI did not find a valid WinAGI game file in the specified directory. Check for the correct location of the game you want to open and try again.

 

 

533: Unable to copy template files to game directory due to error: <directoryname >

This error indicates WinAGI encountered an error when trying to copy the template files from the template directory to the new game directory. The error number and message should provide information you can use to address the problem.

 

 

534: Unable to open the newly created game due to error: <errinfo>

After creating a new game, WinAGI automatically opens it for editing. If an error was encountered while trying to open the new game, this error is raised. The error number and message should provide information you can use to address the problem.

 

 

535: Can't create new game in a directory that already has a wag file

This error is raised if the new game directory already has a WinAGI game file in it. You will need to choose a directory that does not have an existing game file as your new game directory.

 

 

536: Can't create new game in a directory that already has AGI game files

This error is raised if the new game directory already has a AGI game files (VOL and DIR files) in it. You will need to choose a directory that does not have an existing game as your new game directory.

 

 

538: <badversion> is not a valid AGI interpreter version.

There were only nineteen minor version numbers of AGI; fourteen in version 2 and five in version 3 released by Sierra. If the minor version for the target interpreter version found in this game's property file is not one of these versions, this error is raised. This can only happen if the file was edited manually outside of WinAGI or is corrupt. You will need to re-edit the file to change it to a supported version.

 

 

543: multiple WAG files found

This error is raised if a template game folder has more than one WinAGI game files in it. You will need to remove the extra game files so there is only one if you want to use this template.

 

 

544: template is missing resource directory

This error is raised if the template directory does not include a valid resource directory. Either edit the template game so it is correctly configured, or choose a different template.

 

 

545: Unable to move resource directory (<dirname >)

If WinAGI encounters an error when attempting to copy the resource directory from the template to a new game, this error is raised. The error number and message should provide information you can use to address the problem.

 

 

547: Error accessing wag file (<filename >)

This error indicates WinAGI encountered an error when trying to read data from the game property file. The error number and message should provide information you can use to address the problem.

 

 

548: Unsupported WinAGI version in template

Template games must be WinAGI version 3. This error indicates the template game you are trying to use is from an earlier version of WinAGI. If you want to use this game as a template, you will need to open it and upgrade it to WinAGI version 3.

 

 

549: Invalid WinAGI version in template

This error indicates the WinAGI version in the template's game property file is not the current version (3) or an earlier version that needs to be upgraded. It most likely indicates the game file is corrupted, or is not an actual WinAGI game file. You will need to address the problem with the game file if you want to use this template.
