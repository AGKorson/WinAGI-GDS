Working with AGI Games

The Game menu provides access to features for working with games, such as creating new games, opening or importing games, compiling, and modifying game level properties.

AGI games are comprised of a set of resources stored in game files that are located in a single directory. Selecting a directory for opening or creating a new game defines the location where WinAGI will read from and write to game files. Click on the links below for details on creating, opening or importing AGI Games.

Game and Resource Errors and Warnings Check

When a game is imported or opened, WinAGI does a full scan of the directory and volume files to confirm all resources are valid. If there are major errors, such as missing game files, or misconfigured game property file, the game will not load, and an error message will be displayed to explain the reason. If the game does load, minor resource errors will be displayed in the Warnings and Errors List. Issues that are not errors but should still be addressed are displayed as warnings. A complete list of errors and warnings that WinAGI can detect in resource files is provided in the Game and Resource Errors and Game and Resource Warnings topics.

Source code files are also scanned to confirm there are no major errors.
