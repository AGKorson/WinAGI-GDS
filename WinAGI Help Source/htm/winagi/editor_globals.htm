Global Defines

Global defines are defines that need to be available in all game logics (but not defines that represent reserved variables and flags, or resource IDs, which are managed separately).

WinAGI provides a table-based editor to easily manage your global defines. It will automatically update the include file 'globals.txt' and add it to all your logics.

If you do not want to use this feature, you can deselect the auto-include Global Defines option on the the Game Properties dialog.

 

Global Defines Editor

The Global Defines Editor contains a simple grid with three columns, for define names, define values and comments. The last line of the grid (with a dot) in the Row Header is the 'Insert New' row. The currently selected define is marked by a triangle in the Row Header.

To select an individual cell, click on it. To select the entire row, click on the Row Header for the desired row. You can select multiple rows by clicking and dragging on the Row Header. You can also select multiple rows by holding down the Shift key and using the up and down arrow keys.

Opening Global Define Files

To open a global defines file, use the Global Defines command on Tools menu or use the toolbar button.

If a game is loaded and your game is using the auto-include Global Defines feature, the toolbar button will always open or switch to your game's global defines file. Using the Tools menu command will open the game's global defines file if it is not open. If you want to open another global define file in addition to your game file (maybe you want to copy or cut and paste from another file), using the menu command a second time will prompt you to enter a file for opening.

If a game is not loaded, or if your game is not using the auto-include Global Defines feature, you will always be prompted for a file to open when using either the menu command or the toolbar button.

Global defines are stored in text files, with each define entry using the standard format for defines. For an open game, the file is always named 'globals.txt' and is located in the game's resource directory. Do not rename or move it; if WinAGI can't find it, a new, blank file will be created.

You can also open any other include file that has a list of defines in the editor. It provides an alternative to using a text editor and manually managing define entries.

 

Adding New Global Defines

To add a new define, do one of the following:

    • right-click on any row, and choose Insert Row command
    • double-click on the Name cell on the 'Insert New' row at the bottom
    • select the 'Insert New' row at bottom, and press Enter key

Enter the new name, press Enter or Tab to accept, then enter the new value. Press Enter to finish, or press Tab to enter a comment to go with the define.

Press Escape while entering the new name to cancel the new entry.

 

Comments

You can also add comments to your define values. These can be helpful to remind you of the purpose and usage of your various defines. Comments are optional.

You can hide the comment column by using the 'Hide Comments' command on the edit menu, or by dragging the column splitter between value and comment to the far right. To restore the comment column use the 'Show Comments' command.

The comments column is displayed by default when the defines file is opened. If you don't plan on using comments, you can change the default to openn with the comments column hidden by changing the Show Comment Column property in the Game Settings dialog.

Because comments are typically much longer than just a name or value, you can hover the cursor over a selected comment and WinAGI will show a tooltip window with the entire text of the comment.

Add from File

You can also add the entire contents of an existing defines file to the current editor. Use the Add from File command on the Resources menu. WinAGI will parse the file, and add any defines that it finds. If a define is found that already exists in your global file, you will be given an opportunity to ignore it, or to add it (which will overwrite the existing define). If any items are unable to be added for any reason, you will get a status message after the file is added.

 

Editing Global Defines

To edit an existing define name, value or comment, double-click its cell, then make your changes. Define names must not include invalid characters. Define values must be a valid number, argument marker, or a literal string.

If a define name matches a reserved name, or a resource ID, you will be asked to confirm that you want to override the reserved name. If you do override it, the define name is displayed with dark orange text over a light yellow background to make it easy to see that it is an override.

 

Cut/Copy/Delete/Paste

Use the edit menu for cutting, copying, deleting or pasting defines. Use the Clear List command to delete all global defines at once.

When pasting, WinAGI automatically determines if the clipboard contains defines cut or copied from a global editor, or if it contains regular text. If the clipboard is regular text (for example, if cut from a logic) the Paste option will only be enabled if the text consists of lines of properly formatted define entries (i.e. '#define name val [ comment') The Paste option will not be available if the clipboard format is not correct.

Invalid Defines

When editing, WinAGI will prevent you from intentionally creating an invalid define entry. However, when pasting, a define value may get added that has error conditions such as invalid characters or invalid argument marker values. If so, the value will be highlighted in red until you correct the issue.

Valid controller argument values are c0 through c49 (c39 in some early versions. Similarly, valid string argument values are s0 through s23 (s11 for a few specific versions). If a controller or string argument value in the list uses a value outside the valid range, it will be highlighted with dark orange text on a yellow background.

 

Sorting

As you add defines to your list you may want to change the order they are in to help with organization. Defines can be easily moved around the list by dragging them - select the row or rows you want to move, then click and hold on the Row Header of the selected row. Drag it up or down until it is in the location where you want it to stay then release.

You can also adjust the order by cutting a row or rows and then pasting them in the desired location.

You can sort the list of defines by double-clicking on one of the Column Headers. Click it again to reverse the sort.

Find in Logics

The Find in Logics command on the Edit menu allows you to search for a global define name in all of your logic resources. To use this feature, select the define name you wish to search for, then choose the Find in Logics command from the Edit menu.

Undo

WinAGI also includes an undo feature for global edits. Undo is accessible from the Edit menu, or by pressing CTRL+Z. A very handy feature if you change your mind about the edits and changes you've made.

Automatic Updates

When you save an in-game global defines file, you can choose to have WinAGI automatically update all logic source files in your game to reflect your new list of global defines. For changes to existing defines, WinAGI will replace all instances of the old name with the new name. For new defines with values other than text-strings or numbers, WinAGI will also replace all instances of the value with its new name.
