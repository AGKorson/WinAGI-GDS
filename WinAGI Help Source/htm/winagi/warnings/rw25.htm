Resource Load Warning RW25

Upper case characters detected in one or more words

When parsing player input, AGI converts all characters in the input line to lower case before checking against the words in WORDS.TOK. Therefore, it is impossible to match input to a word that has upper case characters in it.

The format that AGI uses to store words in the file do not prevent the inclusion of capital letters. Sierra's tool for compiling WORDS.TOK files (WORDS.EXE) did enforce lower case, but not all fan created tools do.

The affected words do not cause any errors, but they should be removed, as they serve no purpose and waste space in the file.

WinAGI will not allow you to compile a WORDS.TOK file that has upper case characters in it. You will need to delete the offending words first.
