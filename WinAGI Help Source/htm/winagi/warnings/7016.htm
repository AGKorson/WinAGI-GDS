Logic Compiler Warning 7016

Character '\x1a' is treated as a token separator when token length > 0

Only applicable when using Sierra syntax.

Sierra's logic compiler CG.EXE normally treats character '\x1a' as an end-of-file (EOF) marker. But if the character is encountered after the compiler has started building a token, it is treated as a separator and does NOT end the file.
