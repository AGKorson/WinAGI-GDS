Logic Compiler Warning 5081

Test command 'return.false()' is not compatible with AGIStudio

There is an undocumented test command in AGI with a byte code value of zero. The function (called return.false), does nothing and returns a value of false. It has no practical value, but WinAGI will compile it, as it will run without raising a runtime error. However, AGI Studio does not recognize it as a valid command. If you try to use this source code in AGI Studio, return.false will generate an unknown command compiler error. Also, if you decompile a logic that uses return.false, AGI Studio will identify it as a corrupt logic. It's best if you don't use AGI Studio anyway, since WinAGI is so vastly superior in all respects.
