Logic Compiler Warning 5098

Negative numbers are converted to unsigned 2s complement value

In most cases, AGI treats all byte values as unsigned numbers. The reposition command however, treats its argument values as signed values, ranging from -128 to +127. To make it easier to write source code, WinAGI will take negative numbers and convert them automatically to the appropriate unsigned value. This warning is displayed as a reminder that a negative number was used in source code. If not being used in conjunction with the reposition command, you should avoid use of negative numbers.
