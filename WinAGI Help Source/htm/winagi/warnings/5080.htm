Logic Compiler Warning 5080

Setting COL to >39 in the get.string command will cause the PROMPT to appear in unexpected location

Valid range for COL argument in the get.string is 0 - 39. If you pass a column value greater than 39, the first character of the prompt string will be placed slightly below the desired ROW value, with a vertical offset in pixels equal to INT(COL/40), in a column equal to the COL value MOD 40. The rest of the prompt will be left adjusted, one row below the desired ROW value. You should edit your logic to use a valid COL value.
