Logic Compiler Warning 5037

Attempting to set a view that does not exist will cause a runtime error

Using the set.view command on a view that does not exists will raise trappable error #3. It is possible to recover, but that depends on the game programmer writing good error handling code. You should check your code to make sure you are referencing a valid view resource.

NOTE: WinAGI can only detect attempts to set a non-existent view; it can't detect cases where your code attempts to set a valid view that hasn't been loaded. That will also raise trappable error #3.
