Logic Source Decompiler Warning DW12

return command is missing at end of logic

Normally all logics should end with a return command. This warning indicates that the decompiled logic does not. This isn't necessarily a problem if the last command is quit (several Sierra games did this) or if the logic never gets to the end (i.e. there are branches earlier in the logic that always exit early). Regardless, it is bad coding practice to leave out the ending return command.

When recompiling, WinAGI will enforce the requirement to include the ending return command by raising an error (or by adding it automatically if using Sierra Syntax).
