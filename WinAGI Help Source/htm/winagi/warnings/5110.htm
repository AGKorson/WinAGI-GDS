Logic Compiler Warning 5110

Cel width less than 3 will not draw control box correctly

There is a bug in the code that the add.to.pic command uses to draw the control box when the margin property is three or less. If the width of the cel being added is one pixel wide, the control line will draw 255 pixels (which will obviously extend over several lines). If the width is exactly two, AGI will try to draw a line of 64K pixels, which overflows AGI's memory and crashes AGI.

This warning is raised if you attempt to add a cel with a width of 2 or less and also include a control box. You should either use a cel with a width of at least 3, or pass a margin value greater than 3 (which will mean no control box will be drawn).
