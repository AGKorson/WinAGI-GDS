Logic Compiler Warning 7020

Tokens beginning with '@', '&' or '|' are considered symbols, not identifiers

Only applicable when using Sierra syntax.

Sierra's logic compiler CG.EXE has a bug which treats all tokens beginning with '@', '&' or '|' as symbols, not identifiers. This means those tokens could never be used as a define name (only identifiers can be a name) or any other value because the compiler will never recognize them as a valid symbol. You will need to adjust the token to use a different starting character.

%define @abc  99 [ NOT OK; name can't be a symbol
%define abc   @v99 [ compiles, but of no value
...
