Logic Source Decompiler Warning DW11

quit command at resource index <pos> coded with no argument

This warning is raised when WinAGI determines that a quit command for a version that requires an argument is inside an if-block or else-block, but was compiled without including the extra command.

If WinAGI detects a mismatched block end that is off by exactly one byte, and there is a quit command inside the block, and the target interpreter version requires an argument for the quit command, the decompiler assumes the mismatch is caused by the quit command missing its required argument.

There are several known cases of this exact problem happening in some Sierra games. The cause appears to be logics that were compiled for a version that has no arguments for the quit command being added to a game using a newer interpreter without recompiling the logic.

WinAGI will add the missing argument to the quit command, but you should recompile the affected logic so it aligns with its target interpreter version.
