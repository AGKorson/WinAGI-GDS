Logic Compiler Warning 5034

'<token>' is overriding a global define

You created a local define with the same name as a global define. You can do this without causing a compiler error, but it is not recommended. WinAGI uses the following hierarchy when attempting to resolve a define name:

    1. Local defines
    2. Global defines
    3. ResourceIDs
    4. Reserved Names

If you use the same name for a local define, it will override the global define. Unless you meant to do this, it may cause unexpected results.
