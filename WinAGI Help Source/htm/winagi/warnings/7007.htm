Logic Compiler Warning 7007

Non-numeric characters in this token will be ignored (<tokenvalue>)

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, tokens that start with a number digit will include all characters, even non-numeric, until a separator character is found. The value of the token is determined by the numeric digits up until the first non-numeric character. All other characters in the token are ignored.

%define anum 123abc456   [ OK, same as '%define anum 123'
v99 +=0xa; [ same as 'v99 += 0;' it does not convert '0xa' to '10'
...

Usually, this warning indicates a syntax error. You should check your source code carefully, and edit as necessary to correct the condition.
