Logic Compiler Warning 7017

Unexpected 'else' keyword will be treated as 'goto'

Only applicable when using Sierra syntax.

Unlike most modern compilers, Sierra's logic compiler CG.EXE does not restrict the usage of the else keyword to if ... else blocks. At the byte code level, the else keyword is identical to the goto keyword. When CG.EXE encounters an else statement that does not immediately follow the closing block marker ('}') of an if block, it treats the else command as a goto, with the jump position set to the end of the else block.

if (v99 == 1) {
    v99 = 2;
}
else { [ normal expected use of 'else'
    v99 = 3;
}
else { [ abnormal use of 'else'; allowed in Sierra syntax
    v99 = 4;
}

[ equivalent code:
if (v99 == 1) {
    v99 = 2;
}
else {
    v99 = 3;
}
goto _label1
    v99 = 4;
:_label1

To maintain compatibility with CG.EXE, WinAGI allows the same behavior. However, there is no practical value, and it is a very bad coding practice. You should review your code and replace the non-standard else statement with an appropriate goto statement and label.
