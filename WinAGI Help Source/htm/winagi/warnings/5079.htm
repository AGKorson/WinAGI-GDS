Logic Compiler Warning 5079

Setting PRIORITY to 1, 2 or 3, or > 15 in add.to.pic command may cause unexpected results

AGI priority values should be in the range 4 to 15. Using a priority value outside this range may have unexpected results.

For the add.to.pic command, a value of zero is allowed for PRIORITY, because that tells AGI to set the priority based on the Y position. Other values outside the range 4 to 15 will exhibit the same behaviors described in Warning 5050. Setting PRIORITY to < 4 or > 15 may cause unexpected results. See also the the Object Priority topic for a detailed discussion on managing priorities of objects on screen.
