Logic Compiler Warning 7015

Character '\x1a' gets changed to '\xff' when added to literal string

Only applicable when using Sierra syntax.

Sierra's logic compiler CG.EXE normally treats character '\x1a' as an end-of-file (EOF) marker. But if the character is inside a literal string, as in a %message declaration, it gets converted to '\xff', and does NOT end the file.

To maintain compatibility with CG.EXE, WinAGI will also replace '\x1a' with '\xff'.
