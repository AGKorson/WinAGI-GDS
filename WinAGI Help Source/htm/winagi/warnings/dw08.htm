Logic Source Decompiler Warning DW08

Block end at resource index <pos> is outside nested block (jump location <jumppos>)

This warning is raised when an if-block inside another if-block has an end-if location that is past the end-if location of the first if-block. This is easier to see with an example:

[ if decompiled literally, the code would be this:
if (TestA) {
    command1;
    if (TestB) {
        command2;
}  [ end-if TestA
command3;
        } [ end-if TestB
command4;
[ the syntax rules for AGI source code does not allow you to put 
[ the end-if for TestB past the end-if for TestA

Even though the logic byte code can't be decompiled into readable source code, it will actually work when AGI processes the logic. Since it can't be decompiled literally, WinAGI will add an else and goto statement that results in valid source code that does exactly the same thing:

[ WinAGI creates an equivalent else-goto construct that
[ exactly mimics the existing byte code effect
if (TestA) {
    command1;
    if (TestB) {
        command2;
        } else {
        goto(Label1);
    }
}
command3;
:Label1
command4;

You should confirm that the VOL file containing this logic is not corrupt. Regardless, you will need to carefully examine the decompiled source code, and adjust the code as necessary before recompiling.
