Logic Compiler Warning 7019

Using ';' as an argument separator is allowed, but not preferred

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, WinAGI treats both ',' and ';' characters as argument separators in a command statement.

addn(v99; 1); [ same as 'addn(v99, 1);'
if (posn(ego; 0; 0; 50; 50))  [ same as 'posn(ego, 0, 0, 50, 50)'
...

Although this does not raise an error, it is not good coding practice, and you should replace the ';' character with a ',' character.
