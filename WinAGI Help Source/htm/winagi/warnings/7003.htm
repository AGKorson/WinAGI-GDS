Logic Compiler Warning 7003

Numerical values outside byte range will be converted to a byte value

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, all numbers that are outside byte range (0 - 255) will be converted to a byte value. The conversion is applied by returning only the eight lowest bits of the number (i.e. BYTEVAL = NUMBER ^ 255).

Since there aren't many situations where using numbers larger than a byte value is useful, this warning most likely indicates a syntax error or extra unintended digit. You should check your code carefully to be sure the number in your source code is the number you intended to use.
