Logic Source Decompiler Warning DW09

Command '<cmdname>' at resource index <pos> is not valid for selected interpreter version (<versionnum>)

This warning is raised if WinAGI detects a command in the byte code that is not not supported in the game's target interpreter version. The number of commands varied between versions, so some commands that are valid in later versions will cause an error if encountered in an earlier interpreter version that doesn't support it.

A likely cause for this is having the wrong target interpreter version for the game. If the target interpreter is correctly set, it may be that Sierra accidentally mixed compiled logics from different game versions that were compiled for different interpreters.

You should confirm that the game's target interpreter version is correctly set. If it is, the invalid command is probably bad coding in the original source code. Regardless, you will need to carefully examine the decompiled source code, and adjust the code as necessary before recompiling.
