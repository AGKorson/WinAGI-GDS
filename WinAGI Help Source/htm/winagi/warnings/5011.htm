Logic Compiler Warning 5011

Using clear.line command with bottom < top and/or either value > 24 will have unpredictable results

The clear.line command does not validate the TOP or BOTTOM values passed when it tries to clear lines on the screen. If either value exceeds the display size (24 lines) or if the bottom value is less than the top value, the results in AGI will be unpredictable, and may crash AGI.
