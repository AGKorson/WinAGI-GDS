Logic Compiler Warning 7014

Character '\x1a' forces end of file; no text after this character will be processed

Only applicable when using Sierra syntax.

In MSDOS, the character '\x1a' was often used as an end-of-file (EOF) marker. Sierra's logic compiler CG.EXE also treats the character as an EOF marker when getting a token from source code, so to maintain compatibility, WinAGI will also stop processing source code if this character is encountered.

[ '→' is the MSDOS glyph for character '\x1a'
[ it may display differently depending on the editor's selected font

v99 = 0;
→ [ processing ends here
v99 = 1;  [ this line is never reached
...

There is not much practical value to use this feature. Unless you have a good reason to use it, you should remove this character from your source code.
