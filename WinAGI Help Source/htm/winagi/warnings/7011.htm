Logic Compiler Warning 7011

Null label name is allowed, but unusable

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, WinAGI allows you to use a null value for a label name. The null label has no usefulness, because it is impossible to write a goto command to use it.

Allowing the null label is likely a bug in CG.EXE. A null label is created if the character immediately following the ':' character is a separator (space, tab, new line), number or symbol.

v99 += 1;
:          [ creates a null label, but you can't use it
v99 += 2;
...

If this warning is raised, it most likely represents a typo in your code. You should review and update your code as necessary to eliminate null labels.
