Logic Compiler Warning 7010

'<token>' is an object being used as a number

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, WinAGI allows a token defined as an object to be used as a number argument in commands but only when using alternate syntax. This is mostly useful when testing reserved variable v4 (object touching edge). But it will work for all alternate syntax commands, both action and test.

%var    v4    4
%object obj   1
if (v4 == obj) {  [ OK, same as 'if (v4 == 1) {'
if (equaln(v4, obj) { [ NOT OK
v4 += obj;  [ OK, same as 'v4 += 1'
addn(v4, obj); [ NOT OK
...

Unless you are intentionally using the object token as a number, you should check your source code and edit it as needed to use the correct token or number in the affected command.
