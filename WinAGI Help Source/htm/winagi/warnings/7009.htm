Logic Compiler Warning 7009

return command not needed at end of logic source

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, WinAGI automatically adds an ending return command byte code (0x00) to the end of a logic after compiling. If you include a return command at the end of your source code the output code for the logic will contain an extra return byte code.

The extra byte doesn't cause a problem. But it does waste space and is not necessary. You should remove the extra return command from your source code.
