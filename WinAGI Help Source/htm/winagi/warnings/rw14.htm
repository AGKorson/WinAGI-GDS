Resource Load Warning RW14

Loop <number> in view <number> has invalid mirror assignment

Loop mirroring in AGI is accomplished by having two loops use the same cel data, with each shared cel having a special bit set so AGI knows to flip the cel when the loop changes. To be a valid mirror pair, a loop must:

    • have loop index value less than eight;
    • share cel data with exactly one other loop;
    • have each cel include the mirror bit marker;

In addition, views from version 2.089 do not support mirroring. If a view in a version 2.089 is detected with mirroring, this warning is also shown.

This warning indicates WinAGI found a loop that is not a properly formatted mirror loop. Possible causes include the mirror bit marker is not set in all cels, more than two loops sharing the same loop data, loops with an index value larger than seven.

These anomalies usually don't cause a problem in AGI. The AGI engine would usually just skip over them so the view's creators didn't even know the view was not formatted correctly. There are a few Sierra games that are known to have views with some of these anomalies.

WinAGI will treat the affected loops as though they are not mirrored. Opening and then saving the view in WinAGI will update the view with correct formatting and will clear the warning.
