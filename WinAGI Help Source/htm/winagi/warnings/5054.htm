Logic Compiler Warning 5054

Using random command with lower bound > upper bound will have unexpected results

AGI expects the lower bound to be less than or equal to the upper bound. If it is not, AGI will either lock up (if upper bound is exactly one less than lower bound) or will provide random number in range 0-255 regardless of pass values. See the random command for more details on how random numbers are generated in AGI.
