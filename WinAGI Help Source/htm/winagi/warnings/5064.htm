Logic Compiler Warning 5064

Using add.to.pic command with an invalid view will raise a trappable error

Attempting to use the add.to.pic command with a view that does not exist will raise trappable error #3. It is possible to recover, but that depends on the game programmer writing good error handling code.

NOTE: WinAGI can only detect attempts to access a non-existent view; it can't detect cases where your code attempts to access a valid view that hasn't been loaded. That will also raise trappable error #3.
