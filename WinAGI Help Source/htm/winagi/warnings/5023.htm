Logic Compiler Warning 5023

Object position is off screen (X > 159 and/or Y > 167)

Positioning an object with coordinates that are off screen will not cause a runtime error, but when AGI validates the position, it first moves it to the edge (i.e. x=159 or y=167) and uses the shuffle function to determine a position for the object that fits on the screen and doesn't overlap other objects or control lines. This can result in very unexpected results, due to the spiral method used by shuffle function.

NOTE: this warning also applies to objects whose X position plus width or Y position plus height exceed the values of 159 or 167 respectively. WinAGI cannot detect this at compile time, but be aware that unexpected positioning of objects can be due to this condition.
