Logic Compiler Warning 7006

Consecutive NOT operators (!) will toggle the NOT staus

Only applicable when using Sierra syntax.

To maintain compatibility with Sierra's logic compiler CG.EXE, WinAGI allows multiple consecutive NOT operators in if statements.

CG.EXE compiles the extra NOT operators without error. For example:

SOURCE                 BYTE CODE
if (isset(f85))        FF 07 55 FF
if (!isset(f85))       FF FD 07 55 FF
if (!!isset(f85))      FF FD FD 07 55 FF
if (!!!isset(f85))     FF FD FD FD 07 55 FF

Each extra '0xFD' byte will toggle the NOT status; meaning two NOTs will result in the test passing if it is TRUE, three NOTs will pass if the test is FALSE, etc.

if (!!isset(f85))  [ same as 'if (isset(f85))'

The extra NOT operators are likely a syntax error, which may need to be fixed for your code to operate as you expect.
