Logic Source Decompiler Warning DW20

Unable to create gamedefs.h file (error: <message>)

Only applicable when using Sierra syntax.

Sierra's original code writers always used a separate file, usually called 'gamedefs' or 'gamedefs.h' to declare variables, flags and other defines that would be used across all logics in the game.

When decompiling all logics during a game import, WinAGI will do the same thing, creating a 'gamedefs' file with defines that are global in scope.

This warning is raised if WinAGI encounters an error while trying to create the 'gamedefs' file. You should use the information from the error message to determine the problem, correct it and attempt to re-import the game. Otherwise, you will need to manually create the 'gamedefs' file.
