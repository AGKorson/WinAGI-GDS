Logic Source Decompiler Warning DW10

<resource> in '<cmdname>' command at resource index <pos> does not exist

During decompilation, WinAGI will check that any resources referenced in decompiled commands are present in the game. This warning is raised when a command references a resource that does not exist. Attempting to reference an invalid resource in a command will cause errors, and crash AGI.

This could be caused by corrupt VOL or DIR files that don't point to the desired resource. It may also be errors in the original Sierra source code. Regardless, you will need to carefully examine the decompiled source code, and adjust the code as necessary and/or reinsert the missing resource before recompiling.
