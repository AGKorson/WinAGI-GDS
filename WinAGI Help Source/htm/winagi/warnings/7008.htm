Logic Compiler Warning 7008

Using '!=' when a test is already negated will toggle the NOT status

Only applicable when using Sierra syntax.

Sierra syntax allows the NOT operator (!) to be placed in front of alternate syntax test commands. In Sierra's logic compiler CG.EXE, using the binary operator '!=' when a test is already negated just adds a second 'NOT byte' (0xFD). For example:

SOURCE                 BYTE CODE
if (v85 == 2)          FF 01 55 02 FF
if (!v85 == 2)         FF FD 01 55 02 FF
if (v85 != 2)          FF FD 01 55 02 FF
if (!v85 != 2)         FF FD FD 01 55 02 FF

When AGI processes the NOT byte value, it toggles an internal flag. Which means two consecutive NOT bytes will cancel each other out.

v85 = 2;
if (!v85 != 2)   [ this will return TRUE

You should edit your source code to use no more than one NOT byte when it compiles.
