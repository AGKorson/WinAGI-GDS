Logic Compiler Warning 5088

<command> is incompatible with MSDOS AGI interpreter

WinAGI is intended for creating AGI games that run on the MSDOS platform of AGI. Several AGI commands that were made available in later versions of AGI were not intended to run on the MSDOS platform. The MSDOS interpreter calls a null function when one of these commands (discard.sound, fence.mouse, and mouse.posn) is encountered, but it does not process the command's arguments. This will cause an error, because AGI will try to process the argument as the next command byte. This will almost certainly crash AGI.

You should remove these commands, unless you intend to run your game with a non-MSDOS AGI interpreter. Just keep in mind that WinAGI isn't designed to support that.
