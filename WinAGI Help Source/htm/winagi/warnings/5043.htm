Logic Compiler Warning 5043

Setting the horizon greater than 166 will crash AGI

The acceptable range for the horizon is 0 - 166. Passing a value greater than 166 will cause AGI to lock up as soon as it tries to draw any object to the screen, because the shuffle function that AGI uses to determine the final position of every object will go into an infinite loop. AGI doesn't actually crash, but because of the infinite loop, the only thing you can do is to force AGI to quit.

Note that AGI automatically resets the horizon to 36 every time a new room is loaded. See the Horizon topic for for information.
