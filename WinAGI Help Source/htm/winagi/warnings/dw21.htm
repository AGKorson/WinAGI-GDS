Logic Source Decompiler Warning DW21

Consecutive 'NOT' operators detected at resource index <pos>

Only applicable when using Sierra syntax.

AGI actually allows consecutive NOT byte codes ('\xfd').

BYTE CODE               ORIGINAL SOURCE         DECOMPILED CODE
FF FD FD 07 55 FF       if (!!isset(f85))       if (isset(f85))
FF FD FD FD 07 55 FF    if (!!!isset(f85))      if (!isset(f85))

Each extra '\xfd' byte will toggle the NOT status; meaning two NOTs will result in the test passing if it is TRUE, three NOTs will pass if the test is FALSE, etc.

This warning indicates WinAGI detected consecutive NOT bytes when decompiling a logic. The decompiled source will match the net effect of the extra NOT bytes.

Most likely the original source code had a syntax error, but it is possible the logic is corrupt. You should examine the decompiled source carefully to confirm it is as expected.
