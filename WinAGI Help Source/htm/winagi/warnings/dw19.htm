Logic Source Decompiler Warning DW19

Unable to create separate message file for <logic>; messages are defined in the source (error: <message>)

Only applicable when using Sierra syntax.

Sierra's original code writers almost always used a separate file to store messages and then used the %include preprocessor statement to add the file to the main logic source code. When decompiling WinAGI will attempt to use the same format.

This warning is raised when WinAGI encounters an error while trying to create the separate message file. The messages will be added to the main source file instead. You will have to manually create a message include file and move the messages if you want separate files.
