Resource Load Warning RW12

Sound <number> has one or more tracks missing end of track marker

All tracks in a PCJr sound resource must end with a note length value of 0xffff (two 0xff bytes in a row). That is how AGI knows to stop reading notes from the resource for the given track. If a track does not have the ending track marker, AGI would continue reading data from the heap outside of the sound, leading to unexpected results.

When WinAGI detects a missing end of track marker, it automatically adds it when opening the sound for editing. Saving the sound will then clear this warning.
