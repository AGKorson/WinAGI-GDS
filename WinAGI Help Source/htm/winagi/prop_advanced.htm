Advanced Game Properties

Alternate Code Page Support

AGI has very limited support for extended characters, and does not directly support the use of non-English language keyboard layouts and character code pages in other languages. Although AGI doesn't directly support code pages other than 437, it is possible to run AGI in MSDOS with other code pages. This only affects the glyphs used for extended characters. WinAGI includes a code page property that allows you to use a non-English code page during game development that will show extended characters in your selected language in all resource editors. The Character Map will adjust automatically to show the correct extended character glyphs when the code page is changed.

Unless you intend to create a game that use extended characters from code pages other than the default (which is not easy), there is no reason to change the code page setting.

You can get more information on this topic, as well as ask additional questions by visiting the AGI Programming Techniques online forum.

Original Sierra Logic Syntax Support

When compiling logics, WinAGI by default uses the established 'fan syntax', originally developed around the time the first AGI fan tools were created. If you want to switch to the same syntax that Sierra used, or if you are interested in compiling original Sierra source code, you can check this box to use original Sierra syntax for compiling logics.

When importing with Sierra syntax, WinAGI can work with either VOL and DIR files, or individual resources that are stored in Sierra's standard folder structure (with logic source files named like "RM0.CG" in a 'SRC' sub directory, pictures named like "PIC.0" in a 'PIC' directory, sounds named like "SND.0" in a 'SND' directory and views named like "VIEW.0" in a 'VIEW' directory.) This means you can import original Sierra source files in their original state, as well as any fully compiled game with decompiled source in Sierra syntax.

 

Power Pack Support

Although there is no checkbox on the Game Properties dialog, WinAGI does include a Power Pack support option. This option provides additional capabilities for WinAGI's resource editors that are specifically designed to assist with programming games that use the Power Pack features. See the Power Pack topic for more information.
