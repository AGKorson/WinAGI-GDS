Logic Settings

Preview Font

The logic preview window is a simple text box. It does not use syntax highlighting. Change the font name and font size of the preview window here. The background for the preview window can't be changed; WinAGI will automatically use a light green background to indicate a logic that is compiled, and light red to indicate a logic which has source code changes that have not been compiled.

Editor Font

Select the font name and size for the logic editor. The Logic Editor font name and size are also used in the Text Editor, WORDS.TOK editor, OBJECT editor, Global Defines Editor and Layout Editor.

 

Enable Syntax Highlighting

Logic Syntax highlighting allows you to specify different colors and attributes for different syntax elements. Syntax highlighting for logics is enabled by default, but it can be disabled by clearing this check box. When disabled, all text in logic editors will use the 'Normal' settings for foreground color and attributes.

Syntax Highlighting Styles

The color, bold style and italics style can be set individually for each syntax element. Select a syntax element in the list, then use the color button and check boxes to select a new color, set bold style and/or set italics style. The eleven syntax elements are:

Normal: symbols, brackets, etc

Comment: comments are ignored by the compiler

Strings: literal text strings offset by double quote marks

Keywords: if, else, goto, #define, #message, #include

Test Commands: AGI test commands

Action Commands: AGI action commands

Invalid Commands: commands that are not valid based on the game's current target interpreter version

Numbers: numbers, as in 1, 2, 3, etc.

Argument Markers: argument markers such as v99, f100, o2, etc.

Define Names: identifiers that are declared as define names

Background: to set the background color of the editor; bold and italics properties do not apply

Use the preview button to show a sample of text to see how your selected styles will look in the logic editor.

 

Highlight Syntax in Text Files

Sets the default behavior for the Text Editor. When checked, syntax highlighting is enabled when Text Editors are first opened.

 

Default Logic Source File Extension

The extension for logic source files is added to new and imported source files, as well as decompiled logics. This can be changed for each game individually on the Game Properties dialog. If you always want to use a different extension for all your games, you can set that default here.

 

Enable Code Snippets

When this option is enabled, you can use code snippets to easily insert commonly used code fragments in logics.

 

Show Argument Tooltips

This option enables or disables the Argument Tooltips feature for the Logic Editor.

 

Show Define Tooltips

This option enables or disables the display of define tips when hovering over defines in the Logic Editor.

 

Show Document Map

When enabled, Logic Editors and Text Editors will show the document map panel by default.

 

Show Line Numbers

When enabled, Logic Editors and Text Editors will show the line number column by default.

 

Maximize Editor on Open

When enabled, the Logic Editor will automatically resize to fill the WinAGI program window.

 

Tab Width

The tab width property determines the number of spaces inserted when the Tab key is pressed in the Logic Editor and Text Editor.
