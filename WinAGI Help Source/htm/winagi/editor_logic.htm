Logic Editor

The Logic Editor allows you to create and edit source code for logic resources.

 

 

Line Numbers

The editor includes the option to display line numbers in a column to the left of the editing area. The first line is numbered zero. The line number column can be toggled on or off by using the Edit menu command or by pressing CTRL+SHIFT+N.

 

Document Map

The document map is a miniature display of the text to the right of the editing area that shows the relative position of the editor window to the entire text. You can click anywhere on the map to immediately jump to that location. You can also drag the map like a scrollbar to quickly navigate through the text. The document map can be toggled on or off by using the Edit menu command or by pressing CTRL+SHIFT+M.

 

Navigation

   Use the naigation buttons to move the insertion point to previous locations or to return to a more recent location from a previous location.

 

Toolbar

The toolbar contains button shortcuts for all basic editing commands, as well as buttons for common commands applicable to logic source code editing.

 

Editing Tools

           The logic editor is a typical text based editor. Basic text editing commands (cut, copy, paste, delete) are available. You can also drag and drop text into and out of the text area. The Logic Editor supports multi-level undo and redo.

Find/Find Next/Replace

  Use the Find command on the Edit menu (or the toolbar button) to search for text.

The find window will remain on top of all WinAGI editors until you dismiss it by clicking Cancel.

Enter the search string in the 'Find what:' text box. WinAGI does not support wild card searches.

The search scope can be one of the following: the current logic, all logics currently open for editing, or all logics in your game.

    • Current Logic: searches only the currently active open logic editor
    • All Open Files: searches all open logic editors and text editors
    • All Logics in Game: only available if a game is open; searches all logics in a game whether they are open for editing or not; any open text editors are skipped

The direction of the search can be set to:

    • Down: searches until end of file is reached
    • Up: searches in reverse, until beginning of file is reached
    • All: searches forward, until end of file is reached, loops back to the beginning of the search scope, then continues until starting point of search is reached

Indent/Outdent

    Using indentation to manage blocks of code is essential to writing good, readable source code. To easily change the indentation level to multiple lines, use the Block Indent and Block Outdent functions. In addition to the toolbar buttons, you can also use shortcuts (TAB key for block indent, SHIFT+TAB for block outdent).

Commenting and Uncommenting Code Blocks

    You can easily comment out multiple lines of code by using the Block Comment action. WinAGI will insert a comment marker '[' in front of all lines currently selected in the code window. Use the Uncomment Block action to remove any comment markers from the selected lines.

Extended Character Support

  Although it was extremely rare to see in Sierra games, AGI does provide limited support for use of extended character codes. Use the Character Map Tool (on the toolbar or Edit menu) or press CTRL+INS if you want to insert an extended character into a logic resource.

Message Cleanup Tool

  AGI uses the message data type for displaying text messages on screen. As described in the syntax rules, messages can be assigned a specific number using the #message precompiler command, but this is not required. In commands that take messages as arguments, you can refer to the message by number ('m1', 'm2', etc.) or with a literal string.

If you have entered a large amount of source code and used literal strings to define messages, you can use the Message Cleanup Tool command to re-build the defined message list for your logic.

The Message Cleanup Tool will scan your code for literal message strings, assign them a number, and create a #message entry for you automatically, and use that number in the compiled resource.

You also have the option to have any existing defined messages removed if they are not being used in the logic. NOTE: if your source code references messages indirectly, via the "%m" format code, do NOT use the 'remove unused messages option, as you may unintentionally delete messages that are used within other messages!

Compiling Logics

  Use the Compile command (or the toolbar button) to compile the source code of the logic being edited.

See the Compiling Logics topic for detailed information on compiling logics.

Syntax Highlighting

WinAGI provides automatic syntax highlighting commonly found in code editors for source code in the Logic Editor. You can adjust the colors used for the different language components on the Logics tab of the Settings window.

Argument Tooltips

The Argument Tooltips feature displays the syntax for AGI commands. When you enter the name of a valid command, the syntax is shown immediately below the current line, with the first argument in bold. After you enter the first argument value, the second argument appears in bold, and so on.

You can enable or disable this feature on the Logics tab of the Settings window.

Inserting Commands

To get a display of all available AGI commands, you can use the List Commands feature (from the context menu, or keyboard shortcut CTRL+SHIFT+J). You can then select a command from the list to add it automatically.

Inserting Defines

To make it easier to insert defines when coding, use the List Defines command. You can use the context menu (right click on the logic editor, and select the command from the pop-up context menu) or use keyboard shortcut (CTRL+J).

Hovering the mouse pointer over the listbox will show what the define represents. Each define is represented by an icon that includes a red dot for reserved defines, a blue dot for global defines, and no dot for local defines:

Define Tooltips

If you hover the mouse cursor over a reserved define, global define, or local define name in logic source code, WinAGI will display a tool tip that shows the value of the define name. This feature can be disabled on the Logics tab of the Settings window.

Resource Quick Edit

WinAGI provides a handy shortcut to open other resources referenced in source code. You can select the 'Open <resource> for Editing' option from the Edit menu (or press CTRL+SHIFT+E) when the cursor is on any Resource ID. You can also hold the CTRL key and double-click a resource ID to open it for editing.

This feature also works on #include files, as long as the file name points to a valid file.

Code Snippets

If you have blocks of text that you find yourself typing over and over, you can create a code snippet as a shortcut to enter the text into your logic. Use the Snippet Manager dialog to create, edit or delete your snippets.

An alternate way to create a snippet is to select the text you want to reuse and use the Create Snippet menu item or keyboard shortcut (CTRL+SHIFT+T). This will open the Snippet Manager and add a new snippet using the selected text as the snippet value.

To insert a snippet, you can use the Insert Snippet menu item or keyboard shortcut (CTRL+SHIFT+T), or more easily, just type the snippet name in the text editor, surrounded by hashtags, for example:

#newroompic#

will insert the text assigned to the snippet named 'newroompic' at the current cursor location. If the inserted text is multi-line, WinAGI will auto-indent subsequent lines to the same level as the starting insertion point.

You can also include arguments in your snippets. In your snippet value text, use argument placeholders ('%1', '%2', etc.). In the snippet shortcut, add your desired argument values in parentheses, separated by commas, following the snippet name. For example:

#exitleft(rm.LeftRoom)#

will replace all instances of '%1' with 'rm.LeftRoom' in the snippet value text before inserting it at current cursor location.

More information on snippets can be found in the Logic Code Snippets topic.

TODO Entries

To help manage tasks that need to be done during game development, WinAGI allows you to create TODO items. To create a TODO item, add a comment to your logic code followed by 'TODO: '. Everything following the colon is treated as a TODO item.

When the logic is saved, all TODO items are added to the Warnings and Errors List. You can easily navigate to TODO items by double-clicking them just like warnings and errors.

Integration with Layout Editor

The logic editor and the Layout Editor are closely integrated. When you make and save changes to a logic source file, any changes in exits (i.e. using the new.room command) will be automatically saved to the game's layout file, and will show up in the layout editor.

When changes to the layout editor are saved, WinAGI will automatically add or comment out code for any exits that have changed. See the Layout Editor topic for more information on this excellent feature.
