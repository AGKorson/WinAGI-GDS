General Settings

Resource List Type

Select the type of resource list to display as a panel on the left of the main WinAGI window. You can choose between a tree list view, or a combobox/list view. Select none to hide the resource list panel.

 

Display Resources by Number

The default for WinAGI is to display resources in the resource list by their resource ID.

If you don't want to use resource IDs, enable this option to display resources by number instead.

 

Resource Name Format

When the Display Resources by Number option is set, use the Resource Name Format drop down list to choose the format for resources. The available formats are essentially the same as those found in AGI Studio.

 

Include Resource Number on Resource List

If using resource IDs, select this option to include the resource number in the caption for each resource. This can be useful if you want to easily look up a number for a given resource, without having to select it.

 

Default Resource Directory

When a new game is created, or an existing game is opened in WinAGI for the first time, the resource directory is the subdirectory that WinAGI will create and use for source code files and exported resources.

 

Show Splash Screen

When checked, the WinAGI splash screen is shown at startup. Clear this option to skip the splash screen.

 

Open Most Recent Game

When checked, the most recently edited game will be opened when WinAGI is executed. This is very useful if you are working with one game all the time.

 

Show Warning List on Open

When a game is opened, if there are any errors or warnings, the Warnings and Errors List will be displayed if this option is set.

 

Show Preview Window

When checked, logic, picture, sound and view resources are shown in a preview window when a game is loaded.

This option is only available if the Show Resource List option is checked.

 

Selection Brings Preview Window to Top of ZOrder

When checked, choosing a resource in the resource list will force the preview window to the top of the zorder. (Meaning the preview window will move to to the front, and display on top of all other WinAGI windows.)

This option is only available if the Show Preview Window option is checked.

 

Hide Preview Window on Lost Focus

This option will hide the preview window when you are working with editors (i.e., when the preview window loses focus). This keeps the preview window from being a distraction while working on resources. When you click on a resource node or the property window in the resource list, the preview window will be re-displayed, and will move to front, on top of all other windows.

This option is only available if the Show Preview Window option is checked.

 

Automatically Export Resources

When checked, the Auto Export Feature is enabled.

 

Backup Resource Files When Removing From Game

When checked, this option will automatically export the resource being deleted to the game's resource directory. This is useful if you are not using the Auto Export feature and you want to preserve any resources that you decide to remove from your game. Note that this only affects picture, sound and view resources; logic source code is not affected, and should always remain in your game's resource directory even if the logic is removed.

 

Default MaxScreenObjects

When creating a new OBJECT file, WinAGI will use this value for the default Max Screen Object value.

 

Maximum VOL.0 Size

Most original Sierra AGI games were distributed on floppy disks. When run directly from the floppy disks (instead of from a hard drive) AGI expects each disk to include a copy of VOL.0 along with the volume file matching the disk number. Also, the first disk contained all the interpreter files, the OBJECT and WORDS.TOK files and other files needed to install the game. As a result, VOL.0 was usually much smaller than the maximum allowable size.

This property allows you to specify a smaller size for VOL.0. Since games today are rarely put on floppies (and even if they are, they are even less likely to be played directly from a floppy drive), this property is not of much use. It is provided more for nostalgia than anything else.

 

Reset Message Box Warnings

Several message box warnings in WinAGI allow you to choose an option to not show the warning anymore. Set this property to force WinAGI to reset all message box warnings so you will see them again.

Choosing this option will also reset all ignored compiler warnings to their default status.
