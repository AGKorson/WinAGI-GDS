Managing Resources

WinAGI includes commands for creating, opening, saving, importing and exporting all AGI resource types. Deleting resources from a game is done by simply selecting or opening the resource and choosing 'Remove from Game' menu option.

The way that WinAGI manages resources depends on whether or not a game is currently open:

    • If a game is open, WinAGI resource commands such as New, Import, Open, assume you are working with the resources that are actually included in your game (i.e. they are in one of the VOL files).
    • If a game is NOT open, WinAGI will only allow you to manage resources that are in individual resource files.

As you add and remove resources to your game, WinAGI will automatically expand or contract the directory (DIR) files based on the number of resources. This helps keep memory usage to a minimum if you plan to run your game with an original MSDOS interpreter.

 

All in-game resources have the following properties: resource number, resource ID, and description.

Resource Numbers

Logic, picture, sound and view resources are identified internally by the AGI interpreter by their resource number. The resource number corresponds to the index position of a resource in its directory (DIR) file. You can change the resource number for a resource by using the Renumber Resource command, which opens the Renumber Resource dialog window:

The list will only show numbers that are NOT currently in use. Select the new number you want from the list, then click OK. WinAGI will automatically update the VOL files and DIR files with the new resource number. The resource ID is NOT changed when a resource is renumbered.

If renumbering a logic resource that is marked as a room, you can also automatically update the room's picture number.

Resource IDs

WinAGI introduces the concept of resource IDs as a way to easily track your game resources (logics, pictures, sounds, views) so you can remember each resource's purpose. When browsing your resources in the Resource List, resource IDs make it is much easier to find a resource you are looking for.

WinAGI will automatically create and maintain an include file with your game's resource IDs, and will automatically add this file as an include file in all your game's logic source files.

If you do not want to use this feature, you can deselect the auto-include Resource IDs option on the Game Properties dialog.

Resource IDs improve readability of logic code. The WinAGI compiler treats resource IDs just like defined names. When it encounters a resource ID, it substitutes the appropriate resource number, just like a define entry.

For example, if view 4 in your game has the ID "vw.Snake", you can write code as follows:

set.view(o1, vw.Snake);

Or perhaps logic 94 is your death handling logic. If its ID is "lgc.DeathHandler", you can use this code to call it:

call(lgc.DeathHandler);

Maybe sound 1 is your game's theme music. Set its ID to "m.Theme" and use the following code:

load.sound(m.Theme);
play.sound(m.Theme, f99);

Resource IDs are also used in the name of logic source code files, and as the default filename for resources you choose to export (for example, DeathHandler.lgc).

Resource IDs can contain letters, numbers, and some special characters. No spaces are allowed. Resource IDs are case-sensitive, so for example "LOGIC1" is not the same as "logic1". Resource IDs must conform to the following rules:

    • ID can't be numeric
    • ID can't be an AGI action command
    • ID can't be an AGI test command
    • ID can't be a compiler keyword (if, else, goto, etc)
    • ID can't be a data type marker (v1, f2, o3, etc)
    • ID must be unique (i.e. no other resources can share same ID)
    • Name can only include letters, numbers, the underscore, and '!', '@', '$', '%' characters

Description

All resources include a description property. It is only used by WinAGI to provide information to the game programmer, and is entirely optional.

Editing Resource IDs and Descriptions

You can change resource IDs and resource descriptions in an existing resource by using the ID/Description command on the Resource menu. The Change ID/Description dialog window will be displayed:

If you change a resource ID, WinAGI will automatically update the name of the corresponding resource file in your resource directory, if one exists. For example, if you have a logic that has an ID of "Logic3", the source code file in your resource file will be named "Logic3.lgc". If you change the resource ID to "rmHallway", WinAGI will automatically change the source code filename to "rmHallway.lgc".

WinAGI can also automatically update all logics in your game with the new name. If the "Update Logics" option is set when you change a resource ID, WinAGI will search all logics in your game, and replace the old resource ID with the new ID.

Click OK after making your changes. If you attempt to enter an invalid resource ID, WinAGI will ask you to correct it before allowing you to continue.

Resource Directory

The main game files are the *DIR, VOL.*, OBJECT, and WORDS.TOK files. When working with a game, your in-game resources will be stored in the VOL files that make up your game.

The resource directory is a sub directory in your game directory where logic source code files (*.lgc or *.txt) are stored.

It is also the default directory for any exported resources.

If you have the AutoExport feature enabled, this is the directory where resource files are are automatically stored.

You can change the resource directory for this game on the Game Properties dialog.

The default resource directory (when creating new games) can be changed on the General tab of the Settings form.
