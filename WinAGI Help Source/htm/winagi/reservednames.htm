Reserved Define Names

WinAGI automatically manages define names to represent reserved values such as reserved variables and reserved flags that have pre-defined meaning in AGI, as well as several universal numerical constants. WinAGI creates and automatically adds the file 'reserved.txt' as an include file to all game logics.

If you do not want to use this feature, you can deselect the auto-include Reserved Defines option on the Game Properties dialog.

When editing logics, use the List Defines command to display the reserved defines, along with global and local, in a list box to easily select them.

Reserved Define Editor

The names that are used by the reserved defines can be edited, if you prefer different names other than the WinAGI defaults.

To change reserved define names, open the Reserved Defines Editor from the Tools menu. A dialog box will display the list of reserved defines:

You can edit the names for the listed variables, flags and constants. The values are NOT editable. Any names that are changed will be displayed in red text so they are easy to see.

Use the 'Reset' button to restore WinAGI's default values for all reserved define names.

Automatic Updates

When you save changes to the reserved defines list, you can choose to have WinAGI automatically update all logic source files in your game to reflect the new names. WinAGI will replace all instances of the old name with the new name.
