Compiling Games

As you add, delete, and edit the various resources in your game, the volume files where resources are stored eventually get more and more fragmented as WinAGI tries to find space within existing volume files to insert new and updated resources. You should compile your game before releasing it to compress the volume files, and make sure all the your logic resources are compiled from the latest changes to your source files. WinAGI provides a number of features to assist in assembling your game for testing and final release.

Compiling Game vs. Compiling Logics

Compiling a game refers to the process of rebuilding the volume files and directory files to remove any slack. Compiling a logic refers to the process WinAGI uses to convert the text based source code into binary code that can be read by an AGI game interpreter.

Compile Game

When you choose the Compile option from the Game menu, WinAGI checks that all resources are valid and completely rebuilds the game's volume files and directory files. In addition, before logic resources are added the source code for each logic is run through WinAGI's source compiler to ensure the compiled game has the most up-to-date resources. If WinAGI detects any errors (either in compiling logics, or in invalid resources), the compilation process is halted. Any warnings generated during the compilation will be displayed in the Errors and Warnings List.

You can also cancel the compilation before it completes. In this case, your original volume and directory files remain as they were before the compilation started; no changes are saved if compilation is canceled.

Compile To...

You can choose to compile a game to a new directory using the Compile To command. This option will recompile all logics, rebuild all volume files and place the resulting game files in the directory of your choice. No changes are made to any files in your original game directory.

Rebuild VOL Files

As you edit and save resources in your game, the changes in resource size will result in slack space in the volume files. To remove the slack you can use the Rebuild VOL Files command on the Game menu. Rebuilding the volume files does not recompile logics. It uses the existing resources in the current volume files and reorganizes them to eliminate slack space. As with the Compile command, the old volume and directory files are renamed in case you want to restore the original files.

Compile Changed Logics

WinAGI automatically detects when logic source files are changed, thus requiring compilation. The changed logics can be compiled all at once using the Compile Changed Logics command. When using this command, the volume files are not rebuilt. The compiled logics are stored in the first VOL file with available space.
