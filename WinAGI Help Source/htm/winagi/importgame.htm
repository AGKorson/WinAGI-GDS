Importing an AGI Game

Importing Games

Any existing AGI game (Sierra or fan made) can be imported for editing in WinAGI. To work with an existing game in WinAGI you need to first import it using the Import Game command.

A limited number of properties can be set when importing, using the Import Game dialog:

 

  • Import Directory: Select the directory of the game you wish to import. This directory must contain the game's DIR, VOL, OBJECT and WORDS.TOK files.
  • Resource Directory Name: This is the directory that WinAGI will use to store decompiled logic source files, as well as include files and exported resources. If the directory does not exist WinAGI will create it.
  • Logic Source File Extension: WinAGI will use this extension for the game's logic source files. The default is '.lgc'. Other common choices are '.txt' or '.cg', but you can use any valid file extension.
  • Auto-Include ResourceIDs, Reserved Defines, Global Defines: Use these checkboxes to change the behavior of WinAGI's auto-include features. See the Properties dialog help topic for more information.
  • Advanced: Click the Advanced button to see options to change the target character code page, or to import a game using Sierra's original syntax. These options are rarely needed, but are available if desired.

 

After importing, WinAGI will create a game property file for the game. You can then make adjustments to the game's properties using the Properties command on the Game menu. Once created, use the game property file to open the game in subsequent sessions.
