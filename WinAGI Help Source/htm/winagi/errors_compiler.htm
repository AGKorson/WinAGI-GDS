Logic Source Compiler Errors

The compiler errors identified in this list represent conditions in source code that prevent successful compilation. Errors must be fixed in order to successfully compile the source.

Compiler errors are displayed in red to easily distinguish them from warnings. Minor errors don't stop the compiler, so all minor errors in a logic are identified in a single pass. This makes it easier and more efficient to make corrections. All errors are added to the Errors and Warnings List.

Compiler errors beginning '4xxx' apply to all logics regardless of syntax being used. Errors beginning '6xxx' apply only to logics using original Sierra syntax.

For detailed information on syntax rules, see the AGI Fan Syntax and Sierra Syntax topics.

Where compiler error messages include dynamic text, such as an unknown command, or define names that have errors, the dynamic text is represented by '<' and '>' characters, for example '<definename>'.
