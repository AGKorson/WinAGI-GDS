Objects Editor

Use the Objects Editor to edit OBJECT files in WinAGI.

Objects Editor Components

Inventory Item Table

Inventory items in the OBJECT file are listed in numerical order. Each item has a description and a starting room number.

Items are numbered sequentially. Item numbers cannot be edited.

The first item (Item #0) is by default a single question mark, representing no item. This is not strictly required though, and it can be edited if desired.

Adding Items

Add an item by using the Add Item command on the Edit menu. A new item is added to the end of the item list, and is automatically selected for editing.

Deleting Items

To delete an item, select it, then use the Delete Item command on the Edit menu.

If the item is the last item in the list, it will be removed.

If it it not the last item, it is not actually deleted, since this would change the item number for the items that follow it. Instead, the item is changed to the null item (a single question mark), and its starting room is set to zero.

Use the Clear Object List menu command to delete all items. This removes all items, except item #0, hich is set to the null item.

Editing Items

To edit an item description or starting room number, double-click it (or select it, then use the Edit Item command on the Edit menu). Make any changes, then press enter to accept the changes. To cancel editing, press the ESC key.

You can change item 0 to something other than the null item, but it is usually best to leave it.

Although it was extremely rare to see in Sierra games, AGI does provide limited support for use of extended character codes. Use the Character Map Tool (on the Edit menu, or press CTRL+INS) if you want to use an extended character in an inventory item description.

Duplicate Items

If you edit an item so that it is a duplicate of an existing item, you will get a warning, and be asked if you wish to keep the duplicate.

AGI does not prohibit having inventory items with identical description text. However, the compiler will be unable to determine which item you really mean if a duplicate item is used in a logic.

If you must have duplicate items, you should refer to them by their item number (i.e. 'i1', 'i2', etc.) instead of with a literal text string.

Find and Replace

Use the Find and Replace commands on the Edit menu to search and replace inventory items. The search is limited to the OBJECTS file.

Find in Logics

The Find in Logics command on the Edit menu allows you to search for an inventory item in all of your logic resources. To use this feature, select the item you wish to search for, then choose the Find in Logics command.

WinAGI will search all logics in your game for the selected item.

Maximum Screen Objects

The Maximum Screen Objects property sets the maximum number of screen objects that can be in use at one time during the course of a game. This value is stored in the game's OBJECT file.

To edit this value, just select the text box, and enter the new value. Remember that a higher number will consume more memory. You should set this value to the minimum number necessary needed.

Encryption

In the original AGI Interpreters, OBJECT files were encrypted in version 2.411 and above.

If the OBJECT file being edited is part of a game, WinAGI will automatically set the file encryption to the correct value. If you want to manage the encryption state manually, use the Encryption command on the Resources menu that allows you to toggle the file encryption.
