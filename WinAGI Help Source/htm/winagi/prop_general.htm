General Game Properties

GameID

The WinAGI GameID is used by WinAGI to identify a game you are working on. For version 3 games, the GameID is the prefix for the directory files and volume files. The GameID defaults to "AGI" for new games.

Note: The GameID property is not directly associated with the set.game.id AGI command. Changing the GameID property will not affect any set.game.id commands in your game. For help with avoiding game ID issues, see Game IDs and Loaders.

For version 2 games, the GameID property can be up to six alphanumerical characters long. This is a limitation of the old DOS '8.3' file naming convention. The file name for saved game files was 'SG' plus the game ID with extensions of 001, 002, etc. This meant the game ID could be no longer than six characters.

For version 3 games, the GameID property is limited to five characters. The directory file for a version 3 game includes the three-character sequence "DIR" which leaves five characters available for the GameID.

The GameID property can be accessed in logic source code through a reserved define name. When logics are compiled, the reserved name is replaced with the GameID text.

Target Interpreter Version

This is the interpreter version that your game is designed to use. It must be one of the nineteen known AGI release versions. If you are creating a game from a template, this will be a read-only property. If you are creating a new blank game, you may choose any version, and WinAGI will correctly format the DIR and VOL files based on the major version selected. Please note setting this property DOES NOT copy any interpreter files to your game directory, or prevent you from running your game with any other interpreter version. It is merely used by WinAGI to manage game properties such as VOL and DIR file naming, OBJECT file encryption, and AGI command syntax.

If you are editing an existing game, changing the interpreter version from one major version to another (from 2 to 3 or vice versa) will result in a complete rebuild of VOL and DIR files to the correct format based on the new version.

For WinAGI, your choice here will only determine the style of the VOL and DIR files (based on which major version you select), and the AGI command set.

The number of AGI commands available in logic resources varies with the AGI version. In WinAGI, if you choose a version that does not support all the AGI commands, a syntax error will be generated for any commands encountered in your source code that are not supported in the selected version.

Keep in mind that version 3 games needs an interpreter file that has a built-in GameID that matches your game's GameID. You will need to modify an interpreter file with a hex editor to match your GameID, unless you are planning to run your game with an AGI emulator such as AGILE, ScummVM or NAGI.

See the AGI Interpreter Versions topic in the AGI Reference for more information on the differences in AGI versions.

Game Directory

The directory where the game files are located. For a new game, your game files will be created here. For new games, the directory must not already contain a WinAGI game file or an existing AGI game (VOL and DIR files).

If you are editing an existing game's properties, the game directory property is read-only.

Resource Directory

The name of the subdirectory in the game directory where logic source files will be stored. This is also the default directory for exporting other resources.

If this property is changed for an existing game, files in the old directory will be moved to the new directory automatically.

Source File Extension

This is the file extension that will be added by default to you game source files.

Auto Include ResourceIDs / Reserved Defines / Global Defines

WinAGI can automatically create and manage include files to add defines for resource IDs, reserved variables and flags, and global defines that need to be accessible in all logics. When enabled, WinAGI will automatically create and manage the specified include file, and add the appropriate include statement to the beginning of every logic. Each auto include option can be toggled individually if you prefer to manage these types of defines manually.

Use Layout Editor

If you want to use the Layout Editor to assist in managing how the rooms in your game are connected, enable this option.
