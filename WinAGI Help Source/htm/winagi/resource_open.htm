Opening Resources

To open a resource that already exists, double-click the resource in the resource list, use the Open Resource command on the Resources menu, or the Open Resource buttons on the main toolbar.

 

Opening Logic/Picture/Sound/View Resources

      

If a game is NOT open:

A dialog window will open, allowing you to select the resource file you wish to open. Once you select a file, it will open in the appropriate editor.

If a game IS open:

The Open Resource command only opens resources that exist in your game. A dialog window will open, allowing you to select an existing resource. Once selected, the appropriate editor will open.

You can also open a resource in your game for editing by double-clicking it in the Resource List, or by double-clicking on the Preview Window.

If you want to open a resource from a file but don't want to add it to your game (for example, maybe you want to open a view file so you can copy a loop into one of your views), use the Import command on the Resource menu.

For logic resources, the Open Resource command opens the logic's source code file. The source code file must be located in the game's resource directory. The name for the source code file must is the logic's Resource ID with the game's source file extension.

Decompiling Logics

When WinAGI opens a logic source for editing, if the source code file does not exist (such as when first importing an existing AGI game) WinAGI will decompile the logic by analyzing the raw byte codes from the logic resource and store the resulting source code in the game's resource directory.

The appearance of the decompiled code can be adjusted by changing the options available in the Decompiler section on the Logics (2) tab of the Settings window.

Opening OBJECT and WORDS.TOK files

 

If a game is NOT open:

This command opens an existing OBJECT file or WORDS.TOK file in an editor. You will be shown a dialog for selecting the file you want to open.

If a game IS open:

The open OBJECT file and open WORDS.TOK file commands only open the files in the open game.

If you want to open an OBJECT file or WORDS.TOK file other than the one in the open game, use the Import command on the Resource menu.
