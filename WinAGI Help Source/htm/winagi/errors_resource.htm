Game and Resource Errors

When WinAGI imports or opens an AGI game, it does a full scan of the directory and volume files to confirm all resources are valid. Source code files are also scanned to confirm there are no major errors. Any issues found are displayed as errors or warnings. Errors represent conditions that must be addressed before the game files and/or resources can be edited or compiled.

Detailed explanations of each error that may be encountered are provided below.

Where resource error messages include dynamic text, such as a resource type, or detailed error information, the dynamic text is represented by '<' and '>' characters, for example '<restype>'.

 

RE01: <restype> <number> is in a VOL file (<volfile>) that does not exist

RE02: <restype> <number> is in a VOL file (<volfile>) marked readonly

RE03: <restype> <number> is invalid due to file access error (<volfile>)

RE04: <restype> <number> has an invalid location (<location>) in volume file <volfile>

RE05: <restype> <number> has an invalid resource header at location <location> in volume file <number>

RE06: Unable to expand <restype> <number> at location <location> in <volfile>

RE07: Logic <number> source file (<filename>) is marked readonly

RE08: Logic <number> source file (<filename>) is invalid due to file access error

RE09: Sound <number> contains no data

RE10: Sound <number> has one or more invalid track pointers

RE11: View <number> contains no data

RE12: View <number> has no loops

RE13: OBJECT file is missing

RE14: OBJECT is readonly

RE15: File access error, unable to read OBJECT file (<errortext>)

RE16: OBJECT file contains no data

RE17: Invalid data in OBJECT file, unable to decrypt

RE18: Invalid OBJECT file header, unable to read item data

RE19: WORDS.TOK file is missing

RE20: WORDS.TOK is readonly

RE21: File access error, unable to read WORDS.TOK file (<errortext>)

RE22: WORDS.TOK file contains no data

RE23: Invalid word index table in WORDS.TOK

RE24: globals.txt file not found

RE25: globals.txt file is read only

RE26: File access error, unable to read globals.txt file (<errortext>)
