Logic Decompiler Errors

When analyzing the raw byte codes from a logic resource, if WinAGI is unable to fully decompile the resource, a decompilation error is generated. Depending on the error, the resource may be partially decompiled.

Detailed explanations of each error that may be encountered are provided below.

Where decompile warning messages include dynamic text, such as a resource type, or resource number, the dynamic text is represented by '<' and '>' characters, for example '<restype>'.

The <pos> argument points to the byte index position of the invalid value in the raw byte code. It is provided in case you wish to examine the data in a hex editor to further study the logic.

 

DE01: Logic contains no data to decompile

DE02: Invalid test command byte code (<code>) found at resource index <pos>

DE03: Invalid action command byte code (<code>) found at resource index <pos>

DE04: Block end at resource index <pos> is past end of resource; adjusted to end

DE05: Destination for 'goto' at resource index <pos> is past end of logic; adjusted to end

DE06: Expected block end does not align with calculated block end at resource index <pos>

DE07: Message section has an invalid end-of-text marker

DE08: Message <msgnum> has an invalid offset (resource index <pos>)

DE09: Invalid message section data

DE10: Unexpected end of input

DE11: Valid reference to WORDS.TOK is missing

DE12: Word group <number> does not exist in WORDS.TOK
