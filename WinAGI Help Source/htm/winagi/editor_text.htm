Text Editor

The Text Editor provides basic text editing functionality. It is useful for editing included files or any text file you wish to edit without having to leave WinAGI.

The Text Editor is nearly identical to the Logic Editor, with the following differences:

    • Text files cannot be compiled.
    • Syntax Highlighting is optional for text files. Use the Enable Text Syntax Highlighting option on the Logics tab in the Settings dialog to toggle syntax highlighting for text files.
    • The Define Tooltips feature, List Defines command and Resource Quick Edit command are only available if a text file is designated as an include file.
    • The Message Cleanup Tool is not available for text files.

Include Files

The most common use of text files is to put define statements that will be used in multiple logics in a single file and then add the contents of the that file to logics by using the #include preprocessor statement. This makes it much easier to manage defines throughout a game.

Although they are not resources, WinAGI allows you to add include files to your game to help manage them, just like game resources. They are displayed in the resource list, and can be previewed in the same way that logic source files are previewed.

IMPORTANT: Include files are not required to be added to your game to use them in a logic source. Any #include statement in a source file will work even if the file is not added to you game as an include. Adding them to a game helps manage the files and allows WinAGI to provide features such as showing define tooltips and define lists.

To add a text file as an include, open the file in a text editor, then choose the Add to Game command on the resource menu. You can remove an include file by using the Remove from Game command.

Designating a file as an include does not automatically add it to logic source code. You have to add it with the #include statement. Similarly, removing a file from the includes list does not delete it, nor remove #include entries.

Auto Include

When using the Auto Include feature for resource IDs, reserved defines or global defines, the files will be listed as includes in the game's resource list, but they are handled differently. "globals.txt" can only be viewed and edited in the Global Defines Editor. "reserved.txt" can only be edited in the Reserved Defines Tool. "resourceids.txt" is managed internally by WinAGI and cannot be edited or viewed.
