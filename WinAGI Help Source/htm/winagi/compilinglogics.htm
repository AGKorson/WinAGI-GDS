Compiling Logics

WinAGI tracks the compile state of your logic source files. If the source changes the Compiled property of the source code is reset to FALSE. If you are using the preview window, the logic will appear with a light red background to indicate it is not compiled. In the resource tree, uncompiled logics are displayed in red text.

You can easily compile all logics that are currently marked as uncompiled by choosing the Compile Changed Logics command from the Game menu.

Individual logics can be compiled when opened in a Logic Editor by choosing the compile toolbar button or menu item.

Scope and Hierarchy of Defines

When the compiler encounters an argument value that is not a number, argument marker or literal string (i.e. '99', 'v1', 'f5', "look" etc.) it will look for a define name in the following order:

    1. Locally defined names for the logic being compiled
    2. Global define names
    3. Resource IDs
    4. Reserved Names (a special group of defines for reserved values that have preset purposes in AGI)

This means you can override a reserved name with a resource ID, or a globally defined name with a locally defined name, etc.

WinAGI will generate warnings for defines if the name is a reserved name, or if the value is already defined by another reserved name.

Error Detection Level

When a logic source file is compiled, WinAGI will check for errors and warnings. The error detection level of the compiler can be adjusted on the Logics tab of the Settings window. There are two possible settings:

    • Low - as long as the compiler can create executable code, it won't return errors or warnings.
    • Medium - the compiler will create code, but if it detects conditions that could potentially cause errors in the logic once it is executed by AGI, it generates warnings indicating where potential problems exist.

If any errors are found, they will be displayed in the Warnings and Errors List.

In addition to errors, any unusual conditions or items of interest will also be displayed as warnings in the Warnings and Errors List.

If no errors are encountered, WinAGI writes the compiled resource to your game files.

Support for Original Sierra Source Code Syntax

The syntax rules used by WinAGI (which are compatible with the original fan syntax created in AGI Studio) differ slightly from the syntax rules used by Sierra's original MSDOS based compiler CG.EXE. WinAGI does provide the option to use Sierra original syntax rules instead of the fan-based syntax rules. To access this feature, select the Use Sierra Syntax option on the Game Properties Dialog.
