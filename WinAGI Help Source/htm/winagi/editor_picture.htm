Picture Editor

Use the Picture Editor to create and edit AGI Picture resources.

See the Drawing Pictures topic for help with using drawing command tools to create pictures.

See the Manipulating Commands topic for help with manipulating commands to edit and adjust drawing commands.

To use an image file as a background that you can trace over to draw your picture, see the Picture Background Image topic.

See the View Testing Mode and Text Testing Mode topics for help with WinAGI's testing modes.

Picture Editor Components

For a description of the various components of the Picture Editor window, click on the area of interest:

 

 

Toolbar

The toolbar allows you to switch between edit mode for drawing pictures and text and view test modes. You can also select drawing tools for use on the drawing surface. Tools for manipulating selected drawing commands, such as undoing, cutting, copying, pasting, and flipping are also available here.

 

Command List

The draw commands that make up the picture are listed here. The currently selected command will determine where subsequent commands are added when drawing tools are used; when you click on a command in the list, the first new draw commands will be inserted BEFORE the selected command (it pushes all commands down). After that, subsequent commands are inserted AFTER, with each command following the previous one.

The End command is just a placeholder to mark the end of the picture; it can't be deleted or moved.

Select multiple commands by dragging the mouse over commands in the list (or by holding down the shift key and using cursor keys).

When selected, a bounding rectangle is shown in the drawing area to indicate the extent of commands selected.

Errors in Picture Resources

While it is extremely rare, you may encounter picture resources with invalid color values or invalid draw commands. Sierra was notorious for failing to check for errors in AGI, which includes the drawing functions that read the data from picture resources. In most cases, AGI ignores bad data that it encounters, so the picture still draws as expected. But when editing those resources, the errors need to be dealt with. WinAGI displays invalid commands in red text and marks any invalid data that it encounters with the tag 'ERR'. WinAGI ignores the bad data in the same way that AGI does. You should delete/replace that bad data if you intend to use the picture in your game. (For an example of bad picture data, check out Manhunter I, picture 86.)

Because AGI did not support plot commands in versions 2.089 and 2.272 (and only partially in version 2.411), picture resources in those versions will show plot and plotstyle commands in red and marked as errors as well. The commands are ignored in those versions, but should be removed as they waste space.

 

Coordinate List

When a command is selected in the command list, all the coordinates for the command are shown in the coordinate list. Select a coordinate to edit it. A flashing cursor will be shown on the drawing surface that can be dragged to reposition the coordinate.

Cursor Options

WinAGI has two cursor marker options. The original WinAGI flashing cursor highlights just the selected coordinate with a color changing block which is easy to see and select.

   

There is also an 'X' style cursor, which highlights all coordinates in the command with a small 'X' and a small flashing square over the selected coordinate. In the 'X' style cursor mode, you can select other coordinates in the command by clicking on the 'X' in the drawing surface. To change between the two cursor modes, go to the Picture Settings options page.

The color of the 'X' can be changed by pressing CTRL and clicking on the desired color on the Pen Color Palette. CTRL + left-click will change the color of the markers on the visual screen; CTRL + right-click will change the color of the markers on the priority screen.

 

Drawing Surface

The drawing surface can display the visual picture, the priority picture or both. Scroll bars are included as needed.

You can draw commands on either picture surface, regardless of current pen settings. WinAGI will add the commands to the correct picture (visual or priority) automatically. For example, if you want to trace something to add control lines, you can position the cursor on the visual screen to aid in tracing, but the command line will only be added to the priority screen.

Splitter Bar

When both pictures are visible, drag the splitter between them to change the size of each picture.

When moving the splitter, you will notice that it stops moving when a minimum window size is reached for either picture. If you continue dragging the mouse all the way to the top or all the way to the bottom, the splitter will 'snap' up or down, which will completely hide one of the pictures.

You can then swap the visual and priority pictures by right-clicking the drawing surface and choosing the swap picture command.

To switch back to split picture display, just drag the splitter back toward the middle of the drawing area.

Priority Bands

To aid in placing picture elements, you can display thin lines, color coded to the respective priority value that shows where each priority band begins. You can toggle these by choosing Show|Hide Priority Bands command on the Edit menu.

If editing a picture that is in a game for a version that supports the set.pri.base command, you can also adjust the priority base using the Priority Base command on the Edit menu; WinAGI will then draw the priority band lines according to the selected base. This feature is automatically disabled for games that are in earlier version 2 games that don't support adjusting the priority base.

Drawing Pictures

See the Drawing Pictures topic for details on how to add draw commands to pictures.

 

Scale Control

   Use the scale control toolbar buttons to adjust the scale of the pictures. You can also use the mouse wheel to scale in or out.

 

Pen Color Palette

The current status of the drawing pens is shown on the color palette. The letter 'V' for visual and 'P' for priority provide a visual display of the current pen status.

To insert a pen color change, click on the desired color. (The 'X' box on the left represents setting the pen to off). Left clicking will change the visual pen color. Right clicking will change the priority pen color.

If the currently selected command is a pen command of the same type, WinAGI will just change the current pen to the newly selected value. This helps eliminate unnecessary commands. If you want to force a new command instead of changing the current command hold down the SHIFT key when selecting the new color.

The pen palette can also be used to change the color of the coordinate markers when using the 'X' coordinate mode by pressing CTRL and clicking on the desired color. CTRL + left-click will change the color of the markers on the visual screen; CTRL + right-click will change the color of the markers on the priority screen.

Data Position

The position of the selected command in the picture data is provided here. It can't be edited.
