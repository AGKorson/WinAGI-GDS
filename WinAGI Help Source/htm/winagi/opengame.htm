Opening Games in WinAGI

Opening Games

Use the Open Game menu command to open a WinAGI Game property file and its associated game files for editing. WinAGI expects to find the game's VOL and DIR files in the same directory as the WinAGI Game File (You can also open a game by double-clicking its game property file in the Windows explorer.)

Most Recently Used Files

The last four games to be opened in WinAGI GDS are listed on the Games Menu. You can use these shortcuts to quickly load a recent game, without having to use the Game Open menu item.

You can also have WinAGI automatically open the last game you were working on the last time you closed WinAGI. You can enable or disable this feature by checking or clearing the "Open most recent game on startup" option on the General tab of the Settings window.

Game and Resource Errors and Warnings Check

When a game is imported or opened, WinAGI does a full scan of the directory and volume files to confirm all resources are valid. If there are major errors, such as missing game files, or misconfigured game property file, the game will not load, and an error message will be displayed to explain the reason. If the game does load, minor resource errors will be displayed in the Warnings and Errors List. Issues that are not errors but should still be addressed are displayed as warnings. A complete list of errors and warnings that WinAGI can detect in resource files is provided in the Game and Resource Errors and Game and Resource Warnings topics.

Source code files are also scanned to confirm there are no major errors.

 

Upgrading from Previous WinAGI Version

Game files from version 2 of WinAGI are fully compatible with this new version. When first opened in WinAGI version 3, a version 2 game file and all logics will be updated to include the additional features of WinAGI version 3.

In version 2, WinAGI used MSDOS character encoding for saved source files. Version 3 will convert the source code to use default Windows encoding, which makes the files more usable by outside editors and viewers.

In addition, all source files will be updated to add auto-include statements. (WinAGI Version 2 did not use include files to manage resource IDs, reserved defines, and global defines, which made them difficult to manage outside of WinAGI.)
