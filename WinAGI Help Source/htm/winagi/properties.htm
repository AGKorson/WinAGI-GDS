Game Properties

All WinAGI game editing sessions use a game property file (usually the game's GameID with a '.wag' file extension) that contains the properties WinAGI uses to manage the game during editing. This file must be located in the game's root directory with its DIR and VOL files. It is a text file, but you should not try to edit it manually outside of WinAGI unless you fully understand the purpose and structure of the various entries.

The Game Properties dialog can be used to make changes to game properties after the game has been created or imported.

The Game Properties dialog allows you to change/update several properties of your AGI game. Properties are sorted by topic and displayed on four tabs. If the resource tree is visible, you can also access most game properties directly in the properties window of the resource tree.

General Properties

On the General properties tab, you can set the WinAGI GameID, target Interpreter version, game directory and source file sub-directory name. You can also specify the default logic source file extension, and if your game uses WinAGI's built-in reserved defines and the Layout Editor.

Version Properties

On the Version properties tab, you can specify the game's designer, version and other descriptive properties.

Platform Properties

When you want to test your game, use the properties on the Platform tab to configure an AGI engine to run your game.

Advanced Properties

The advanced properties tab displays options for foreign language font support and alternate logic compiler settings. These settings only need modification in very rare situations. Most games can ignore this tab.
