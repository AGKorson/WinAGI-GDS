Compiler Error Level

The error detection level of the compiler determines how the compiler responds when it detects errors or ambiguity. There are two settings:

    • Low - as long as the compiler can create executable code, it won't return errors. No checks are made for conditions that will or may cause runtime errors.
    • Medium - the compiler will create code, but if it detects conditions that could potentially cause errors at runtime when executed by AGI, it generates warnings indicating where potential problems exist.

 

Auto Display Warning List

When this option is set, if the Warning List is not already visible, it will be automatically be displayed whenever a warning or error entry is added such as during logic compilation.

 

Auto Include Default Options

These settings control the default values for the auto-include defines properties when creating new games:

Always Include Resource IDs

This option sets the default value for the Include Resource IDs property for new games. If you never intend to include Resource IDs in new games you create you can clear this option. (You can still enable the property when creating each new game; this option only affects the default value when the New Game properties dialog is first displayed.)

Always Include Reserved Defines

This option sets the default value for the Include Reserved Defines property for new games. If you never intend to include reserved defines in new games you create you can clear this option. (You can still enable the property when creating each new game; this option only affects the default value when the New Game properties dialog is first displayed.)

Always Include Global Defines

This option sets the default value for the Include Global Defines property for new games. If you never intend to include global defines in new games you create you can clear this option. (You can still enable the property when creating each new game; this option only affects the default value when the New Game properties dialog is first displayed.)

 

Globals Editor Settings

The globals editor is used to edit defines that are available to all logics in a game.

Show Comment Column

When this option selected, the comment column will be visible by default every time you open the globals editor. If you never want to use comments, you can deselect this option so the comment column won't be visible. You can always show/hide the comment field as you wish once the editor is open.

 

Decompiler Settings

These settings control how WinAGI decompiles existing logics:

Display Messages After Code

If messages are shown as literal strings in commands, set this option to also list all messages at the end of the decompiled code, as message declarations, using the #message preprocessor statement.

Show Alternate Syntax

Some commands have alternate syntax. For example:

assignn(v200, 5);

can also be represented as

v200 = 5;

When this option is enabled, WinAGI uses the alternate syntax for these commands. See the Syntax Rules topic for the complete list of alternate syntax for action commands and for test commands.

Show Reserved Variable and Flag Names

When this option is enabled, the decompiler inserts reserved define names for variable and flag arguments that are reserved for use by AGI.

Display Messages by Number

When this option is enabled, the decompiler will insert literal strings as message arguments in decompiled code, i.e.,

print("This is a message.");

When unchecked, message arguments will be represented by their argument marker (i.e. 'm1', 'm2', etc.)

Display Inventory Items by Number

When this option is enabled, WinAGI will use the text of inventory items as the argument value for relevant commands in decompiled code. When unchecked, inventory items will be represented by their argument marker (i.e. 'i1', 'i2', etc.)

Display Words by Number

When this option is enabled, WinAGI will use the text of first word in the word's group as the argument value in said commands. When unchecked, vocabulary words will be represented by their group number.

Decompiler Code Style

WinAGI currently supports three different styles for formatting output code when decompiling:

    • Default

      • open braces '{' placed on new line, indented

      closing braces '}' placed on new line, indented

      else keyword placed on new line, not indented

    • Alternate 1

      • opening braces '{' placed on same line as if or else keyword

      closing braces '}' placed on new line, not indented

      else keyword placed on new line, not indented

    • Alternate 2

      • opening braces '{' placed on same line as if or else keyword

      closing braces '}' placed on new line, not indented

      else keyword placed on same line as closing brace

The style Preview button will display sample decoded text in the selected style
