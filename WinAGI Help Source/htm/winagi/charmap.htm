Character Map Selector Tool

Use the Character Map Selector Tool to easily insert extended characters when editing AGI resources.

Extended Character Support

Although it was extremely rare to see in Sierra games, AGI does provide limited support for use of extended character codes that allow display of extended characters (characters 128 - 255) from the MSDOS default code page (437 - US/English). Most modern text editors default to a modern code page (Unicode or Windows standard) which can result in displayed text showing garbled characters for extended characters. However, WinAGI automatically supports code page 437 and will convert byte values in AGI resources to the correct character glyphs (and also convert extended character glyphs to the correct byte value when adding them to resources). If a character is detected that is not supported in the current code page, WinAGI will insert a '?' character in its place.

Because of the way AGI draws character glyphs on screen, extended characters will not display properly in print messages or menus. When using the display command on the graphics screen, extended characters will only display correctly if the background color is black. On the text screen, extended characters will always display correctly.

Character Map

WinAGI includes a Character Map tool that can be used to easily add extended codes to resource editors and tools that deal with text to be displayed or manipulated in AGI.

In editors that use text, the tool can be accessed by using the 'Character Map' menu item on the Edit menu, clicking the Character Map toolbar button (on Logic and Text Editors only), or by pressing CTRL+INSERT. Double-click any character to add it to the selection box. Clicking the Insert button will add the selected characters to the resource editor at the current cursor position. Non-printable characters (those with byte values <32, 127, 255) don't have glyphs in most modern fonts, so WinAGI automatically inserts a compiler escape code for these characters.

Keep in mind that even though AGI will accept extended character codes in AGI game text, properly displaying them in a game can be a challenge. See the Extended Characters topic for more information.
