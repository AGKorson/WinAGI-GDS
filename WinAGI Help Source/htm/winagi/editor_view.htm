View Editor

The WinAGI view editor makes it easy to create and edit view resources.

See the Managing Loops and Cels topic for help with adding, deleting, cutting and copying loops and cels.

See the Drawing View Cels topic for help with drawing cels, and bitmap manipulation.

View Editor Components

Click on the area of the View Editor window for detailed information about that component:

 

Toolbar

The toolbar provides easy access to several editing functions, including cut, copy, paste, delete and flip. You can adjust the scale of cels.

The toolbar is also used to select drawing tools when drawing cels.

View Tree

The view tree is a hierarchical tree list of the view, and all its loops and cels. Depending on what node you select, the properties window will display appropriate set of properties.

You can select the view node, a loop node or a cel node depending on what you want to do. You select a node by either clicking on it, or by using arrow keys to step through the tree.

Selecting a cel will also display the cel in the cel drawing area, allowing you to draw and edit cels.

Cel Drawing Area

The cel drawing area is where you actually draw the cel. See the Drawing Cels topic for more detailed information.

 

Scale Control

   Use the scale control toolbar buttons to adjust the scale of the cel. You can also use the mouse wheel to scale in or out.

The drawing area includes a grid to assist in drawing and aligning pixels. The grid can be toggled off and on from a menu item on the edit menu. You can set the default grid setting to 'off' on the View Settings tab on the Game Settings dialog.

 

Loop Preview Window

The preview window lets you see in real time how your loops will look when animated. As you make changes to cels, the preview window will automatically update to show the changes. There are controls to change the scale of the displayed preview, adjust alignment, and show or hide the transparency color.

The preview background color can be changed by double-clicking on the preview background area (useful to get a feel for what a view might look like in a game). Choose one of the standard AGI colors, or use the preview window default color. For more thorough testing, you can use the View Test mode in the Picture Editor to see exactly how your view will interact with a particular picture.

You can toggle the display of the preview window by choosing Show/Hide Preview from the Edit menu. Also, you can set the default state by changing the Show Loop Preview option on the View Settings tab of the Game Settings dialog box to show/hide the preview window when a new view editor is opened.

Palette

The color palette is used to set the pen colors used when drawing cels. Left or right click on the palette to change the color for the respective button. The current colors are marked with an 'L' for the left button, and an 'R' for the right button.

The transparency color of the current cel is indicated by a 'T' on the palette. You can easily change the transparency color by pressing Ctrl and left-clicking.

Properties

The properties window displays properties associated with the currently selected node in the Cel List. The following tables describe the properties available for each:

When the View Resource node is selected:

Property Description Read Only?
ID The Resource ID for the view. Use the Edit ID/Description option on the Edit menu to change the ID. Y
Description The resource description for the view. Use the Edit ID/Description option on the Edit menu to change the Description. Y
ViewDesc The view description for this view. Used only for views that are displayed when you are showing inventory items with the show.obj command. N

 

When a loop is selected:

Property Description Read Only?
Mirrored TRUE if this loop is a mirror of another loop in the view. This property can't be edited directly; it updates when the MirrorLoop property changes. Y
MirrorLoop The loop which is the mirror of this loop. If the loop does not have a mirror, this property displays none. N*
Cel Count The number of cels in the loop. Y

* If the view is from a version 2.089 game, the MirrorLoop property is readonly, because that version does not support mirroring.

 

When a cel is selected:

Property Description Read Only?
Width Width of the cel in pixels. N
Height Height of the cel in pixels. N
TransCol The transparency color for this cel. This color will be 'masked out' when the cel is drawn on the AGI game screen. N

Although it was extremely rare to see in Sierra games, AGI does provide limited support for use of extended character codes. Use the Character Map Tool (on the Edit menu, or press CTRL+INS) if you want to use an extended character in the ViewDescription property.
